## 关键认知

### 一句话定位

> **「把塔防做成卡牌 Roguelike 长征」** — 怪物火车 × 月圆之夜 × 塔防的融合体。
>
> 每局 Run 从随机卡组开局，连闯 8 关 + 终战 Boss，关间商店 vs 秘境二选一，水晶 HP 全程继承，水晶碎裂从第 1 关重开。

### 市场差异化（与传统 TD / 卡牌 Roguelike 对比）

- 阵容来源：**本局随机抽 12 张卡组**（每次玩都不同），不是固定塔池或卡组构筑。
- 关卡组织：**连闯 8 关 + 终战的 Roguelike 长征**，不是独立关卡。
- 关卡间策略：**商店 vs 秘境二选一**，影响下一关，不是无策略也不是节点节点选。
- 失败代价：**从第 1 关重开，但永久卡池保留**，不是单关重打也不是全部清零。
- 长线驱动：**永久卡池解锁 + 流派构筑 + 最远闯关记录**，不是三星也不是无尽。

### 参考竞品锚点

- **怪物火车 (Monster Train)** — 核心参考（卡组构筑 + 楼层放置 + Pyre 血量持续继承）
- **月圆之夜** — 秘境参考（关间事件房间 + 风险收益博弈）
- **Kingdom Rush** — 关卡设计参考（精致度标杆）
- **Slay the Spire** — 永久解锁参考
- **皇室战争** — 部署交互参考（手牌拖放）

### 目标用户结构

| 层 | 占比 | 特征 |
|---|---|---|
| 核心用户 | 70% | 25-35 岁，喜欢 Roguelike 卡组构筑，单局 30-45 分钟 |
| 次要用户 | 20% | 18-24 岁学生，几何美术 + 收集卡池 |
| 边缘用户 | 10% | 轻度休闲，前 2 关刻意降低难度做入门引导 |

### 单局 Run 闭环（高阶视图）

```
主菜单 → 抽 12 张卡组 → 关卡 1（8-10 波）→ 关间节点（商店 OR 秘境）
       → 关卡 2 → 关间节点 → … → 关卡 8 → 终战 Boss
       → Run 胜利 / Run 失败（水晶碎裂） → 结算火花碎片 → 主菜单
```

详细状态机：`Drafting → Battle ⇄ WaveEnd → InterLevelChoosing → (Shopping | MysticEvent) → Battle … → FinalBossBattle → RunVictory / RunDefeat → RunResult → MainMenu`。

### 三大支柱（v3.0 创新）

1. **卡组随机性** — 12 张卡决定本局流派（近战墙流 / 法术爆发流 / 生产抗压流），重玩性来自组合爆炸。
2. **关间节点抉择** — 商店 vs 秘境二选一是 Rogue 决策核心。
3. **血量继承张力** — 水晶 HP 全程不重置，前期失血会在后期付出代价。

### 验收文档（01-acceptance-criteria）的角色

- 是产品「做完」的判定基准，每个验收项可执行、可观测。
- v3.0 全文重写后保留了 v1.1 仍然适用的项（如基础渲染、网格、波次推进），并新增了 v3.0 形态特有的回归项（手牌区、关间面板、商店、秘境、卡池解锁、Run 结算、火花碎片）。
- 修改设计时必须同步检查验收清单，确保新增/废弃项在文档中有明确登记。

## 实现侧提示

- **不要在 00-vision 中加数值**。数值都去 `50-mda`。00-vision 只描述定性范畴（如「单局 30-45 分钟」「12 张卡」）。
- **任何宣传性表述需先在此层定稿**，再向下渗透到玩法 / UI 文档，避免「市场材料 ≠ 实际游戏」的偏差。
- **验收清单是闭环工具**：每完成一项功能 → 勾选验收项 → 写 dev-log，三件一起。
