## 设计哲学

### 几何美术 + 程序化生成

**没有美术外包**。所有视觉元素由 PixiJS `Graphics` API 程序化绘制（直线、圆、贝塞尔），用**复合几何**组合出复杂形态：

- 角色 = 头（圆） + 身（矩形/六边形） + 武器（线段 + 三角形） + 装饰。
- 等级升级 = 改颜色 / 加装饰几何 / 改尺寸 / 加发光效果。
- 优势：策划改配置 = 视觉自动变。
- 缺点：风格强约束，必须接受"几何感"美术。

### 符号优于文字

按钮、状态、效果用**几何符号 + 颜色**表达，文字仅做辅助：

- 攻击 → 红色三角形 + 数字。
- 治疗 → 绿色十字 + 数字。
- 减速 → 蓝色雪花叠加单位。

这条原则在 `00-vision/00-game-overview.md` 提出，在本层落地。

## UI/UX 结构（40-ui-ux.md）

### 主菜单 → 关内 → 关间 的界面切换

```
[MainMenu]
  ├→ [开始新 Run]
  ├→ [继续上次 Run]（如有关间存档点）
  ├→ [卡池界面]（永久解锁 + 塔科技树）
  └→ [设置]

[Battle]（关内）
  ├→ HUD: 水晶 HP、能量 E、金币 G、波次进度、关卡名
  ├→ 手牌区（底部）：4 张默认，最大 8 张
  └→ 战场（中央）：拖卡部署区

[InterLevel]（关间）
  ├→ [商店]：买卡 / 升级非塔卡 / 移除卡 / 恢复 HP / 路径重置券 / 碎片包
  └→ [秘境]：文字事件 + 风险收益选项

[RunResult]
  └→ 英雄分、亮点回放、火花碎片入账 → 返回 MainMenu
```

### 手牌区（v3.0 新增，本作核心交互）

- **位置**：屏幕底部固定。
- **交互**：触摸/鼠标拖动卡牌 → 拖动中显示射程预览 + 落点合法性提示 → 释放部署。
- **能量条**：手牌区上方，显示当前能量 / 上限。
- **手牌满**（>= 上限）：抽到的新卡进弃牌堆。

### 关间面板（v3.0 新增）

二选一的视觉表达：左商店 vs 右秘境，玩家点击其一进入对应界面。

### 商店 UI

- 4-6 个商品槽（商品类型见 `10-roguelike-loop §3.2`）。
- 显示金币余额、刷新按钮、离开按钮。

### 秘境 UI

- 类似月圆之夜：文字情境 + 2-3 个选项卡 + 选择后立即结算 + 离开。

### 卡池界面（v3.1 重写）

- 显示玩家已解锁的所有卡。
- 点击塔卡 → 进入**塔科技树视图**（路径互斥的节点图）。
- 用火花碎片解锁未解锁的卡 / 科技树节点。

## 响应式布局（41-responsive-layout.md）

### 锚点定位系统

设计分辨率 **1920×1080**，但要支持多种屏幕比例（16:9 / 16:10 / 18:9 / 21:9 等）。

**做法**：用**锚点 + 偏移**而不是绝对坐标定位 UI 元素。

```
锚点（如 BOTTOM_CENTER）+ 偏移（如 dx=0, dy=-50）
   → 实际像素位置随屏幕尺寸自动计算
```

### v3.0 的 9 类锚点

41 文档定义了 9 类锚点（具体见原文），涵盖：手牌区、关间面板、商店、秘境、卡池、HUD 四角、设置按钮、关卡名标题。

### 鼠标/触摸点击映射

- 屏幕坐标 → 通过锚点反算 → 判断属于哪个 UI 区域。
- UI 区域**优先于战场**：玩家点 UI 按钮时不会触发战场点击。

## 美术资产规范（42-art-assets.md）

### 11 层场景分层

42 文档把场景从背景到前景分 **11 层**（与 `45-layer-system.md` 的「6 层垂直空间层级」不是同一概念，详见下文区分）。

> ⚠️ **45-layer-system 是垂直方向的空间层（地下 / 地面 / 空中 / UI 等）；42-art-assets 的 11 层是渲染顺序（Container 添加顺序），覆盖更细。** 不要搞混。

### 复合几何

详见前文「设计哲学」。

### 等级升级视觉

塔的等级提升通过**几何修改**表达，不是替换素材：

- L1 → L2：底座加一圈装饰、塔身换色。
- L2 → L3：加发光特效、武器尺寸放大。

### 卡牌视觉规范（v3.0 新增 §13）

每张卡的视觉布局：

- 卡背（统一蓝色调） / 卡面（按稀有度配色）。
- 顶部：能量消耗（左上）+ 稀有度（右上）。
- 中部：单位/法术的**复合几何头像**。
- 底部：卡名 + 简要效果描述。

## 场景表现增强（43-scene-decoration.md）

- **动态环境生物**：地图上有不参与战斗的小动物（鸟、鱼、蝴蝶），增加生动感。
- **全屏环境特效**：天气与时段相关的全屏滤镜（雨/雪/雾/夜）。
- **静态装饰物升级**：玩家升级卡池/科技树时，背景的装饰物（如远景城堡、地标）会升级演化，给玩家长线进度可视化。

## 视觉特效（44-visual-effects.md）

### PixiJS 图层结构

- **Container（普通图形）**：Graphics、Text 等。
- **ParticleContainer（粒子）**：用于天气、爆炸、火焰等大量同质粒子，性能远优于 Container。
- **Filter（滤镜）**：用于全屏特效（暗化、模糊、色调）。

### 粒子系统

44 文档列出常用粒子预设（爆炸、火焰、闪电、治疗光、寒冰碎片、毒雾），由 `ParticleRenderer` 统一管理。

### 过渡动画

- 关卡切换：淡入淡出 + 摄像机平移。
- 卡牌入手：从牌堆飞到手牌区。
- 卡牌打出：从手牌区飞到目标位置 + 释放特效。

## 层级系统（45-layer-system.md）

### 6 层垂直空间层级

> 注意：这是**单位所在的高度层**，不是**渲染顺序**。一架飞龙在"空中层"和地面塔不在同一射程内（除非塔有反空攻击）。

| 层 | 包含 | 谁能打谁 |
|---|---|---|
| 地下 | 矿工、地穴 | 地下 / 地面（部分） |
| 地面 | 大多数单位 | 地面 / 空中（如有 anti-air）/ 地下（部分） |
| 空中 | 飞龙、风灵 | 空中 / 地面（如有 anti-air） |
| 水面 | 船 / 水陆两栖 | 水面 / 地面（视情况） |
| 投射物层 | 弹道飞行中 | 投射物穿透 / 拦截 |
| UI 层 | HUD / 手牌区 / 弹窗 | 不参与战斗 |

### 层级交互规则

- 单位攻击目标必须满足**层级匹配**：地面塔不能打空中，除非配置了 `canTargetAir: true`。
- 渲染时层级影响 Y 排序：地下 < 水面 < 地面 < 空中 < 投射物 < UI。

## 音效系统（46-audio.md）

- **63 个音效**（v3.0 新增 24 个用于卡牌、关间、商店、秘境、碎片相关交互）。
- 分类：单位（攻击/死亡/技能）/ 玩家操作（出卡/拖动）/ UI（按钮/翻页）/ 环境（天气/波次）/ 系统（结算/解锁）。
- 触发方式：在 ECS 系统或 UI 事件中调用 `playSound(id)`，由 `AudioSystem` 统一管理音量与重叠去重。

## 实现侧关键约束

1. **UI 用锚点定位，禁止硬编码绝对坐标**。要 1920×1080 不报错 ≠ 真的支持多比例屏幕。
2. **UI 区域点击优先于战场点击**。`InputManager.flush()` 中先判 UI hit。
3. **粒子用 ParticleContainer，不用普通 Container**。天气 + 爆炸同屏时性能差异显著。
4. **视觉等级升级靠几何修改，不靠素材替换**。新增视觉等级时同步更新 42 文档的等级表。
5. **音效要走 `playSound(id)` 统一接口**，禁止直接 `new Audio().play()`。
6. **卡牌视觉规范固定**：能量、稀有度、头像、卡名、描述的布局参数在 42 §13 锁定，所有卡共享同一模板。
