## 核心结构：Run 长征循环

### 三个层级的时间尺度

| 时间尺度 | 范围 | 玩家关注 |
|---|---|---|
| **单波** | 30-60 秒 | 抽手牌 → 拖卡部署 → 看敌人按路径走 → 全清结算 → 下一波 |
| **单关** | 5-8 分钟（8-10 波） | 关卡主题 + 路径 + 波次组合 + 关底 Boss + 水晶 HP 折损 |
| **单 Run** | 25-40 分钟（8 关 + 终战） | 卡组随机性 + 关间商店/秘境抉择 + 水晶 HP 全程继承 |

### 8 关 + 终战的结构

- 第 1-2 关：低难度入门，介绍机制。
- 第 3-7 关：递增难度，玩家在商店/秘境中构筑流派。
- 第 8 关：尖兵关，验证流派强度。
- 终战 Boss：超长单波，混合所有敌人，**无关间节点接续**。

权威：`13-map-level.md`。

### 卡牌系统（10-roguelike-loop §2）

- **卡牌定义**：`CardConfig { id, name, type: 'unit'|'spell', energyCost, rarity, unitConfigId?, spellEffectId?, spellSubtype? }`。
- **类型分支**：
  - `type='unit'` → 引用 `unitConfigId`（沿用 21-unit-roster 中的单位定义），可生成单位卡 / 建筑卡 / 陷阱卡 / 生产卡。
  - `type='spell'` → 引用 `spellEffectId`，五种子类：damage / control / buff_instance / buff_card / utility。
- **手牌区**：默认 4 张，最大 8 张；跨波保留。
- **抽卡时机**：每波开始抽满。

### 关间节点（10-roguelike-loop §3）

每关结束后**二选一**（不是 3 选 1）：

- **商店**：花本局金币买卡 / 升级非塔卡 / 移除卡 / 恢复水晶血量 / 买路径重置券 / 买碎片包（v0.6 新增）。⛔ **塔卡的卡级升级在 v3.1 已移除商店**，转到关外科技树。
- **秘境**：随机事件，文字情境 + 风险收益选项。

### 终战 Boss（10-roguelike-loop §4）

- 第 8 关后**直接进入**，无关间节点。
- 超长单波，混合所有敌人。
- 击败 → Run 胜利 → 结算英雄分 + 火花碎片。

## 三资源体系（11-economy.md）

| 资源 | 范围 | 来源 | 消耗 | 互转 |
|---|---|---|---|---|
| **能量 E** | 关内单波 | 每波 +5 恢复，上限 10 | 出卡部署（1-10） | ⛔ 不可转 |
| **金币 G** | 整局 Run | 击杀敌人 + 波次结算 | 关间商店 / 秘境 | ✅ Run 结束未花完按 1:1 → 碎片 |
| **火花碎片** | 跨局 meta | Run 结束按到达关卡发放 | 主菜单永久解锁卡 / 塔科技树节点 | ⛔ 不可转 |

**核心设计**：三资源**作用范围、来源、消耗都不交叉**，认知边界清晰。

## 地图与关卡（13-map-level.md）

- **网格**：21×9 = 189 格。
- **6 个 PRNG 流**：地图、波次组成、怪物属性扰动、关间事件、秘境事件、抽卡 —— 各流独立 seed，便于回放与调试。
- **路径模型**：图模型（节点 + 边），支持单/多路径与分叉。
- **波次结构**：每关 8-10 波，每波由怪物组合 + 难度乘数定义。

权威：`13-map-level.md`，关卡 YAML schema 见 `60-tech/64-level-editor.md`。

## 游戏模式（12-game-modes.md）

- **Run 模式（仅此一种）** — v3.0 删除了所有独立关卡模式与无尽模式。
- **关卡内难度 4 阶段**：每关内按波次推进，难度乘数递增（开局缓 → 中期硬 → 关底 Boss）。

## 天气系统（14-weather.md）

- **5 种天气**：晴 / 雨 / 雪 / 风 / 雾（具体效果见原文）。
- **数值影响矩阵**：天气**仅影响战斗**（射程、命中、攻速等），**不影响经济**（金币 / 能量 / 碎片）。
- v3.1 audit 确认：天气与经济解耦的设计仍然成立。

## 实现侧关键约束

1. **手牌不重置**：每关结束保留手牌进入下一关，**不是**每关重抽。
2. **能量每波回满**：不是每关回满，也不是累积制。
3. **水晶 HP 跨关继承**：是 Run 级别状态，存在 `OngoingRun` 中。
4. **关间二选一，不是 3 选 1**：v3.0 删除了 v2.4 的波间 3 选 1 机制。
5. **波次衔接无中断**：单关内波与波之间 2-3 秒过渡，没有玩家操作介入；玩家操作介入只发生在关间节点。
6. **PRNG 流必须严格隔离**：6 个流共用一个 master seed 但派生独立子 seed，混用会导致回放不可重现。

## 历史版本演化（v0.1 → v0.6）

10-roguelike-loop.md 的版本演化记录了关键设计决策：

- **v0.1 → v0.2**：删无尽模式 / 金币改本局 Run 货币 / 关内出卡资源改「能量 E」/ 波间 3 选 1 → 关间二选一 / 手牌默认 4 上限 8 / 技能卡双轨升级 / 跨波保留手牌。
- **v0.2 → v0.3**：大本营更名「水晶」（红色辉光晶体 + 无敌 + 秒杀 + 每杀 -1 HP + 碎裂动画）。
- **v0.3 → v0.4**：状态升级为「已批准，进入实施」；Phase A 改为直接搭 8 关骨架。
- **v0.4 → v0.5**：Phase A1+A2+A3 落地（6 流 PRNG / 4 个事件驱动管理器 / RunContext / WaveSystem 波次钩子）；累计 749/749 测试 0 回归。
- **v0.5 → v0.6**：塔卡升级移出商店/技能卡，转到关外科技树（`22-tower-tech-tree`）；新增「路径重置券」「碎片包」商店商品。

> ⚠️ 字段命名兼容：v3.0 语义叫「水晶」，但代码与存档保留 `baseHp` / `baseHpMax` / `crystalShatter` 等旧字段名，仅 UI 文案/设计文档/玩家可见处用「水晶」。
