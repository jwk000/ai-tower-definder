## 顶层心智模型

塔防游戏在 v3.1 的形态可以用四个核心范式概括，理解这四点就抓住了所有设计文档的骨架。

### 1. 一切皆单位（ECS + 配置驱动）

塔、士兵、敌人、中立机关、Boss 在底层都是 ECS 实体（bitecs `eid`），只是配置不同。

- 静态属性、行为规则、生命周期事件、AI 节点 **全部来自 YAML 配置**。
- 系统（如 `AttackSystem`、`MovementSystem`）是纯函数，对组件数据做统一处理。
- 想加新单位/塔/敌人 → 写一份 YAML 配置即可，不改系统代码。

权威出处：`60-tech/60-architecture.md`、`20-units/20-unit-system.md`、`20-units/21-unit-roster.md`。

### 2. 一切皆卡牌（v3.0 出卡部署）

玩家关内可部署的**所有内容**都是卡牌，卡牌只是「单位/法术配置 + 打出方式 + 升级状态」的封装。

- 卡牌四类：单位卡（部署 Tower/Soldier）/ 法术卡（立即生效）/ 陷阱卡（地块触发）/ 生产卡（生成可消耗资源）。
- 出卡消耗**关内能量 E**（每波回满，上限 10），不是金币。
- 手牌区默认 4 张，最大 8 张；跨波保留手牌。

权威出处：`10-gameplay/10-roguelike-loop.md` §2 卡牌系统。

### 3. Run 长征 + 三资源严格分层

一局游戏 = 一次 Run = 8 关连闯 + 终战 Boss。

- **水晶 HP**（前称大本营 baseHp）全程继承，红色悬浮辉光晶体；无敌但对入侵敌人秒杀，每秒杀 1 个 -1 HP；HP=0 即破碎，Run 失败 → 从第 1 关重开。
- **三资源不可互转**（仅 Run 结束未花完金币按 1:1 转碎片）：
  - **能量 E**（关内单波）：每波回满 +5，上限 10 → 出卡。
  - **金币 G**（整局 Run）：击杀 + 波次结算获得 → 关间商店 / 秘境。
  - **火花碎片**（跨局 meta）：Run 结束按到达关卡发放 → 主菜单永久解锁卡 / 塔科技树节点。
- 关与关之间插入「商店 vs 秘境」二选一节点（不再是波间 3 选 1）。

权威出处：`10-gameplay/10-roguelike-loop.md`、`10-gameplay/11-economy.md`、`50-data-numerical/50-mda.md`。

### 4. v3.1 塔升级 = 关外卡池科技树

v3.0 把卡升级保留在关间商店；v3.1 把**塔卡升级单独剥离**到关外卡池科技树（路径互斥 + 节点线性解锁），关内禁升级。

- 塔卡的 `instanceLevel` 仍可通过法术卡递增（仅影响本局战斗），但**形态切换**（如冰塔 → 火塔）只能在关外科技树解锁。
- 非塔卡（士兵 / 法术 / 陷阱 / 生产）仍走商店升级。
- `ice_tower` → `elemental_tower`（默认冰形态，路径覆盖冰/火/毒）；废弃毒藤塔 / 弩炮塔。

权威出处：`20-units/22-tower-tech-tree.md`（v3.1 权威），`60-tech/61-save-system.md`（`CardEntry.techTree` 字段）。

## 实现层关键约束

虽然这是设计文档综述，但有 5 条**实现侧的硬约束**贯穿全部文档，记忆系统在协助实现时必须记住：

1. **数值改动只改 `50-mda`**，其它文档与代码据此同步。残留数值视为 BUG。
2. **Pipeline 8 阶段顺序不可乱**：Managers → VFX → Modifiers → Gameplay → Lifecycle → Creation → AI → Render。详见 `60-tech/60-architecture.md`。
3. **6 个 PRNG 流隔离**：地图 / 波次组成 / 怪物属性扰动 / 关间事件 / 秘境事件 / 抽卡，各流独立 seed，便于可重现调试与回放。
4. **阵营语义重构已完成**：禁止 `isEnemy` 双轨布尔，统一用 `isHostileTo` API（`62-faction-refactor`）。
5. **存档版本 v2.0.0，无迁移代码**：游戏未发布，设计可随意调整存档 schema。
