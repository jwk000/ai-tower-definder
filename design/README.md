## ⚠️ 数值真理源规则（必读）

**[21-MDA数值设计](./21-mda-numerical-design.md) 是全部游戏数值的唯一权威来源。**

- 修改数值时**只改 21 文档**，然后据此同步代码（`src/data/gameData.ts`）与测试。
- 其它文档（如 03、05、06、07、11）**只描述字段语义/公式骨架/规则边界**，不再持有数值表。
- 若发现旧文档残留数值与 21 冲突，**视为 BUG，必须删除残留并以 21 为准**。
- 提 PR 时若改了数值，PR 标题应以 `[数值]` 前缀，便于审阅。

## 架构核心思想

1. **一切皆单位** — 塔、我方单位、敌人、中立机关本质相同，差别在于配置
2. **配置驱动** — 静态属性 + 动态行为规则全在配置中，新增单位只需写配置
3. **行为树驱动AI** — 所有单位的行为（攻击/移动/技能）统一由行为树配置驱动，专用系统仅执行原子动作（详见 [23-AI行为树方案](./23-ai-behavior-tree.md)）
4. **关卡可随机** — 天气、禁用塔、中立单位、装饰物从配置池随机初始化
5. **符号优于文字** — 场景中零文字标签，靠形状轮廓+符号特征+动态特效让玩家一眼识别单位
