# 08 — 游戏模式

> 关卡模式与无尽模式规则
>
> **数值真理源**：本文档不持有数值，所有数值（HP/速度系数/起始资源等）以 [21-MDA](./21-mda-numerical-design.md) 为准。

---

## 1. 关卡模式

### 1.1 规则

- 预设地图 + 固定波次配置（10-15 波）
- 每关有独特主题、敌人配置、可用塔类型
- 通关后解锁下一关
- 总计 5 个关卡，分布在 4-5 个主题章节

### 1.2 三星评定（P2-#13 修复 v1.1）

> **旧版问题**：旧的 ★/★★/★★★ 互斥（限时通关 vs 无损通关），玩家无法同时拿到两个星，造成奖励不可累计。
>
> **新规则**：三星**独立可累计**，每颗星独立评定，三项条件都达成即三星。

| 星级条件（每项独立评定，累计计星） | 评定 |
|--------------------------------------|------|
| **★1 通关星**：消灭所有波次敌人，基地存活（HP > 0） | 1 颗星 |
| **★2 防守星**：基地最终 HP ≥ 80%（满血上限的 80%） | +1 颗星 |
| **★3 速通星**：通关时间 ≤ 关卡目标时间（每关单独设定） | +1 颗星 |

**规则说明**：
- 每颗星独立判定，互不冲突（无损守家 + 速通可同时获得）
- 若通关失败（基地被摧毁）→ 0 星，无视其他条件
- 三颗星都达成 = ★★★，三星通关
- 关卡卡片显示当前最佳星数（取历次最高，不取每次平均）
- 每关的 `targetTime` 配置在 21-MDA §6.X 关卡参数中

**重通策略**：玩家可多次重玩同一关，每次取**单项最佳**（如第一次拿 ★1+★2 但超时，第二次速通成功 → 三星永久解锁）

### 1.3 难度曲线模型（P2-#12 修复 v1.1）

> **旧版问题**：08 只给单波系数（HP +15%/波等），但缺乏**关卡内**的难度曲线模型，导致玩家"早期太简单晚期太难"或反之。

#### 1.3.1 关卡内难度阶段

每关内部分为 4 个难度阶段（具体波次按关卡总波数比例划分）：

| 阶段 | 比例 | 设计目的 | 难度系数（敌人 HP/ATK 乘数） |
|------|------|----------|-------------------------------|
| **开局阶段** | 前 20% | 让玩家建立初始经济 | 0.8× |
| **建设阶段** | 20%-50% | 试探玩家防线 + 提供回本时间 | 1.0×（基准） |
| **压力阶段** | 50%-80% | 强敌登场，逼迫玩家升级/优化 | 1.3× |
| **高潮阶段** | 80%-100% | Boss/精英波/混合波 | 1.5× + 特殊机制 |

> 示例：10 波关卡 → 波 1-2 开局 / 波 3-5 建设 / 波 6-8 压力 / 波 9-10 高潮

#### 1.3.2 关卡间难度递增

跨关卡的难度提升通过 **DIFFICULTY_MULTIPLIER** 表达（值见 21-MDA §6）：

```
关卡 N 的敌人最终强度 = 关卡 N 的敌人基础数值 × LEVEL_DIFFICULTY[N] × WAVE_DIFFICULTY[wave]
```

- `LEVEL_DIFFICULTY[N]` 由 21-MDA 定义（如 L1=1.0, L2=1.15, L3=1.30, L4=1.50, L5=1.75）
- `WAVE_DIFFICULTY[wave]` 由 §1.3.1 阶段模型决定

#### 1.3.3 玩家成长与难度同步

为防止"塔升级速度 < 敌人变强速度"：

| 关键时刻 | 玩家应有能力 | 设计校验 |
|----------|--------------|----------|
| **建设阶段开始（20%）** | L1 主力塔 ×2-3 + 1 个生产建筑 | 经济回本时间 ≤ 建设阶段时长 |
| **压力阶段开始（50%）** | 至少 1 个 L3 塔 + 4-5 个塔 | 总收入应足够支付 1 个 L3 升级 |
| **高潮阶段开始（80%）** | 至少 1 个 L5 塔 或 2 个 L4 塔 | 总收入应足够支付 1 个 L5 或 2 个 L4 |

> 上述校验数值在 21-MDA §10 经济节奏验证中体现，每关数值调整时必须验证。

### 1.4 关卡选择

- 主菜单 → 关卡选择网格（3×2 排列 + 无尽模式卡片）
- 每个关卡卡片显示: 名称、主题、波数、星级、最佳成绩、目标时间（速通星条件）
- 未解锁关卡显示锁定状态和解锁条件
- 点击卡片进入关卡

## 2. 无尽模式（P2-#14 修复 v1.1）

### 2.1 规则

- 从**已解锁地图池**随机选 1 张（玩家可在 UI 选"上次地图"或"随机"）
- 无限波次递增，无波次上限
- 每 5 波出现随机 Boss（Boss 池从 21-MDA §3 BOSS 配置中抽）
- 超过 20 波后，每波均为 Boss 波 + 大量敌人

### 2.2 难度递增

> 每波难度乘数由 21-MDA `ENDLESS_WAVE_SCALING` 定义。本文档仅说明递增规则结构：

| 维度 | 规则 |
|------|------|
| 敌人 HP | 每波递增（指数衰减或线性，由 21-MDA 定义） |
| 敌人速度 | 每波递增至上限 200% |
| 金币掉落 | 每波递增（防止后期经济不足） |
| 护甲/魔抗 | 每波递增 |
| 生成间隔 | 每波缩短（不低于 0.25 秒） |

### 2.3 敌人类型解锁波次

| 波次 | 解锁敌人 |
|------|----------|
| 1+ | 小兵 |
| 2+ | 快兵 |
| 4+ | 自爆虫 |
| 6+ | 重装兵 |
| 8+ | 法师 |
| 10+ | 飞行敌（如热气球） |
| 12+ | 分散敌 |
| 15+ | 自定义敌人池（22 新单位） |

### 2.4 特殊规则触发

| 波次 | 规则 |
|------|------|
| 10 | Rush（生成间隔减半） |
| 15 | Fortified + NoBuild |
| 20 | DoubleSpeed + Fortified |
| 20+（每 3 波） | 从规则池随机抽 1 条（见下表） |

#### 2.4.1 后期规则池（v1.1 新增）

| 规则 | 效果 | 抽取权重 |
|------|------|----------|
| Rush | 生成间隔减半 | 30% |
| Fortified | 敌人 HP+25%，攻击力+10% | 25% |
| DoubleSpeed | 敌人速度+50% | 20% |
| NoBuild | 该波内不可建造新塔/单位 | 10% |
| BossRush | 该波 100% 由 Boss/精英构成 | 5% |
| EnergyDrain | 该波内能量回复速度 -50% | 5% |
| Camouflage | 普通敌人具备 50% 隐身（短距离才可见） | 5% |

> 抽取概率以 21-MDA 中规则池为准。同一波内同类规则不重复（如 Rush + Rush 不会同时出现）。

### 2.5 随机种子与可复现性（v1.1 新增）

- 进入无尽模式时，系统生成 `seed = timestamp + randomBytes`
- `seed` 保存在存档元数据，玩家可从设置中"复现上次种子"
- 所有随机选择（地图/Boss/敌人池/规则池）均使用同一 `seed` 派生的 PRNG
- 实现侧：使用 mulberry32 或 xoshiro128，禁用全局 `Math.random()`

### 2.6 评分

```
得分 = 存活波次 × 100 + 击杀数 × 10 + 击杀 Boss × 200 + 完整波数（无伤通过）× 50
```

最高分与对应种子存入存档。

## 3. 胜负条件

### 3.1 胜利
- 消灭关卡所有波次的敌人
- 基地（水晶）未被摧毁

### 3.2 失败
- 基地血量归零（敌人走到终点或远程摧毁）
- 特定关卡：限时内未消灭所有敌人

### 3.3 基地
- 初始血量与最终血量由 21-MDA §6 定义
- 位于路径终点（Base tile）
- 敌人到达终点 → 对基地造成伤害（伤害值由敌人类型决定，见 21-MDA §3）
