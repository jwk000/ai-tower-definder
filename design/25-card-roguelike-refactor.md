# 25 - 卡牌化 + Roguelike 长征改造方案

> 版本: v0.3 (水晶机制) | 日期: 2026-05-13 | 状态: **未实施，待评审**
> 不修改任何现有设计文档，本文档仅作为改造提案，待评审通过后再批量更新 01/02/03/06/07/08/13/14/21 等文档。

**变更记录**
- v0.2 → v0.3：**大本营更名为「水晶」**。水晶逻辑：无敌（免疫所有伤害）、对进入攻击范围内的敌人秒杀（无视护盾/无敌/Boss 除外），每次秒杀消耗 1 HP；HP=0 即破碎，Run 失败。视觉：红色水晶悬浮 + 辉光呼吸；死亡时碎裂成粒子飘散消失。详见 §6.2「水晶机制」、16-art §13.13、12-vfx §10。
- v0.1 → v0.2：根据评审意见调整 — 删除无尽模式 / 金币改为本局 Run 货币（关间商店与秘境消耗）/ 关内出卡资源改为「能量 E」/ 删除波间 3 选 1 改为关间二选一（商店或秘境）/ 手牌默认 4 上限 8 / 技能卡双轨（实例升级 + 卡级升级）/ 跨波保留手牌

## 0. 方案速览（TL;DR）

**改造一句话**：把游戏从「5 个独立关卡 × 自动经济 × 工具栏选塔」改造成「Roguelike 长征 × 卡组抽牌 × 关间商店秘境节点」的塔防 + 卡牌 + Rogue 融合体（参照《怪物火车》+《月圆之夜》）。

**三个核心改动**：
1. **关卡结构**：5 独立关卡 → 一次 Run 连闯 8 关 + 终战，**水晶**（前称大本营）HP 全程继承，死亡从第 1 关重开
2. **部署机制**：工具栏拖图标 → 开局从「永久卡池」抽 12 张构成本局卡组，每波开始抽卡到手牌，拖卡到地图部署消耗**能量**
3. **关间节点**：波次结束直接进入下一波（无中断），**关与关之间**插入「商店 / 秘境」二选一节点，用本局金币培养卡池 + 恢复血量

**三个保留**：
- 一切皆单位的 ECS 架构（02 文档）继续生效，卡牌只是「单位/法术的生成入口」
- 行为树驱动 AI（23 文档）继续生效，敌方「沿路径走 + 优先级停下攻击」机制是其特例
- 生产建筑（金矿/能量塔）作为「可选单位卡」继续存在，但不再是默认经济链

**三个删除**：
- **无尽模式**（08 文档）— 完全删除
- 关卡级三星评定（13 §3）→ 改为整局通关后的「英雄分」评分
- 关卡选择菜单（13 §4）→ 改为「开始新 Run / 继续上次 Run（关间存档点）」入口

**两条核心资源轴**：
- **金币 G**（本局 Run 范围）：本局击杀 + 波次结算获得，关间商店/秘境消耗。Run 结束未花完按比例转 meta 资源
- **能量 E**（关内战斗范围）：每波开始恢复，上限 10。手牌打到场上消耗能量

**关外资源**：
- **火花碎片**（meta 货币）：Run 结束（无论胜负）按到达关卡发放，主菜单卡池界面消耗用于永久解锁/升级卡牌

## 1. 设计目标与定位变更

### 1.1 玩法定位（更新 01-game-overview）

| 维度 | 旧定位 | 新定位 |
|------|--------|--------|
| 核心循环 | 部署 → 战斗 → 波次结算 → 下一波 | 抽卡组 → 战斗（多波） → 关间商店/秘境 → 下一关 → 终战 Boss |
| 单局时长 | 5-10 分钟/关，单关独立 | 25-40 分钟/Run，连闯 8 关 + 终战 |
| 失败代价 | 重打当前关，已通关进度保留 | 从第 1 关重开，已解锁的永久卡池保留 |
| 关内策略来源 | 选哪些塔、放哪里、何时升级 | 抽到什么卡、能量怎么分配、技能卡何时打 |
| 关间策略来源 | 无 | 商店买啥卡、秘境冒不冒险、金币该不该屯 |
| 长线驱动 | 通关 5 个关卡 + 三星 + 无尽 | 永久卡池解锁 + 流派构筑 + 历届最远闯关记录 |

### 1.2 三大支柱（新版）

1. **卡组随机性** — 开局抽到的 12 张卡决定本局流派（近战墙流/法术爆发流/生产抗压流），重玩性来自卡组组合爆炸
2. **关间节点抉择** — 每关结束的商店 vs 秘境二选一是本作 Rogue 决策核心，玩家在「稳健买卡」与「冒险换强力效果」之间博弈
3. **血量继承张力** — 水晶 HP 不重置，前期失血会在后期付出代价，每个决策都有长期权重

### 1.3 受影响的设计哲学

- ✅ **保留**：一切皆单位、配置驱动、行为树驱动 AI、符号优于文字
- ✅ **保留**：关卡可随机（天气、装饰物），范围扩大到「波次组成、关间事件池、秘境事件池」
- 🆕 **新增**：**卡组即配置** — 玩家卡组本质是一组「单位/法术配置 ID 引用 + 卡级升级状态」，仍然走配置驱动

## 2. 卡牌系统

### 2.1 卡牌定义

卡牌不是新增的实体类型，是一个**「单位/法术配置」+「打出方式」+「升级状态」的封装**。

```typescript
interface CardConfig {
  id: string;                    // 卡牌 ID（如 card_archer_tower）
  name: string;                  // 卡牌名（如 "箭塔"）
  type: 'unit' | 'spell';        // 卡牌大类
  energyCost: number;            // 关内能量消耗（1-8）
  rarity: 'common' | 'rare' | 'epic' | 'legendary';

  // type=unit 时，引用现有单位配置
  unitConfigId?: string;         // 如 "tower_archer"（沿用 03-unit-data）

  // type=spell 时，引用法术效果与子分类
  spellEffectId?: string;        // 如 "spell_arrow_rain"（沿用 04-skill-buff）
  spellSubtype?: 'damage' | 'control' | 'buff_instance' | 'buff_card' | 'utility';

  // 部署/释放约束
  placement: {
    targetType: 'tile' | 'enemy' | 'ally' | 'area' | 'global';
    range?: 'cursor' | 'path' | 'anywhere' | 'left-half' | 'right-half';
  };

  // 跨波保留标记（重要：默认 false 表示波末自动弃牌；true 表示可留到下波）
  persistAcrossWaves?: boolean;

  // 卡级升级（关间培养，本局生效）
  // 运行时数据：cardLevel: 1 | 2 | 3
}
```

### 2.2 卡牌三大分类

| 大类 | 子类 | 说明 | 数量目标 |
|------|------|------|----------|
| **建筑卡（Tower）** | 防御塔（单体/AoE/控制）、生产塔（金矿/能量塔）、辅助塔（治疗/增益） | 部署到地格，持续存在，可被摧毁 | 我方 8 张 |
| **单位卡（Soldier）** | 近战（盾兵/剑士）、远程（弓手/法师）、空中（飞兵）、特殊（祭司/工程师/召唤师） | 部署到地格，沿驻守圈巡逻并攻击敌人，可被击杀 | 我方 8 张 |
| **法术卡（Spell）** | 伤害、控制、Buff（实例升级 / 卡级升级两类）、Debuff、特殊 | 一次性效果，打出即消耗，进入弃牌堆 | 我方 4 张 |

**合计我方 20 张**。敌方 20 张见 §6。

### 2.3 抽卡组与手牌机制

```
开局（进入本局 Run）：
  1. 从「永久解锁卡池」按稀有度权重随机抽 12 张组成本局卡组
     - 权重：Common 60% / Rare 25% / Epic 12% / Legendary 3%
     - 保底：至少 1 张建筑卡 + 1 张单位卡 + 1 张法术卡
  2. 第 1 关第 1 波开始前，从卡组顶部抽 4 张到手牌

每波开始：
  1. 从卡组顶部抽牌，补满手牌至当前手牌上限（默认 4）
  2. 上一波未打出的卡：
     - persistAcrossWaves = true（特定技能卡如「时停」「全屏冰冻」）：保留在手牌
     - persistAcrossWaves = false（绝大多数卡）：进入弃牌堆，本波重新抽

战斗中：
  1. 玩家从手牌拖卡到地图部署，消耗能量 E
  2. 单位卡：在地图上生成实体，存活直到被击杀（沿用现有 02 单位系统）
  3. 法术卡：立即执行效果后进入弃牌堆
  4. 建筑卡：放置到地格，可被攻击摧毁

波次结束：
  1. 结算金币（击杀奖励 + 波次完成基础奖励）
  2. 短暂过渡（2-3 秒）后进入下一波（不弹任何选择面板）

关卡通过（最后一波包括关底 Boss 被击败）：
  → 进入「关间节点」（见 §3）

卡组耗尽时：洗弃牌堆为新卡组（标准 Rogue Deckbuilding）
```

**手牌规则**：
- 默认手牌上限 4 张
- 每张「手牌扩容」永久升级 +1，上限 8 张
- 抽到第 9 张时（已满）→ 该卡直接进入弃牌堆 + 屏幕飘字「手牌已满」

### 2.4 卡牌升级状态（关键变更）

**卡级升级在本局只能通过两种途径推进**：
- **关间商店**：花金币购买「升级 X 卡」服务（永久整局生效）
- **关内技能卡 - 卡级临时升级**：如「精炼术」点击手牌中的某张卡 → 本局内该卡永久变 L2

**关内技能卡 - 实例级临时升级**（独立于卡级）：
- 如「狂怀术」点击场上已部署的某个箭塔实例 → 该实例从 L1→L2，**仅作用于该实例直到死亡**
- 该实例死亡后重新部署的同名卡仍是卡本身的等级（L1 或本局已升级到的等级）

**关键规则**：
- 实例升级与卡级升级**叠加生效**：场上箭塔实际等级 = max(卡级, 实例修正后的等级)
- L3 是单局上限，超出无效

### 2.5 卡牌死亡 / 销毁

- 单位卡在战场上的单位被击杀 → **不回到弃牌堆**（决策：避免免费换血）
  - 例外：特定技能卡「英灵召还」可以让一只死亡单位返回手牌
- 建筑卡被摧毁 → 同上，不回弃牌堆
- 法术卡打出后 → 进入弃牌堆，下一轮洗牌可再抽

> 这与 v0.1 的「回弃牌堆」假设不同。新方案让死亡有重量，避免单位回收死循环。

## 3. 关间节点系统（核心 Rogue 抉择，替代波间 3 选 1）

### 3.1 节点结构

```
关卡 N 通过 → 弹出节点选择面板 → 玩家选择 1 个节点：

┌──────────────────────────────────────────────┐
│  关卡 3 通过！选择前往：                          │
├──────────────────┬───────────────────────────┤
│  🏪 商店         │  🌀 秘境                    │
│                  │                            │
│  确定收益：       │  随机事件：                  │
│  • 4 件商品可买   │  • 高风险高回报              │
│  • 可移除卡       │  • 可能恢复血量              │
│  • 进入下一关血量 │  • 可能损失资源              │
│    +5 HP         │                            │
└──────────────────┴───────────────────────────┘
```

### 3.2 商店节点（稳健分支）

```
商店面板（怪物火车风格）：

┌────────────────────────────────────────────────┐
│  商店 — 剩余金币 245 G                            │
├────────────────────────────────────────────────┤
│  [新卡] 火球术      80 G    [✓ 可买]            │
│  [升级] 箭塔→L2     60 G    [✓ 可买]            │
│  [移除] 移除1张卡   40 G    [✓ 可买]            │
│  [恢复] 水晶 +25 HP 100 G   [✓ 可买]            │
├────────────────────────────────────────────────┤
│  [刷新商品 30 G]   [离开商店 → 下一关]           │
└────────────────────────────────────────────────┘
```

| 商品类型 | 数量/商店 | 价格范围 |
|----------|----------|---------|
| 新卡（加入卡组） | 1-2 张 | 50-200 G（按稀有度） |
| 升级卡牌 | 1-2 个选项 | 50-120 G |
| 移除卡牌 | 1 个 | 40 G |
| 水晶恢复 | 1 个 | 100 G / 25 HP |
| 火花碎片兑换 | 偶现 | 200 G / 50 碎片 |

**商店刷新**：单次商店内 30 G 刷新所有商品，无次数限制（成本本身就是限制）。

### 3.3 秘境节点（冒险分支）

秘境是**怪物火车风格的事件房间**，进入时显示一段情境文字 + 2-3 个选择，每个选择有明确收益和潜在代价。

```
秘境面板示例：

┌────────────────────────────────────────────────┐
│  🌀 秘境 — 古老的祭坛                              │
│                                                │
│  你发现一座血色祭坛，散发着不祥的气息。            │
│  上面刻着三种契约：                                │
├────────────────────────────────────────────────┤
│  [A] 献祭 30% 水晶 HP                             │
│      → 获得 1 张 Legendary 卡（强制加入卡组）      │
│                                                │
│  [B] 献祭 1 张随机卡                              │
│      → 水晶恢复 40 HP + 获得 80 G                 │
│                                                │
│  [C] 拒绝契约离开                                  │
│      → 不消耗任何资源直接进入下一关                │
└────────────────────────────────────────────────┘
```

**秘境事件池**（初版 12 个，每次从未见过的事件中随机抽 1）：

| 事件 | 收益 | 代价 |
|------|------|------|
| 古老祭坛 | Legendary 卡 / 40 HP+80 G | 30% HP / 1 张卡 |
| 治愈泉 | 50 HP | 无（但本关下一波敌人 +20% HP） |
| 流浪商人 | 折扣 50% 的特殊商品 | 必须立即买，否则失 20 G |
| 试炼之门 | 完成挑战获 100 G + 1 张 Epic | 失败水晶 -20 HP |
| 神秘宝箱 | 随机 1: 100 G / 抽 2 张/ 升 1 卡 | 随机 1: -10 HP / 失 1 卡 |
| 老兵营地 | 立即让卡组所有「兵」卡 +1 级 | 80 G |
| 远古图书馆 | 抽 3 张卡选 1 加入卡组 | 50 G + 弃 1 张卡 |
| 战神祝福 | 下关全部单位攻击 +30% | -1 HP/秒 持续整关 |
| 时之沙漏 | 跳过下一关（直接通过） | 失去 30% HP + 失去本关所有金币 |
| 黑市 | 用 HP 换金币（1:2 比例，最多 30 HP） | 自定 |
| 幻境 | 抽 4 张本不在卡池中的特殊卡选 1 | 50 G |
| 水晶加固 | 水晶最大 HP +20（永久本局） | 80 G |

**秘境特性**：
- 至少有一个「无代价」选项（如「拒绝契约离开」），玩家可零成本退出
- 事件结果立即生效，无倒计时
- 已遇过的事件本局不重复

### 3.4 商店 vs 秘境的设计意图

| 维度 | 商店 | 秘境 |
|------|------|------|
| 不确定性 | 低（明码标价） | 高（事件随机） |
| 收益期望 | 稳定中等 | 方差大（最佳 > 商店，最差 < 商店） |
| 触发玩家心态 | 资源管理、规划 | 冒险、博弈 |
| 适合场景 | 资源充足、卡组成型 | 资源紧张、需要爆发 |

每个关间节点**只能选一个**，做完即进入下一关，无法回头。

### 3.5 关卡终战与终战后

第 8 关 + 终战 Boss 关之间**没有关间节点**，必须直接面对终战。这是设计上故意的张力放大。

终战通过 → Run 胜利结算：
- 显示总金币、英雄分、击杀数等
- 未花完金币按 1:1 转换为火花碎片
- 火花碎片入账 meta 资源池

## 4. 能量系统（关内出卡资源）

### 4.1 能量定义

**Energy (E)** 是关内唯一的出卡消耗资源，与金币完全分离。

```
能量规则：
  - 上限：10
  - 初始（每关第 1 波）：5
  - 每波开始恢复：+5（不超过上限）
  - 战斗中恢复：无（怪物火车风格，靠回合制恢复）

  - 永久升级可扩展上限：+1/+2，最高 12
  - 部分特殊卡/秘境效果可额外恢复能量
```

### 4.2 能量消耗示例

| 卡 | 能量消耗 |
|----|----------|
| 基础箭塔（L1） | 3 E |
| 高级炮塔（L1） | 6 E |
| 火球术（伤害法术） | 2 E |
| 箭雨（AoE 法术） | 4 E |
| 治疗水晶 | 5 E |
| 终极召唤：龙 | 10 E |

数值表将在 21-MDA 文档新增章节《卡牌能量与稀有度数值表》统一管理。

### 4.3 能量与金币的分工

| 资源 | 范围 | 来源 | 消耗 |
|------|------|------|------|
| **能量 E** | 关内单波 | 每波开始恢复 | 出卡 |
| **金币 G** | 整局 Run | 击杀 + 波次结算 | 关间商店/秘境 |
| **火花碎片** | 跨局 meta | Run 结束按表发放 | 主菜单卡池/永久升级 |

> 三者完全不互通（除 Run 末未花完金币按比例转碎片外）。这是设计上的简洁性原则。

## 5. 经济系统改造（替换 06-economy-system）

### 5.1 资源精简

| 资源 | 旧设计 | 新设计 |
|------|--------|--------|
| 金币 G | 击杀+生产+波次 | **击杀 + 波次结算**（删除被动生产，作为本局 Run 货币） |
| 能量 E | 能量塔产出，升级/技能消耗 | **改为出卡消耗资源**（关内每波回复制，不与塔相关） |
| 人口 P | 限制单位数量 | **改造为「场上单位上限」**，由永久升级解锁，默认 8 |

### 5.2 金币来源（新版）

```
本局 Run 金币产出：
  - 击杀普通敌人：3-8 G
  - 击杀精英敌人：15-25 G
  - 击杀 Boss：80-150 G
  - 波次完成结算：基础 20 G + 5 G × 关卡序号
  - 关卡通过：100 G + 50 G × 关卡序号
  - 秘境事件：视事件随机

初始金币：0 G（本局从零开始累积）
```

**重要变化**：关内战斗中金币**不能花**（无商店、无升级入口）。金币纯粹是「累积到关间消费」的资源。这给玩家明确的「攒钱节奏感」。

### 5.3 生产建筑保留（已确认）

- **特别说明**：能量塔（重命名「能量水晶」）作为「关内能量加速器」继续可选，效果是「每波开始 +3 E 上限上限上限」

## 6. 关卡结构改造（替换 07/08）

### 6.1 一次 Run 的完整结构

```
[主菜单 → 开始新 Run]
  → [开局抽 12 张卡组]
  → [关卡 1: 平原]（8-10 波）
      → 水晶 HP 从初始值开始
  → [关间节点：商店 vs 秘境 二选一]
  → [关卡 2: 森林]
  → [关间节点]
  → ...
  → [关卡 8: 王城]
  → [终战 Boss 关]（单波超长，混合所有敌人类型，无关间节点接续）
      → 击败 → Run 胜利
      → 失败或中途任何一关水晶 HP=0（水晶破碎） → Run 失败
      → 任一情况：发放火花碎片 + 返回主菜单
```

**总波次**：约 70-90 波 / Run（每关 8-10 波，外加终战），预计时长 30-45 分钟。

### 6.2 水晶机制（核心防御实体）

> 水晶（Crystal）取代旧版「大本营」。它是玩家方的核心防御实体，位于地图左侧路径终点。

#### 6.2.1 定位与外观

- **类型**：特殊单位（unit_kind=`crystal`），固定占用 1 个 tile，不可被部署/移除/移动。
- **视觉**：红色水晶造型，悬浮于空中（上下浮动循环），辉光呈呼吸式亮度变化。详见 [16-art-assets §13.13](./16-art-assets-design.md#1313-水晶大本营视觉) 与 [12-visual-effects §10](./12-visual-effects.md#10-水晶视觉与特效v30-新增)。

#### 6.2.2 无敌规则

水晶**免疫一切伤害源**——任何敌人的普攻、技能、AoE、毒、点燃、自爆、Boss 必杀、反弹伤害等**都无法直接扣除水晶 HP**。

- 唯一的 HP 消耗途径是**主动秒杀**（见 §6.2.3）。
- **正向操作仍然生效**：
  - 「治疗水晶」法术卡、商店「水晶 +25 HP」、秘境治疗事件、「水晶加固」加最大 HP 等均按既定规则结算。
  - 秘境「献祭 X% 水晶 HP」等主动负向操作仍然生效（属于玩家自愿付出的代价，不属于"伤害"）。
- 取消旧规则：**敌人抵达终点不再扣 HP**，这条规则被「秒杀消耗 1 HP」彻底替代。

#### 6.2.3 秒杀攻击（核心战斗机制）

水晶把任何**进入其攻击范围**的敌人立即秒杀（无视目标的 HP、护盾、无敌、霸体、闪避等一切防御机制），每次秒杀**消耗水晶 1 HP**。

| 字段 | 语义 | 默认值 | 真理源 |
|------|------|--------|--------|
| `crystal.attackRange` | 攻击范围（地格数），范围圆形 | 1 格（紧贴自卫） | [21-MDA §18](./21-mda-numerical-design.md) |
| `crystal.attackCooldown` | 攻击间隔（秒），防止单帧多杀掏空 HP | 0.5 s | [21-MDA §18](./21-mda-numerical-design.md) |
| `crystal.hpCostPerKill` | 每次秒杀消耗的水晶 HP | 1 | [21-MDA §18](./21-mda-numerical-design.md) |
| `crystal.maxKillsPerTick` | 单次冷却结束最多击杀几个敌人 | 1 | [21-MDA §18](./21-mda-numerical-design.md) |

**目标选择**：当范围内有多个敌人时，按「最近距离 > 进入范围时间」排序击杀 1 个（冷却结束后再杀下一个）。

**Boss 例外**：终战 Boss（`abyss_lord`）携带 `immune_to_crystal_kill = true` 标记，水晶**无法秒杀 Boss**。Boss 进入水晶范围时不会触发秒杀也不消耗 HP，玩家必须用常规手段处理。Boss 仍然不能给水晶造成伤害（无敌规则继续生效）。

> 该标记可应用到其他单位上作为后续平衡手段。

#### 6.2.4 水晶 HP 与失败条件

```
初始 HP：100（基础值由永久升级 baseHpBonus 决定）

水晶 HP 变化：
  - 秒杀 1 个敌人：-1 HP
  - 商店/秘境/法术卡治疗：+N HP（不超过当前最大 HP）
  - 秘境献祭/试炼失败：按事件配置 -N HP
  - 「水晶加固」事件：当前最大 HP +N（同时按比例补充当前 HP）
  - 关卡通过：不自动恢复

水晶 HP = 0 → 水晶碎裂动画 → Run 失败
```

关间节点期间，允许「保存当前 Run 状态退出」（下次主菜单可继续）。

## 12. 待用户决策的开放问题

下列问题在本提案中已给出默认方案，但实施前建议再确认：

1. **关卡数量**：默认 8 关 + 终战。是否需要调整为 6 关或 10 关？
2. **能量上限**：默认 10。是否要按关卡解锁递增（如第 1 关 8 → 第 8 关 12）？
3. **跨波保留的技能卡占比**：建议 4 张法术卡中 1-2 张支持 `persistAcrossWaves=true`。是否合适？
4. **关间节点二选一互斥**：玩家选了商店就不能再去秘境，永远？还是「先商店再可选秘境，但要付代价」？（建议互斥，保持决策重量）
5. **本局未花完金币转碎片比例**：默认 1:1。是否调整？
6. **手牌满时抽到的卡**：默认进入弃牌堆。是否改为「玩家选弃哪张」？（默认直接弃，简化交互）
7. **终战 Boss 失败惩罚**：是否给「再战一次」机会？还是直接 Run 失败？（建议直接失败，提升终战重量）
