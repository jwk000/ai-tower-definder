# 塔科技树系统

> 版本: v1.0 | 日期: 2026-05-14 | 状态: 设计阶段
>
> 取代旧版「线性等级升级」模型，将塔升级移至关外卡池界面，以「路径互斥科技树」呈现。本文档为塔升级机制的唯一权威来源（Single Source of Truth）。

> 数值占位：所有具体数值（碎片成本、概率、CD、伤害倍率等）以 [21-MDA](../50-data-numerical/50-mda.md) 为权威，本文档只描述字段语义、机制骨架、路径结构。

## 1. 设计目标与核心原则

### 1.1 设计目标

1. **关外 meta 进度** —— 升级是 Roguelike 长线积累的一部分，而不是单局内的临时强化。
2. **策略分叉** —— 每个塔都有清晰的"风格选择"，让玩家面对同样的塔位时，做出不同的 build 决策。
3. **配置驱动** —— 科技树结构、节点能力、解锁成本全部 YAML 化，策划可改不动代码。
4. **互斥路径鼓励复玩** —— 同一存档可点满两条路径但单局只装备一个，鼓励同塔的不同流派切换。

### 1.2 与旧模型的差异

| 维度 | 旧线性升级（v2 残留） | 新科技树（本方案） |
|---|---|---|
| 触发时机 | 关内右键塔，吃本局金币 | 关外卡池界面，吃火花碎片 |
| 形态变化 | 数值递增，外观不变（仅光效） | 节点切换形态名、能力、外观 |
| 持久化 | 仅本局有效 | meta 永久解锁，跨 Run 保留 |
| 分支 | 无，固定 L1→L5 | 1–3 条互斥路径，每路径 3–4 个节点 |
| 关内升级按钮 | 存在 | **彻底移除** |
| 关内临时强化 | 与升级混淆 | 保留为独立 `instanceLevel`（详见 §2.3） |

## 2. 全局规则

### 2.1 关内禁止升级

- **关内不再有"升级塔"按钮**，BuildSystem 不处理任何升级请求。
- 塔在关内一旦放置，形态 = 该塔卡当前装备路径下已解锁的最深节点形态。
- 关内移除塔仅返还能量（按 [25 §6 退卡规则](../10-gameplay/10-roguelike-loop.md) 处理），不返还碎片。

### 2.2 关外科技树解锁

- 每塔基础卡内嵌一棵科技树。
- **路径互斥**：单个塔卡同时只能"装备"一条路径作为本局形态。
- **节点线性解锁**：必须按 节点 1 → 节点 2 → 节点 3（→ 节点 4）顺序购买。
- **跨路径独立**：路径 A 与路径 B 的节点解锁互不影响，**两条路径都能点满**。
- **解锁货币**：火花碎片（meta 货币，详见 [13 §CardCollection](../60-tech/61-save-system.md) / [21 §17](../50-data-numerical/50-mda.md)）。
- **路径切换**：默认免费切换装备路径（路径 A ↔ 路径 B）。
- **路径重置**：若想"忘记"某路径已解锁的节点（极少需要），消耗碎片重置该路径，返还 50% 碎片。重置成本与返还比例进 [21-MDA](../50-data-numerical/50-mda.md)。

### 2.3 关内临时升级（`instanceLevel`）保留

- 单个塔实例可在关内通过特殊机制（特定卡牌效果 / 商店 buff / 秘境奖励等）获得 `instanceLevel +1/+2/...`。
- **仅作用于单个塔实例**，不写入 `CardCollection`，不影响 meta 科技树进度。
- 塔死亡 / 关卡结束 / 退卡时 `instanceLevel` 归零。
- **`instanceLevel` 只调数值，不切换形态**（不能从"双重箭塔"升到"三重箭塔"）。具体数值公式见 [21-MDA](../50-data-numerical/50-mda.md)。

### 2.4 关间节点（商店/秘境）的边界

- 关间节点 **不再提供塔升级**（旧 v3 草案中的"购买永久升级"全部下线）。
- 关间节点可以提供：
  - 临时 buff（影响 `instanceLevel` 或单局加成）
  - 碎片包（间接为科技树买单）
  - **路径重置券**（消耗后免费重置一次某塔的某路径）

> ⚠️ **路径重置只在卡池界面执行**，关间节点至多提供"重置券"作为道具掉落，不直接触发重置 UI（避免和"关外升级"原则模糊）。

## 3. 与 v3.0 卡牌系统的关系

科技树是对 [25-card-roguelike-refactor](../10-gameplay/10-roguelike-loop.md) v3.0 卡牌系统的**深化**，不冲突：

| v3.0 已有 | 科技树新增 |
|---|---|
| 每个塔 = 一张单位卡 | 每张塔卡内嵌一棵科技树 |
| 旧线性升级模型 `baseLevel` L1→L5 | 已删除；改为"已解锁节点 + 装备路径"组合 |
| `instanceLevel`（关内临时升级） | **保留**，仅由法术卡提升（详见 [04 §7](./23-skill-buff.md)），与节点切换解耦 |
| `CardCollection`（永久卡池） | 加 `unlockedNodes` / `equippedPath` 字段 |
| 关间商店/秘境 | 不再提供升级，可提供碎片包/重置券 |

### 3.1 `baseLevel` 字段移除

- 旧字段 `baseLevel: 1–5` 在 v3.1 起从所有结构中**移除**。
- 游戏未上线，无玩家数据需要迁移；存档结构直接以"含 `techTree` / 无 `baseLevel`"为准（详见 [13 §1.1](../60-tech/61-save-system.md) 和 §6 当前版本与策略）。

## 4. 七塔完整科技树

> **图例**
> - 「→」表示节点线性解锁
> - 节点编号 1 = 基础形态（默认拥有），后续节点需碎片解锁
> - 能力描述只标识"新增/变化"，未提及的能力沿用上一节点

### 4.1 箭塔

**塔定位**：物理单体远程，基础经济友好型。

| 路径 | 节点 1 | 节点 2 | 节点 3 |
|---|---|---|---|
| **1 · 多重射击** | 普通箭塔（单发） | 双重箭塔（同时射出 2 支箭，可锁不同目标） | 三重箭塔（同时射出 3 支箭） |
| **2 · 高频火力** | 普通箭塔 | 连弩箭塔（攻速大幅↑、单发伤害↓，单体高频） | 连弩火箭塔（在连弩基础上附加小范围灼烧 DOT） |

### 4.2 炮塔

**塔定位**：物理 AOE 范围伤害，主清杂兵。

| 路径 | 节点 1 | 节点 2 | 节点 3 |
|---|---|---|---|
| **1 · 控场 AOE** | 普通炮塔（AOE 范围伤害） | 重炮塔（AOE + 概率眩晕命中目标） | 超级炮塔（AOE + 眩晕 + 击退） |
| **2 · 狙击穿透** | 普通炮塔 | 狙击炮塔（攻击间隔↑，射程↑，单发伤害↑，转为低频高伤单体） | 战术炮塔（狙击模式 + **弹道贯穿，单发命中沿途 ≤2 个敌人**） |

> 路径 2 终点的"贯穿"机制承接旧版 **弩炮塔** 的核心能力（弩炮塔本身已废弃，见 §8）。

### 4.3 元素塔（原冰塔）

**塔定位**：控制 / 元素效果，是新方案中**唯一拥有 3 条路径**的塔。

- **改名**：旧"冰塔" → **元素塔**（Elemental Tower）。
- 默认形态 = **元素塔 · 冰**，沿用旧冰塔的减速效果，保证兼容。

| 路径 | 节点 1 | 节点 2 | 节点 3 |
|---|---|---|---|
| **1 · 冰系** | 元素塔 · 冰（命中减速） | 冰冻塔（命中按概率触发"冰冻"硬控） | 霜冻塔（冰冻 + 命中目标攻击力 Debuff） |
| **2 · 火系** | 元素塔 · 冰 | 火焰塔（命中附加灼烧 DOT，覆盖减速效果） | 真火塔（灼烧 + **击杀按概率额外 +1 能量**，复用 [`EnergySystem`](../src/unit-system/EnergySystem.ts)） |
| **3 · 毒系** | 元素塔 · 冰 | 巫毒塔（命中附加中毒 DOT，覆盖减速效果） | 病毒塔（中毒 **传染**：跳数 ≤3、半径 ≤80px、每跳伤害衰减 50%） |

#### 元素塔机制说明

- 切换路径 = 切换默认元素属性（冰/火/毒），UI 与外观随之变化。
- 路径之间的元素效果**不叠加**：在火系路径下，命中只附加灼烧，不附加减速。
- 真火塔的"额外能量"按击杀点结算，每次结算独立 roll，不与卡牌本身的能量回收冲突。

### 4.4 电塔（单路径 4 节点）

**塔定位**：链式弹跳，群体压制。

| 节点 1 | 节点 2 | 节点 3 | 节点 4 |
|---|---|---|---|
| 普通电塔（1 次弹跳） | 热电塔（2 次弹跳） | 核电塔（3 次弹跳） | 闪电塔（3 次弹跳 + **概率触发"全屏闪电"**：随机攻击全图敌人一次，CD ≥10s，单次伤害 ≈ 1.5× 普攻） |

#### 全屏闪电规则

- 触发逻辑：每次普通攻击后掷骰，CD 内不可重复触发。
- 目标：随机敌人池（不限阵营/状态/位置），单次伤害独立计算。
- 定位：末期 panic button，**不是清屏 AI 必胜键**。
- 数值（概率、CD、倍率）以 [21-MDA](../50-data-numerical/50-mda.md) 为准。

### 4.5 激光塔

**塔定位**：聚焦 / 持续输出。

| 路径 | 节点 1 | 节点 2 | 节点 3 |
|---|---|---|---|
| **1 · 扇形覆盖** | 普通激光塔（1 道激光） | 中级激光塔（2 道激光，同时锁不同目标） | 高级激光塔（3 道激光 + 攻击 CD↓） |
| **2 · 蓄能聚焦** | 普通激光塔 | 稳压激光塔（在攻击范围内**持续锁单体**，不切换目标） | 充能激光塔（蓄能机制：**持续锁同一目标越久伤害越高**，目标切换 / 死亡后清零） |

> 充能激光塔（原方案命名"特变激光塔"）已改名为更直白的"充能激光塔"。

### 4.6 蝙蝠塔（单路径 3 节点）

**塔定位**：群体单位类塔，受天气影响。

| 节点 1 | 节点 2 | 节点 3 |
|---|---|---|
| 普通蝙蝠塔（3 蝙蝠） | 中级蝙蝠塔（4 蝙蝠 + ATK↑） | 高级蝙蝠塔（5 蝙蝠 + ATK↑） |

#### 蝙蝠塔机制说明

- 所有节点沿用 [21-unit-roster §2.2 蝙蝠塔天气依赖](./21-unit-roster.md#蝙蝠塔bat_tower天气依赖) 的 `weather_dependent_atk` 天气影响。
- 单路径设计因素：蝙蝠塔的"变量"来自天气系统，已经具备策略深度，无需再加路径。
- 节点 2/3 的"+蝙蝠数"会**直接修改单位生成数量**（详见 [02-unit-system §蝙蝠生成](./20-unit-system.md)）。

### 4.7 导弹塔

**塔定位**：战略全图打击，地格评分系统。详见 [19-missile-tower](./26-missile-special.md)。

| 路径 | 节点 1 | 节点 2 | 节点 3 |
|---|---|---|---|
| **1 · 双联齐射** | 普通导弹塔（单发） | 双联导弹塔（**同一地格同时发射 2 个导弹**，单发伤害↓，总伤害↑） | 集束导弹塔（同一地格同时发射 3 个导弹，单发伤害再↓，总伤害再↑） |
| **2 · 战略弹头** | 普通导弹塔 | 温压弹塔（爆炸范围内附加灼烧 DOT） | 核弹塔（爆炸范围扩大 + 灼烧） |

#### 导弹路径 1 机制说明

- 评分逻辑（选哪个地格）不变，复用 [19-missile-tower §地格评分](./26-missile-special.md)。
- **同坐标多发射**：`ProjectileSystem` 在选定地格后，按节点数量循环 N 次发射相同弹道，弹道之间错开极短延迟（视觉区分用，伤害互独立）。
- 总伤害 ≈ 单发 × N，但单发 < 普通导弹塔单发伤害（数值进 [21-MDA](../50-data-numerical/50-mda.md)）。
- 实现优势：复用现有评分流水线，比"打多个地格"简单；视觉效果更冲击。

## 5. 配置结构（YAML）

### 5.1 塔单位配置（`config/units/towers.yaml`）

塔单位 YAML 增加 `techTree` 字段，旧 `upgrades` 字段废弃。

```yaml
arrow_tower:
  id: arrow_tower
  name: 箭塔
  category: tower
  baseStats:
    hp: 100
    atk: 20
    range: 200
    attackInterval: 1.0
  techTree:
    paths:
      - id: multi_shot
        name: 多重射击
        nodes:
          - id: arrow_basic
            name: 普通箭塔
            shardCost: 0          # 默认拥有
            effects: []
          - id: arrow_double
            name: 双重箭塔
            shardCost: 300        # 数值占位，权威值在 21-MDA
            effects:
              - rule: add_projectile_count
                value: 1
          - id: arrow_triple
            name: 三重箭塔
            shardCost: 600
            effects:
              - rule: add_projectile_count
                value: 2
      - id: rapid_fire
        name: 高频火力
        nodes:
          - id: arrow_crossbow
            name: 连弩箭塔
            shardCost: 300
            effects:
              - rule: mul_attack_interval
                value: 0.4
              - rule: mul_atk
                value: 0.5
          - id: arrow_crossbow_fire
            name: 连弩火箭塔
            shardCost: 600
            effects:
              - rule: add_burning_on_hit
                duration: 2.0
                tickRatio: 0.2
```

### 5.2 节点 `effects` 字段语义

- 数组形式，按声明顺序应用（执行 = 规则引擎 dispatch）。
- 每个 effect 引用一个 `RuleHandler`（已注册或新增）。
- `effects` **只描述差量**（相对节点 N-1 新增/覆盖），便于策划读懂"这个节点比前一个强在哪"。
- 累积叠加由配置加载器在装备路径时**全节点合并**为单位实际属性。

### 5.3 元素塔的"切元素"

元素塔特殊处理：路径切换时根据装备的路径 ID 把单位的 `elementType` 字段从 `ice` 改为 `fire` / `poison`。规则引擎据此决定命中附加哪种 DOT/Debuff。

## 6. 存档结构

详细字段定义见 [13-save-system](../60-tech/61-save-system.md) §科技树扩展。

```typescript
interface CardCollection {
  unlockedCards: Record<CardId, CardEntry>;
  // ... v3.0 既有字段
}

interface CardEntry {
  cardId: string;
  unlocked: boolean;
  techTree?: TechTreeProgress;  // 仅塔卡持有
}

interface TechTreeProgress {
  // 每条路径已解锁到第几个节点（1 = 仅基础节点，2 = 解锁了第 2 个节点，...）
  pathDepth: Record<PathId, number>;
  // 当前装备的路径（影响关内出塔形态），未装备时使用默认路径
  equippedPath: PathId | null;
}
```

- `baseLevel` 字段已彻底**移除**，不再出现在任何存档结构中（详见 §3.1）。

## 10. 开放问题

> 后续讨论 / 实现阶段需回填的问题，**暂不阻塞设计落地**。

1. **重置粒度**：路径重置是"整条路径回到节点 1"还是"逐节点退还"？当前方案 = 整条路径，待玩家测试后回看。
2. **路径切换是否有冷却？** 默认无冷却（自由切换装备路径），如果发现平衡性问题（玩家每关切一次）再加约束。
3. **真火塔 vs 能量回收类卡牌叠加上限？** 是否需要"每关每塔最多 +5 能量"类硬上限？暂留作 [21-MDA](../50-data-numerical/50-mda.md) 数值平衡阶段决策。
4. **全屏闪电的视觉表现**：是单条贯穿全图，还是多条独立闪电？[12-vfx](../40-presentation/44-visual-effects.md) 实现阶段决定。
5. **科技树版本化**：未来若调整某节点能力，是否要给已解锁玩家做"补偿"？v1.0 阶段不考虑，留作 [13-save](../60-tech/61-save-system.md) 长期议题。

## 附录 A：术语表

| 术语 | 定义 |
|---|---|
| 路径 (Path) | 一棵科技树下的一条线性升级分支，路径之间互斥装备 |
| 节点 (Node) | 路径上的一个形态阶梯，节点之间线性解锁 |
| 装备 (Equip) | 选定某条路径作为本局塔形态的来源 |
| 解锁 (Unlock) | 永久购买某节点，写入 `CardCollection.techTree` |
| 重置 (Reset) | 主动放弃某路径已解锁的节点，部分退还碎片 |
| `instanceLevel` | 关内单个塔实例的临时强化层级，不持久化、不切形态 |
| 碎片 (Spark Shard) | meta 永久货币，详见 [06](../10-gameplay/11-economy.md) / [21](../50-data-numerical/50-mda.md) |
