# 导弹塔设计

> 新单位：导弹塔 | 战略武器定位 | MDA分析驱动
>
> **v1.2（2026-05-16）核心变更 — 目标选择改为「手动指挥 + 托管」双模式**：
> - **默认手动指挥模式**：装弹（CD）完成后塔进入红色呼吸灯提示状态，**无限等待**玩家指令；玩家点击塔后拖动一个红色目标定位指示器，指示器受 600px 射程圈约束（圈外不可放）；松手即在该位置进入 0.6s 蓄力 → 发射导弹；拖动中途可通过「拖出圈外松手 / 右键 / Esc」三种方式取消（CD 不消耗，仍处呼吸灯态）。
> - **托管模式（可选）**：玩家可在塔右键菜单中切换为托管态，沿用 v1.1 的**地格评分系统**（§三）自动选定目标并发射。托管态与手动态可随时互切。
> - 原 §三「地格评分系统」整章保留，仅在托管模式下生效；手动模式下整章不参与运行时计算。
> - 原 §四「红色靶向标记」「蓄力 0.6s」保留为发射前阶段（玩家松手后触发），不再由系统自动选择目标。
>
> **v3.1（2026-05-14）变更**：原"L1–L5 线性升级"模型整体废弃，导弹塔升级改为**科技树**：
> - 路径 1 · 双联齐射：普通 → 双联（2 个导弹打同地格）→ 集束（3 个导弹打同地格）
> - 路径 2 · 战略弹头：普通 → 温压弹（+灼烧）→ 核弹（爆炸范围扩大 + 灼烧）
>
> 权威定义见 [22a-skill-tree-tower §10](./22a-skill-tree-tower.md#10-导弹塔--missile_tower)。本文档 §二「升级机制」保留作历史参考；§4.3.1 新增双联/集束的同坐标多发射机制说明。

## 〇、MDA 分析

### M — 机制（Mechanics）

| 机制 | 说明 |
|------|------|
| 地格评分系统 | 导弹不锁定单个敌人，而是对所有可达地格打分，攻击收益最大的位置 |
| 超长装弹 | 装弹时间远超现有塔（6-8秒），每次发射都是高价值决策 |
| 大范围爆炸 | 命中后对爆炸半径内所有敌人造成高额伤害 |
| **大射程（非全图）** | 射程 600px（≈ 画面 1/3 宽），可覆盖中央 + 一条出生路径，但**不再覆盖整张地图** |
| 弹道飞行 | 导弹有可见飞行轨迹，从塔飞向目标地格，有飞行时间 |
| **抗性敌制衡** | L4+ 关卡引入飞行敌（地面爆炸不命中）和分散敌（移动中持续分散，降低密度评分） |

### D — 动态（Dynamics）

| 动态 | 玩家行为 | 体验目标 |
|------|----------|----------|
| 时机抉择 | 装弹期间如果敌人分散，这一炮价值打折；必须等敌人密集时"开火" | 高压力决策——等还是不等？ |
| 防线崩溃应对 | 当大量敌人逼近基地时，导弹塔是最后防线 | 救援感——绝境中的逆转 |
| BOSS终结技 | BOSS波对单体高价值目标的精准打击 | 力量感——大数字清屏 |
| 资源博弈 | 高造价决定它不是"随便放"的塔，必须精心选择建造时机和位置 | 资源管理的战略深度 |
| 装弹期脆弱 | 装弹期间导弹塔完全无输出，必须由其他塔掩护 | 防御布局的互补性 |

### A — 美学（Aesthetics）

| 美学类型 | 目标体验 | 实现策略 |
|----------|----------|----------|
| **力量感** | 毁灭性的爆炸带来爽快感 | 大半径爆炸粒子特效 + 屏幕震动 + 重低音效 |
| **策略满足感** | 精准命中敌人密集区，一炮清屏 | 伤害数字跳出 + 连锁死亡动画 |
| **紧张感** | 装弹期间看着敌人逼近的心跳加速 | 装弹进度环视觉 + 装弹完成时的音效提示 |
| **救援感** | 濒临失败时导弹拯救基地 | 导弹飞行时的紧迫音效 + 命中后的满足反馈 |
| **视觉辨识度** | 导弹塔一眼可辨认，与现有塔截然不同 | 复合几何体设计：大型基座 + 发射井 + 导弹 + 雷达天线 |

| 属性 | 值 |
|------|-----|
| ID | `missile_tower` |
| 名称 | 导弹塔 |
| 分类 | Tower |
| 阵营 | Player |
| 层级 | Ground |
| 定位 | **战略武器**——攻击范围大（600px，覆盖约画面 1/3 宽），伤害极高，装弹极长，**默认由玩家手动指定攻击地格**（CD 完成后无限等待玩家指令），可切换为托管模式由系统按地格评分自动选择 |

### 升级机制（结构）⚠️ **v3.1 已废弃**

> 🛑 本节已废弃。导弹塔升级改为科技树（路径 1 双联齐射 / 路径 2 战略弹头），详见 [30 §4.7](./22-tower-tech-tree.md#47-导弹塔)。
> 以下表格保留作历史参考，运行时实际能力以科技树节点配置为准。

| 等级 | 升级效果（结构性） |
|------|---------|
| L1 | 基础 — 单发导弹 |
| L2 | 伤害与爆炸半径小幅提升 |
| L3 | **解锁被动：二段爆炸**——爆炸后 0.5s 产生半径 60% 的二次冲击波，造成 50% 伤害 |
| L4 | 伤害与爆炸半径继续提升 |
| L5 | **解锁被动：热压弹头**——爆炸**中心 10% 半径区域**内的敌人受到伤害 ×1.2（核心定位加成，鼓励精准打击；中心外仍为 100% 伤害，不再有"全场清场"效果） |

> 关键削弱说明（v1.1）：
> 1. **射程**：从 9999（全图）→ **600px**。导弹塔不再是"无视距离的全图武器"，必须根据敌人主路径选择建造位置——这恢复了"塔位选择"这一塔防游戏的核心策略乐趣。
> 2. **L5 热压弹头**：从"中心 30% 半径×1.3 + 边缘 0.7"的连续衰减 → **仅中心 10% 半径×1.2，其余 100%**。不再是"清屏神器"，转向"精准制裁"——奖励玩家瞄准敌群核心。
> 3. **L3 二段爆炸**保留，因为它强调"延迟收益"而非"无限范围"，不与削弱方向冲突。

## 三、目标选择系统（v1.2 重构 — 手动 + 托管双模式）

> **v1.2 核心变更**：导弹塔不再默认自动选目标。装弹（CD）完成后，塔进入「待命/呼吸灯」态，**由玩家手动指挥**目标地格；玩家也可显式切换到「托管」模式，让原 §3.3 地格评分系统重新自动接管。本节先讲两种模式的状态机与交互流程，再讲托管模式沿用的评分公式（§3.3 ~ §3.7）。

### 3.0 模式总览

| 模式 | 触发 | 目标选择方式 | 默认 |
|------|------|-------------|------|
| `manual`（手动指挥） | 默认 / 玩家在塔右键菜单选「手动」 | CD 完成后塔呼吸灯提示 → 玩家点击塔 → 拖动定位指示器 → 松手 → 蓄力 0.6s → 发射 | ✅ 默认 |
| `auto`（托管） | 玩家在塔右键菜单选「托管」 | CD 完成后立即按 §3.3 地格评分公式自动选目标 → 蓄力 0.6s → 发射；射程内无合法地格则空转等待 | ❌ 可选 |

**模式持久化**：每个导弹塔实例独立保存 `targetingMode` 字段（默认 `manual`）。模式切换立即生效；若正处于「呼吸灯待命」或「蓄力中」时切换，按下表处理：

| 切换时机 | 切换为 manual | 切换为 auto |
|---------|---------------|--------------|
| CD 中（未完成） | 待 CD 完成后进入呼吸灯态 | 待 CD 完成后立即评分发射 |
| 呼吸灯待命中 | 维持呼吸灯，继续等玩家 | 立即跑评分，找到目标即蓄力发射；无目标继续空转 |
| 玩家拖动指示器中 | 维持拖动（不打断） | 立即取消拖动，跑评分自动发射 |
| 蓄力中 | 维持蓄力到底（不打断） | 维持蓄力到底（不打断） |

### 3.1 手动指挥模式（Manual）— 主流程

**状态机**：

```
[Idle/Reloading] ──CD 完成──▶ [Ready/呼吸灯]
                                    │
                            玩家点击塔 │  ESC/右键 = 不动作
                                    ▼
                          [Aiming/拖动指示器]
                                    │
              ┌─────拖出 600px 圈外松手──────┐
              │                              │
              │  右键 / Esc                  │
              │       ↓                      ▼
              └─▶ [Cancel] ──回 Ready──▶ [Charging 0.6s]
                                              │
                                              ▼
                                      [Launching/发射]
                                              │
                                              ▼
                                  [Reloading/CD 重置]
```

#### 3.1.1 呼吸灯态（Ready）

| 属性 | 值 |
|------|-----|
| 触发 | 装弹（CD）完成的那一帧 |
| 持续 | **无限**——必须由玩家点击触发，或由托管模式切换接管，否则永远等待 |
| 视觉 | 塔身整体红色辉光呼吸（脉冲 1.2s/周期，alpha 0.2 ↔ 0.6），辉光半径在塔基础 glow 半径上 +8px |
| 音效 | CD 完成瞬间播放 `SFX_MISSILE_READY`（短促"咔嗒"+ 雷达扫描音，详见 [46-audio §5.7](../40-presentation/46-audio.md#57-导弹塔)）；呼吸灯持续期间不重复播放 |
| 鼠标 hover | 鼠标悬浮塔上时光标变为「准星 cursor」，提示"可点击" |
| 与其他系统的关系 | 呼吸灯期间，塔的 `Attack.cooldownTimer` 已归零、不再 tick；下一次 CD 仅在「发射完成那一刻」重置开始计时 |

#### 3.1.2 玩家点击 → 拖动定位指示器（Aiming）

| 属性 | 值 |
|------|-----|
| 触发 | 玩家在呼吸灯态下用左键**按下**（press-down）塔本体 |
| 进入动画 | 塔身辉光从「呼吸脉冲」切换为「常亮高 alpha=0.7」；同时画面叠加显示该塔的 **600px 射程圈**（红色虚线圆环，alpha 0.35） |
| 指示器跟随 | 鼠标按住不放期间，**红色目标定位指示器**跟随光标移动（按下瞬间指示器出现在光标位置） |
| 射程约束 | 指示器中心点必须落在 600px 射程圈内：<br>• 光标在圈内 → 指示器跟随光标，**绿色边缘**（合法目标）<br>• 光标在圈外 → 指示器**吸附到圈内最近点**，并显示**红色边缘 + ✖ 标识**（非法目标，松手 = 取消） |
| 指示器外观 | 红色十字准星（与原 §4.2 红色靶向标记同款，十字线长 16px）+ 外圈旋转环（半径 = 当前等级爆炸半径 × 0.5）+ 中心红色实心圆（半径 4px） |
| 爆炸预览圈 | 指示器中心同时叠加显示一个**虚线圆**（半径 = 当前等级爆炸半径），让玩家直观看到 AOE 覆盖范围 |
| 命中预览 | 落在爆炸预览圈内的所有**地面层级**敌人头顶浮现红色小箭头（飞行敌不浮现，呼应「不命中飞行」机制） |
| 鼠标释放（mouse-up） | 在圈内 → 进入 §3.1.3 蓄力发射；在圈外（红色 ✖ 态） → 进入 §3.1.4 取消 |

#### 3.1.3 松手 → 蓄力 → 发射

| 属性 | 值 |
|------|-----|
| 蓄力时长 | 0.6s（沿用 §4.1） |
| 蓄力期间 | 塔身动画沿用 §4.1（发射井下沉、辉光脉冲、雷达加速旋转）；目标地格的红色靶向标记**固定**在玩家松手时指示器中心位置 |
| 蓄力期间可否取消 | ❌ 不可取消（一旦松手即承诺发射，避免反复横跳） |
| 蓄力完成 | 进入 §4.3 发射 → §4.4 飞行 → §4.5 爆炸（所有原流程不变） |
| 发射完成 | 立即重置 `Attack.cooldownTimer = 1 / attackSpeed`，重新进入 [Reloading] 态 |

#### 3.1.4 取消方式

| 触发 | 行为 | 视觉/音效 |
|------|------|----------|
| 拖出 600px 圈外松手 | 指示器消失 + 射程圈消失 + 塔回呼吸灯态 | 播放 `SFX_MISSILE_CANCEL`（轻"嘟"声）；CD 不消耗 |
| 拖动中按右键 | 同上 | 同上 |
| 拖动中按 Esc | 同上 | 同上 |
| 拖动中游戏被暂停 | 暂停后保持「指示器跟随最后光标位置」状态；恢复后玩家可继续拖动或松手 | — |

**关键规则**：取消不消耗 CD，塔继续待命；这保证了「手动指挥」是无成本的尝试，鼓励玩家试验最佳时机。

#### 3.1.5 异常处理

| 场景 | 处理 |
|------|------|
| 拖动中塔被摧毁 | 立即取消拖动，指示器/射程圈消失 |
| 拖动中切换到托管模式 | 立即取消拖动，转入托管自动选目标（见 §3.0 表） |
| 拖动中游戏切场景（如打开菜单） | 等同 Esc，自动取消 |
| 同时多个导弹塔都处于呼吸灯态 | 玩家依次点击各塔分别指挥；同一时刻只有一个塔处于「Aiming」态（点击新塔时，前一塔的 Aiming 自动 Esc 取消） |

### 3.2 托管模式（Auto）— 流程

托管模式下，塔在 CD 完成后**立即**调用 §3.3 ~ §3.7 的地格评分算法选定目标，无需玩家交互。其余行为（蓄力 0.6s → 发射 → 爆炸）与手动模式完全相同。

| 属性 | 值 |
|------|-----|
| 触发 | 塔的 `targetingMode === 'auto'` 且 `Attack.cooldownTimer <= 0` |
| 视觉差异 | 托管态下塔身**不进入呼吸灯**，而是塔顶常显一个小型「⚙ 齿轮图标」标识托管中（alpha 0.6） |
| 无目标时 | 评分函数返回 null → 塔空转等待（不消耗 CD），下一帧重试 |
| 切换 UI | 玩家右键塔 → 弹出小菜单「指挥模式 ▸ ○ 手动 / ● 托管」（默认 ○ 手动） |

### 3.3 评分概述（托管模式专用）

> **以下 §3.3 ~ §3.7 仅在托管模式（`targetingMode === 'auto'`）下运行；手动模式下整段逻辑不参与计算。**

导弹塔攻击时，不选择单个敌人，而是评估**塔射程内**所有**可达地格**的"攻击收益"，选择得分最高的地格发射导弹。

评估范围（v1.1 修正）：
1. **塔到地格中心的直线距离 ≤ `range`（600px）** —— 超出射程的地格不参与评分
2. 该地格至少有一个**地面层级**敌人站在上面，或该地格在敌人所在格的相邻范围
3. **飞行层级敌人不计入**评分（地面爆炸不命中）

> 若射程内没有任何合法地格 → 不发射，等待。这与"全图射程"时代的行为差异巨大——玩家必须根据敌人路径选择塔位。

### 3.4 评分因子

地格 `(row, col)` 的攻击收益评分由三个因子加权计算：

#### 因子一：距离因子（Base Proximity）

> **敌方单位距离基地越近，分数越高。**

```
距离因子 = (基地距离_最大值 - 敌人到基地的路径距离) / 基地距离_最大值 × 距离权重
```

- 基地位置：地图上的 `base` tile
- 路径距离：敌人到基地的**沿路径步数**（不是直线距离），沿 `enemyPath` 行进方向计算当前位置距基地的剩余步数
- 每个地格取该格上**所有敌人中距离基地最近的那个**作为该格的距离值
- `基地距离_最大值` = 地图路径总长度（步数）
- `距离权重` = 0.35（可调）

> **设计意图**：优先打击最紧迫的威胁——即将到达基地的敌人。这符合塔防游戏"防漏"的核心体验。

#### 因子二：密度因子（Enemy Density）

> **以该地格为中心，爆炸范围内敌方单位越多，分数越高。**

```
密度因子 = min(爆炸范围内敌方单位数 / 密度上限, 1.0) × 密度权重
```

- 爆炸范围 = 导弹塔当前等级的 `爆炸半径`（L1=120px）
- `密度上限` = 8（超过8个敌人不再额外加分，防止极端情况失真）
- `密度权重` = 0.45（可调）

> **设计意图**：鼓励玩家在敌人密集时开火，最大化AOE收益。这是导弹塔"等待最佳时机"的核心动力。

#### 因子三：等级因子（Enemy Tier）

> **攻击目标单位的等级越高，评分越高。**

```
等级因子 = sum(每个在爆炸范围内敌人的 tier_multiplier) / (敌人数量 × 最大tier倍率) × 等级权重
```

| 敌人层级 | tier_multiplier |
|---------|-----------------|
| L1 普通（小兵、快兵） | 1.0 |
| L2 精英（重装、法师、自爆虫） | 2.0 |
| L3 BOSS（指挥官、攻城兽） | 4.0 |

- `等级权重` = 0.20（可调）

> **设计意图**：导弹塔对高价值目标有天然偏好。打中BOSS比打中小兵更有价值，引导导弹优先处理最危险的敌人。

### 3.5 综合评分公式

```
total_score = 距离因子 + 密度因子 + 等级因子 + 同格bonus
```

**同格bonus**: 如果敌人恰好站在被评估的地格上（而非仅在爆炸范围内），额外加0.05分。这确保导弹优先打在敌人"身上"而非空旷的地格。

所有因子归一化到 [0, 1] 范围，权重总和 = 1.0。

### 3.6 计算流程（仅托管模式）

```
每帧（当装弹完成时，且 targetingMode === 'auto'）：
  1. 收集所有存活的【地面层级】敌方单位及其位置（飞行单位排除）
  2. 遍历每个有敌人的地格，过滤掉【塔到该格中心距离 > range】的格子（射程外）
  3. 对每个射程内的地格计算综合评分：
     a. 计算距离因子：取该格上距离基地最近的敌人
     b. 计算密度因子：以该格为中心，统计爆炸半径内所有【地面】敌人
     c. 计算等级因子：对爆炸半径内所有【地面】敌人按tier加权
     d. 加同格bonus
  4. 选取评分最高的地格
  5. 向该地格发射导弹
```

### 3.7 边界规则（手动 & 托管通用）

| 场景 | 手动模式处理 | 托管模式处理 |
|------|-------------|--------------|
| 没有敌人在射程内 | 玩家仍可点击塔进入 Aiming 态拖动指示器，但落点无敌时爆炸为空（仍消耗 CD）；UI 显示提示「无目标」（不阻止发射，由玩家自行决定） | 不发射，等待（装弹完成后空转） |
| 射程内只有飞行敌人 | 同上（玩家可看到飞行敌头顶 ⊘ 提示，知道地面爆炸打不到，但仍可强行发射打地） | 不发射，等待 |
| 多个地格评分相同 | — | 优先选距离基地更近的 |
| 爆炸超出地图边界 | 正常计算，爆炸范围裁剪到地图内 | 同左 |
| 唯一敌人 | 玩家手动选位置 | 直接锁定该敌人所在格（评分 = 距离因子 + 等级因子） |
| BOSS免疫眩晕 | 导弹的爆炸伤害对BOSS完全有效，不受免疫影响 | 同左 |
| **分散敌**（移动中持续散开） | 玩家可亲眼判断聚拢瞬间手动点火，**手感优势明显** | 正常参与评分，但其"动态分散"特性会自然降低密度因子（详见 [07-地图与关卡系统 §4.5](../10-gameplay/13-map-level.md#45-l4-飞行敌和-l6-分散敌)） |

#### 4.3.1 路径 1 · 同坐标多发射（v3.1 新增）

科技树路径 1（双联齐射 / 集束）的实现规则：

| 节点 | 同时发射导弹数 | 单发伤害倍率 | 总伤害倍率 |
|------|---------------|-------------|-----------|
| 普通导弹塔（基础节点） | 1 | 1.00 | 1.00 |
| 双联导弹塔（节点 2） | **2** | ~0.65（数值在 21-MDA） | ~1.30 |
| 集束导弹塔（节点 3） | **3** | ~0.55（数值在 21-MDA） | ~1.65 |

**机制要点**：

- **目标地格不变**：评分逻辑（§3 地格评分系统）只跑一次，选出**单个**最优地格，然后向该地格依次发射 N 个导弹。
- **抛物线弹道独立**：N 个导弹各自有独立抛物线，**水平速度相同，但初始垂直速度小幅扰动**（±15px/s）形成"V 字"或"扇形"出弹视觉。
- **发射时序**：N 个导弹之间错开 80–120ms 极短延迟（视觉区分用），不形成战术差异。
- **爆炸独立结算**：每个导弹独立调用 §4.5 爆炸效果与 §4.8 伤害计算，命中范围内的敌人会被重复打击，每发独立结算伤害（即"叠加打"，不是"取最大"）。
- **视觉冲击设计意图**：路径 1 的核心卖点是"密集爆破"——同点多次爆炸 + 屏幕震动叠加，制造"重锤连击"的力量感。
- **与路径 2 互斥**：装备路径 1 的导弹塔不会附加灼烧/扩大爆炸范围；这些是路径 2 的节点能力。

**实现复用**：

- `ProjectileSystem` 在选定地格后，按 `equippedPath.nodeIndex` 决定发射循环次数 N。
- 每次循环调用现有 `spawnMissileProjectile(target, atk*nodeDamageMult)`，无须新增弹道类型。
- 视觉错开延迟在发射循环内累加 `delayMs *= i`。

|------|-----|------|
| 弹道形状 | **抛物线弧线**——导弹先向上飞出，再沿重力弧线落下 | 区别于所有其他塔的直线弹道 |
| 水平速度 | 280px/s | 恒定水平分量 |

| 尾焰方向 | 始终沿速度反方向 | 火焰向后喷射 |

```
// 弹道计算公式
每帧更新:
  vx = 水平速度 (恒定)
  vy = vy + 重力 * dt  (上一帧垂直速度 + 重力加速度)
  x  = x + vx * dt
  y  = y + vy * dt
  rotation = atan2(vy, vx)  // 导弹朝向速度方向
```

### 4.8 伤害计算

```
基础伤害 = 导弹塔ATK

物理减伤 = 基础伤害 × (1 - 敌人护甲 / (护甲 + 100))

实际伤害 = 减伤后伤害 × 中心加成系数
```

**中心加成**（v1.1 修正 — 仅 L5 热压弹头被动生效）：

| 距爆炸中心距离 | 伤害比例 |
|--------------|---------|
| **0 ~ 10% 半径**（中心核心区） | **120%**（热压弹头加成） |
| 10% ~ 100% 半径 | **100%**（不再衰减） |

> 关键变更（v1.1）：
> - 旧版：30% 半径 ×1.3、60% 半径 ×1.0、100% 半径 ×0.7 的三段衰减——"清屏神器"
> - 新版：仅最中心 10% 半径 ×1.2，其余 100%——"精准制裁，奖励瞄准"
> - 设计意图：让玩家专注于"瞄准敌群核心"的微观决策，而不是"开炮即清场"的大手大脚

**无 L5 被动时**（L1~L4）：全爆炸范围 100% 伤害。

### 7.2 攻击系统集成

2. **导弹飞行**：复用 `ProjectileSystem`，但导弹的目标是 `(x, y)` 坐标而非 `entityId`

### 7.6 导弹定位地格的实现方式（关键设计决策）

导弹塔的与众不同在于**攻击目标不是实体，而是地格坐标**。现有 `ProjectileSystem` 的弹道需要一个 `targetId`（实体ID）来飞行。解决方案：

**创建临时"打击标记"实体**：
1. 导弹塔通过评分系统选出最佳地格 `(row, col)`
2. 将该地格中心转换为像素坐标：`(cx, cy) = (offsetX + col * tileSize + tileSize/2, offsetY + row * tileSize + tileSize/2)`
3. 创建一个不可见的"marker"实体（含 `Position` 组件，不含 `Visual` 组件）
4. 发射的 `Projectile.targetId` 指向该 marker 实体
5. 弹道到达 → `ProjectileSystem.onHit()` 触发 → `applySplash()` 在 `(hitX, hitY)` 处造成AOE伤害
6. marker 实体在命中后销毁

```typescript
// 伪代码（AttackSystem中的导弹塔分支）
const targetRow = bestTile.row;
const targetCol = bestTile.col;
const targetX = scene.offsetX + targetCol * tileSize + tileSize / 2;
const targetY = scene.offsetY + targetRow * tileSize + tileSize / 2;

// 创建临时标记实体
const markId = world.createEntity();
world.addComponent(markId, Position, { x: targetX, y: targetY });

// 发射导弹，目标指向标记实体
spawnProjectile(world, towerId, markId, towerTypeVal);
// 标记实体在 ProjectileSystem.onHit 命中后由 world.destroyEntity 清理
```

**优点**：完全复用现有 `ProjectileSystem` 的飞行、命中、AOE、爆炸特效逻辑，零侵入性修改。

## 八、平衡性考量

### 8.1 与其他塔的对比

> **具体数值以 [21-MDA §4.7 / §4.8](../50-data-numerical/50-mda.md#47-导弹塔战略核武器) 为准**。本节仅描述结构性差异：

| 塔 | 攻速 | 范围 | AOE | 特色 |
|----|------|------|-----|------|
| 箭塔L1 | 高 | 中（180） | 无 | DPS基准 |
| 炮塔L1 | 中 | 中（180） | 中（80px） | 溅射AOE+眩晕 |
| 导弹塔L1 | **极低** | **大（600px，~画面 1/3 宽）** | **大（130~190px）** | **战略 AOE + 不能命中飞行** |

**结构性差异**：
- 导弹塔不再是"全图武器"——必须根据敌人主路径选择塔位
- 导弹塔不能命中飞行敌——L4+ 需要搭配高射程/反空塔
- L5 热压弹头从"清屏"变为"精准制裁"——中心 10% 加成 ×1.2

### 8.2 风险与限制（v1.1 更新）

| 风险 | 缓解措施 |
|------|---------|
| 单体伤害过高，替代所有其他塔 | 极长装弹 + 高造价 + 高人口 + **限制射程 600px**，确保不能"导弹塔海" |
| ~~全图射程过于OP~~ | **已削弱**：射程 600px，塔位必须精心选择，强化空间策略 |
| 评分系统可能不直观 | 可选的发射预览：鼠标悬停时显示第一名地格的评分和预估伤害 + **射程圈视觉**显示 600px 范围 |
| BOSS波过于克制 | BOSS单体评分高是设计意图，但导弹长装弹期让玩家不能完全依赖 |
| **飞行敌制衡** | L4+ 飞行敌不被导弹爆炸命中，玩家必须搭配反空塔 |
| **分散敌制衡** | L6+ 分散敌移动中持续散开，降低密度因子，让导弹"等密集"变得困难 |

### 8.3 建议的关卡开放（v1.1 更新）

| 关卡 | 是否可用 | 制衡机制 |
|------|---------|---------|
| L1 平原 | ❌ | — |
| L2 沙漠 | ❌ | — |
| L3 冰原 | ❌ | — |
| L4 火山 | ✅ 解锁 | **同时引入飞行敌**——导弹无法包打 |
| L5 城堡 | ✅ | 飞行敌持续出现 |
| L6+（如有） | ✅ | **引入分散敌**——降低密度评分 |
| 无尽模式 | ✅ | 混合各种制衡敌人 |

> 理由：导弹塔属于高级战略武器，应在玩家已经熟悉基础塔防后再解锁。L4 火山的窄路径让导弹 AOE 发挥，但**同关卡引入的飞行敌**形成立即制衡，防止"导弹塔流"统治后期。
