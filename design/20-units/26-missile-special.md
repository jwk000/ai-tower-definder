> 新单位：导弹塔 | 战略武器定位 | MDA分析驱动
>
> **v3.1（2026-05-14）变更**：原"L1–L5 线性升级"模型整体废弃，导弹塔升级改为**科技树**：

## 〇、MDA 分析

### M — 机制（Mechanics）

| 机制 | 说明 |
|------|------|
| 地格评分系统 | 导弹不锁定单个敌人，而是对所有可达地格打分，攻击收益最大的位置 |
| 超长装弹 | 装弹时间远超现有塔（6-8秒），每次发射都是高价值决策 |
| 大范围爆炸 | 命中后对爆炸半径内所有敌人造成高额伤害 |
| **大射程（非全图）** | 射程 600px（≈ 画面 1/3 宽），可覆盖中央 + 一条出生路径，但**不再覆盖整张地图** |
| 弹道飞行 | 导弹有可见飞行轨迹，从塔飞向目标地格，有飞行时间 |
| **抗性敌制衡** | L4+ 关卡引入飞行敌（地面爆炸不命中）和分散敌（移动中持续分散，降低密度评分） |

### D — 动态（Dynamics）

| 动态 | 玩家行为 | 体验目标 |
|------|----------|----------|
| 时机抉择 | 装弹期间如果敌人分散，这一炮价值打折；必须等敌人密集时"开火" | 高压力决策——等还是不等？ |
| 防线崩溃应对 | 当大量敌人逼近基地时，导弹塔是最后防线 | 救援感——绝境中的逆转 |
| BOSS终结技 | BOSS波对单体高价值目标的精准打击 | 力量感——大数字清屏 |
| 资源博弈 | 高造价决定它不是"随便放"的塔，必须精心选择建造时机和位置 | 资源管理的战略深度 |
| 装弹期脆弱 | 装弹期间导弹塔完全无输出，必须由其他塔掩护 | 防御布局的互补性 |

| 美学类型 | 目标体验 | 实现策略 |
|----------|----------|----------|
| **力量感** | 毁灭性的爆炸带来爽快感 | 大半径爆炸粒子特效 + 屏幕震动 + 重低音效 |

## 三、地格评分系统（核心机制）

### 3.1 评分概述

导弹塔攻击时，不选择单个敌人，而是评估**塔射程内**所有**可达地格**的"攻击收益"，选择得分最高的地格发射导弹。

评估范围（v1.1 修正）：
1. **塔到地格中心的直线距离 ≤ `range`（600px）** —— 超出射程的地格不参与评分
2. 该地格至少有一个**地面层级**敌人站在上面，或该地格在敌人所在格的相邻范围
3. **飞行层级敌人不计入**评分（地面爆炸不命中）

> 若射程内没有任何合法地格 → 不发射，等待。这与"全图射程"时代的行为差异巨大——玩家必须根据敌人路径选择塔位。

### 3.2 评分因子

地格 `(row, col)` 的攻击收益评分由三个因子加权计算：

#### 因子一：距离因子（Base Proximity）

> **敌方单位距离基地越近，分数越高。**

```
距离因子 = (基地距离_最大值 - 敌人到基地的路径距离) / 基地距离_最大值 × 距离权重
```

- 基地位置：地图上的 `base` tile
- 路径距离：敌人到基地的**沿路径步数**（不是直线距离），沿 `enemyPath` 行进方向计算当前位置距基地的剩余步数
- 每个地格取该格上**所有敌人中距离基地最近的那个**作为该格的距离值
- `基地距离_最大值` = 地图路径总长度（步数）
- `距离权重` = 0.35（可调）

> **设计意图**：优先打击最紧迫的威胁——即将到达基地的敌人。这符合塔防游戏"防漏"的核心体验。

#### 因子二：密度因子（Enemy Density）

> **以该地格为中心，爆炸范围内敌方单位越多，分数越高。**

```
密度因子 = min(爆炸范围内敌方单位数 / 密度上限, 1.0) × 密度权重
```

- 爆炸范围 = 导弹塔当前等级的 `爆炸半径`（L1=120px）
- `密度上限` = 8（超过8个敌人不再额外加分，防止极端情况失真）
- `密度权重` = 0.45（可调）

> **设计意图**：鼓励玩家在敌人密集时开火，最大化AOE收益。这是导弹塔"等待最佳时机"的核心动力。

#### 因子三：等级因子（Enemy Tier）

> **攻击目标单位的等级越高，评分越高。**

```
等级因子 = sum(每个在爆炸范围内敌人的 tier_multiplier) / (敌人数量 × 最大tier倍率) × 等级权重
```

| 敌人层级 | tier_multiplier |
|---------|-----------------|
| L1 普通（小兵、快兵） | 1.0 |
| L2 精英（重装、法师、自爆虫） | 2.0 |
| L3 BOSS（指挥官、攻城兽） | 4.0 |

- `等级权重` = 0.20（可调）

> **设计意图**：导弹塔对高价值目标有天然偏好。打中BOSS比打中小兵更有价值，引导导弹优先处理最危险的敌人。

### 3.3 综合评分公式

```
total_score = 距离因子 + 密度因子 + 等级因子 + 同格bonus
```

**同格bonus**: 如果敌人恰好站在被评估的地格上（而非仅在爆炸范围内），额外加0.05分。这确保导弹优先打在敌人"身上"而非空旷的地格。

所有因子归一化到 [0, 1] 范围，权重总和 = 1.0。

### 3.4 计算流程

```
每帧（当装弹完成时）：
  1. 收集所有存活的【地面层级】敌方单位及其位置（飞行单位排除）
  2. 遍历每个有敌人的地格，过滤掉【塔到该格中心距离 > range】的格子（射程外）
  3. 对每个射程内的地格计算综合评分：
     a. 计算距离因子：取该格上距离基地最近的敌人
     b. 计算密度因子：以该格为中心，统计爆炸半径内所有【地面】敌人
     c. 计算等级因子：对爆炸半径内所有【地面】敌人按tier加权
     d. 加同格bonus
  4. 选取评分最高的地格
  5. 向该地格发射导弹
```

### 3.5 边界规则

| 场景 | 处理方式 |
|------|---------|
| 没有敌人在射程内 | 不发射，等待（装弹完成后空转） |
| 射程内只有飞行敌人 | 不发射，等待（飞行敌不计入评分） |
| 多个地格评分相同 | 优先选距离基地更近的 |
| 爆炸超出地图边界 | 正常计算，爆炸范围裁剪到地图内 |
| 唯一敌人 | 直接锁定该敌人所在格（评分 = 距离因子 + 等级因子） |
| BOSS免疫眩晕 | 导弹的爆炸伤害对BOSS完全有效，不受免疫影响 |
| **分散敌**（移动中持续散开） | 正常参与评分，但其"动态分散"特性会自然降低密度因子（详见 [07-地图与关卡系统 §4.5](../10-gameplay/13-map-level.md#45-l4-飞行敌和-l6-分散敌)） |

## 四、攻击序列（完整流程）

导弹塔的攻击分为三个连续阶段：

```
[装弹完成] → [蓄力阶段 0.6s] → [导弹发射+飞行] → [爆炸+冲击波+震动+破损]
```

### 4.1 蓄力阶段（装弹完成后 → 导弹离开发射井前）

| 属性 | 值 | 说明 |
|------|-----|------|
| 蓄力时长 | 0.6秒 | 固定时长，不受等级影响 |
| 塔身动画 | 发射井微微下沉 + 塔身红色辉光脉冲 | 下沉幅度3px，辉光从alpha 0.1→0.4脉冲 |
| 待发导弹 | 导弹从井口逐渐上升2px | 模拟"上膛"动作 |
| 雷达天线 | 加速旋转至2.0rad/s | 蓄力期间旋转加速 |
| 音效 | 低沉的积蓄能量嗡鸣声（SFX_MISSILE_CHARGE） | 频率从低到高攀升 |

### 4.2 红色靶向标记（蓄力期间出现）

| 属性 | 值 | 说明 |
|------|-----|------|
| 出现时机 | 蓄力开始后0.15秒 | 先蓄力，再显示目标 |

| 旋转环 | `#d50000`，alpha 0.6 | 顺时针旋转，周期1.2s，2px线宽 |

### 4.3 导弹发射

| 属性 | 值 | 说明 |
|------|-----|------|

#### 4.3.1 路径 1 · 同坐标多发射（v3.1 新增）

科技树路径 1（双联齐射 / 集束）的实现规则：

| 节点 | 同时发射导弹数 | 单发伤害倍率 | 总伤害倍率 |
|------|---------------|-------------|-----------|
| 普通导弹塔（基础节点） | 1 | 1.00 | 1.00 |
| 双联导弹塔（节点 2） | **2** | ~0.65（数值在 21-MDA） | ~1.30 |
| 集束导弹塔（节点 3） | **3** | ~0.55（数值在 21-MDA） | ~1.65 |

**机制要点**：

- **目标地格不变**：评分逻辑（§3 地格评分系统）只跑一次，选出**单个**最优地格，然后向该地格依次发射 N 个导弹。
- **抛物线弹道独立**：N 个导弹各自有独立抛物线，**水平速度相同，但初始垂直速度小幅扰动**（±15px/s）形成"V 字"或"扇形"出弹视觉。
- **发射时序**：N 个导弹之间错开 80–120ms 极短延迟（视觉区分用），不形成战术差异。
- **爆炸独立结算**：每个导弹独立调用 §4.5 爆炸效果与 §4.8 伤害计算，命中范围内的敌人会被重复打击，每发独立结算伤害（即"叠加打"，不是"取最大"）。
- **视觉冲击设计意图**：路径 1 的核心卖点是"密集爆破"——同点多次爆炸 + 屏幕震动叠加，制造"重锤连击"的力量感。
- **与路径 2 互斥**：装备路径 1 的导弹塔不会附加灼烧/扩大爆炸范围；这些是路径 2 的节点能力。

**实现复用**：

- `ProjectileSystem` 在选定地格后，按 `equippedPath.nodeIndex` 决定发射循环次数 N。
- 每次循环调用现有 `spawnMissileProjectile(target, atk*nodeDamageMult)`，无须新增弹道类型。
- 视觉错开延迟在发射循环内累加 `delayMs *= i`。

### 4.4 抛物线弹道

| 属性 | 值 | 说明 |
|------|-----|------|
| 弹道形状 | **抛物线弧线**——导弹先向上飞出，再沿重力弧线落下 | 区别于所有其他塔的直线弹道 |
| 水平速度 | 280px/s | 恒定水平分量 |
| 初始垂直速度 | -150px/s（向上） | 导弹先向上"弹射" |
| 重力加速度 | 400px/s² | 模拟重力，产生自然抛物线 |
| 最大高度 | 约60-80px（塔顶上方） | 视觉上清晰可见的抛物线 |
| 旋转朝向 | 导弹头部始终沿速度方向（瞬时切线） | 发射时朝上→顶点水平→下落时朝下 |
| 尾焰方向 | 始终沿速度反方向 | 火焰向后喷射 |

```
// 弹道计算公式
每帧更新:
  vx = 水平速度 (恒定)
  vy = vy + 重力 * dt  (上一帧垂直速度 + 重力加速度)
  x  = x + vx * dt
  y  = y + vy * dt
  rotation = atan2(vy, vx)  // 导弹朝向速度方向
```

### 4.5 爆炸效果

| 阶段 | 时间 | 效果 |
|------|------|------|
| 命中瞬间 | 0ms | 白色闪光（全屏alpha 0.35，持续0.06s），比炮塔更亮 |

| 参数 | 值 | 说明 |
|------|-----|------|
| 触发 | 导弹爆炸瞬间 | 与爆炸同步 |

| 震动频率 | 30Hz | 快速抖动 |
| 持续时间 | 0.4s | |
| 衰减曲线 | ease-out（指数衰减） | `offset = maxOffset × e^(-5t) × sin(2π×30×t)` |
| 影响范围 | 整个棋盘区域（scene偏移量抖动） | 通过修改RenderSystem的sceneOffsetX/Y实现 |

### 4.8 伤害计算

```
基础伤害 = 导弹塔ATK

物理减伤 = 基础伤害 × (1 - 敌人护甲 / (护甲 + 100))

实际伤害 = 减伤后伤害 × 中心加成系数
```

**中心加成**（v1.1 修正 — 仅 L5 热压弹头被动生效）：

| 距爆炸中心距离 | 伤害比例 |
|--------------|---------|
| **0 ~ 10% 半径**（中心核心区） | **120%**（热压弹头加成） |
| 10% ~ 100% 半径 | **100%**（不再衰减） |

> 关键变更（v1.1）：
> - 旧版：30% 半径 ×1.3、60% 半径 ×1.0、100% 半径 ×0.7 的三段衰减——"清屏神器"
> - 新版：仅最中心 10% 半径 ×1.2，其余 100%——"精准制裁，奖励瞄准"
> - 设计意图：让玩家专注于"瞄准敌群核心"的微观决策，而不是"开炮即清场"的大手大脚

**无 L5 被动时**（L1~L4）：全爆炸范围 100% 伤害。

### 7.6 导弹定位地格的实现方式（关键设计决策）

导弹塔的与众不同在于**攻击目标不是实体，而是地格坐标**。现有 `ProjectileSystem` 的弹道需要一个 `targetId`（实体ID）来飞行。解决方案：

**创建临时"打击标记"实体**：
1. 导弹塔通过评分系统选出最佳地格 `(row, col)`
2. 将该地格中心转换为像素坐标：`(cx, cy) = (offsetX + col * tileSize + tileSize/2, offsetY + row * tileSize + tileSize/2)`
3. 创建一个不可见的"marker"实体（含 `Position` 组件，不含 `Visual` 组件）
4. 发射的 `Projectile.targetId` 指向该 marker 实体
5. 弹道到达 → `ProjectileSystem.onHit()` 触发 → `applySplash()` 在 `(hitX, hitY)` 处造成AOE伤害
6. marker 实体在命中后销毁

```typescript
// 伪代码（AttackSystem中的导弹塔分支）
const targetRow = bestTile.row;
const targetCol = bestTile.col;
const targetX = scene.offsetX + targetCol * tileSize + tileSize / 2;
const targetY = scene.offsetY + targetRow * tileSize + tileSize / 2;

// 创建临时标记实体
const markId = world.createEntity();
world.addComponent(markId, Position, { x: targetX, y: targetY });

// 发射导弹，目标指向标记实体
spawnProjectile(world, towerId, markId, towerTypeVal);
// 标记实体在 ProjectileSystem.onHit 命中后由 world.destroyEntity 清理
```

**优点**：完全复用现有 `ProjectileSystem` 的飞行、命中、AOE、爆炸特效逻辑，零侵入性修改。

## 八、平衡性考量

### 8.1 与其他塔的对比

> **具体数值以 [21-MDA §4.7 / §4.8](../50-data-numerical/50-mda.md#47-导弹塔战略核武器) 为准**。本节仅描述结构性差异：

| 塔 | 攻速 | 范围 | AOE | 特色 |
|----|------|------|-----|------|
| 箭塔L1 | 高 | 中（180） | 无 | DPS基准 |
| 炮塔L1 | 中 | 中（180） | 中（80px） | 溅射AOE+眩晕 |
| 导弹塔L1 | **极低** | **大（600px，~画面 1/3 宽）** | **大（130~190px）** | **战略 AOE + 不能命中飞行** |

**结构性差异**：
- 导弹塔不再是"全图武器"——必须根据敌人主路径选择塔位
- 导弹塔不能命中飞行敌——L4+ 需要搭配高射程/反空塔
- L5 热压弹头从"清屏"变为"精准制裁"——中心 10% 加成 ×1.2

### 8.2 风险与限制（v1.1 更新）

| 风险 | 缓解措施 |
|------|---------|
| 单体伤害过高，替代所有其他塔 | 极长装弹 + 高造价 + 高人口 + **限制射程 600px**，确保不能"导弹塔海" |
| ~~全图射程过于OP~~ | **已削弱**：射程 600px，塔位必须精心选择，强化空间策略 |
| 评分系统可能不直观 | 可选的发射预览：鼠标悬停时显示第一名地格的评分和预估伤害 + **射程圈视觉**显示 600px 范围 |
| BOSS波过于克制 | BOSS单体评分高是设计意图，但导弹长装弹期让玩家不能完全依赖 |
| **飞行敌制衡** | L4+ 飞行敌不被导弹爆炸命中，玩家必须搭配反空塔 |
| **分散敌制衡** | L6+ 分散敌移动中持续散开，降低密度因子，让导弹"等密集"变得困难 |

### 8.3 建议的关卡开放（v1.1 更新）

| 关卡 | 是否可用 | 制衡机制 |
|------|---------|---------|
| L1 平原 | ❌ | — |
| L2 沙漠 | ❌ | — |
| L3 冰原 | ❌ | — |
| L4 火山 | ✅ 解锁 | **同时引入飞行敌**——导弹无法包打 |
| L5 城堡 | ✅ | 飞行敌持续出现 |
| L6+（如有） | ✅ | **引入分散敌**——降低密度评分 |
| 无尽模式 | ✅ | 混合各种制衡敌人 |

> 理由：导弹塔属于高级战略武器，应在玩家已经熟悉基础塔防后再解锁。L4 火山的窄路径让导弹 AOE 发挥，但**同关卡引入的飞行敌**形成立即制衡，防止"导弹塔流"统治后期。
