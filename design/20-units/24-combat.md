# 战斗系统：公式骨架与机制规范

> **本文档不再持有数值**。所有具体数值（伤害系数常量、波次缩放、攻速上限等）见 [21-MDA数值设计](../50-data-numerical/50-mda.md)（**唯一真理源**）。
>
> 本文档定义：战斗公式的**形态**、机制的**规则边界**、视觉/反馈的**规范**。

---

## 1. 伤害公式骨架

### 1.1 基础伤害（除法减伤）

```
物理伤害 = ATK × (1 - 护甲 / (护甲 + K_armor))
魔法伤害 = ATK × (1 - 魔抗 / (魔抗 + K_magicResist))
真实伤害 = ATK   （无视护甲与魔抗）
```

- `K_armor` / `K_magicResist`：减伤常数，**在 21 中定义**（当前 = 100）。
- 采用除法形式，确保减伤永远 < 100%，**不会出现零伤或免疫**。
- 减伤常数提取为可配置项 `armorFormulaConstant` / `magicResistFormulaConstant`，关卡/波次可微调（详见 21）。

### 1.2 最终伤害

```
最终伤害 = 基础伤害 × (1 + 伤害加成%) × (1 - 伤害减免%) × 暴击倍率
```

- 暴击率 = 基础暴击率 + Buff 暴击率
- 暴击率上限、暴击倍率基础值与 Buff 加成上限**均在 21 中定义**

### 1.3 伤害下限保护

无论修饰系数如何叠加，最终伤害不会低于 1（仍命中视为生效，避免 0 伤死循环）。

---

## 2. 攻速公式骨架

```
实际攻击间隔 = 基础攻击间隔 / (1 + 攻速加成%)
```

- 攻速加成% 上限**在 21 中定义**（当前 200%，即最高 3 倍攻速）
- 上限保证 `攻击间隔 ≥ 基础间隔 / 3`，防止帧爆炸

---

## 3. 移速与减速公式骨架

```
实际移速 = 基础移速 × (1 + 移速加成%) × (1 - 总减速%)
最低移速 = 基础移速 × MOVE_SPEED_FLOOR
```

- `MOVE_SPEED_FLOOR`：**移速下限系数**，在 21 中定义（当前 20%）。
- 减速永不让单位完全静止——眩晕/冰冻是独立机制，走眩晕通道（§5）。

### 3.1 减速叠加规则（关键）

> 解决 P1-#9：天气 × Buff × 技能叠加导致 100% 卡死。

减速来源分为两类，**按以下规则计算总减速**：

1. **同类来源**（如多个冰塔的减速 Buff）：**取 max**，不相加
2. **不同来源**（如冰塔减速 + 雪天移速惩罚 + 嘲讽减速）：**相加**
3. **跨类总减速上限**：`SLOW_TOTAL_CAP`（在 21 中定义，当前 80%）
4. 触发 `freeze` / `stun` 走独立通道，绕过减速叠加，直接令移速为 0（受 §5 控制）

伪代码：

```ts
const sameClassMax = groupBy(slows, "class").map(g => max(g.values));
const totalSlow = min(sum(sameClassMax), SLOW_TOTAL_CAP);
const actualSpeed = baseSpeed * (1 + boost) * (1 - totalSlow);
const floor = baseSpeed * MOVE_SPEED_FLOOR;
return max(actualSpeed, floor);
```

## 5. 控制机制规范

### 5.1 眩晕（Stun）

- 触发：携带 `stun_on_hit` 机制的攻击命中
- 效果：移速 = 0、攻击禁用、技能禁用
- 视觉：单位金色闪烁（详见 12）
- **Boss 免疫**（`boss_immune_stun=true`）
- 持续时间：在单位配置中定义（如炮塔的 `stunDuration`，见 21）
- **递减返还（Diminishing Return）**：同一目标在 3 秒内被多次眩晕，第 N 次有效时长 = 基础 × max(0.2, 1 - 0.3×(N-1))，防止"眩晕锁"

### 5.2 冰冻（Freeze）

- 触发：`slow_on_hit` 满层后触发 `freeze_at_max_stacks`
- 效果：与眩晕等价（移速 0 + 攻击禁用），但走独立通道
- **Boss 免疫**
- 冰冻结束时若有 `levelPassive` 触发 AOE 伤害（详见 21 冰塔条目）

### 5.3 嘲讽（Taunt）

- 触发：盾卫主动技能（详见 04）
- 效果：范围内敌人强制将目标切换为发起者，并降低部分移速
- **Boss 不可被嘲讽**

## 6. 弹道系统规范

- 攻击塔的攻击通过弹道实体表达，**非即时伤害**
- 弹道形状与单位主色一致（视觉规范见 12 §弹道）
- 弹道速度差异化（具体数值见 21），用于强化"塔的性格"——慢炮弹/快闪电
- 弹道未命中（目标已死亡/超出场景）时按规则销毁：

| 攻击模式 | 命中规则 |
|---------|---------|
| `single_target` | 目标已死亡 → 选最近敌人替代，否则销毁 |
| `aoe_splash` | 落点固定，命中所有范围内敌人 |
| `chain` | 链式跳跃，每跳重新选最近的未被链击过的敌人 |
| `piercing` | 沿固定方向飞行至最大射程，命中线上敌人 |
| `global_aoe` | 飞向预计算地格中心，到达后引爆 |

## 7. 血条与受击反馈规范

- 高度：3px
- 偏移：单位中心上方 -14px
- 颜色梯度（按 HP 百分比）：
  - `> 60%` → 绿 `#4CAF50`
  - `30%-60%` → 黄 `#FFC107`
  - `< 30%` → 红 `#F44336`
- 受击闪白：100ms 白色叠加
- Boss 血条独立 UI（顶部全屏宽，详见 09）

## 8. 难度曲线骨架

> **具体波次倍率表见 [21 §8](../50-data-numerical/50-mda.md#8-波次难度曲线重校准)**。本文档只描述形态：

- 关卡模式：1-3 入门 → 4-8 发展 → 9-15 挑战 → 16-20 极限
- 无尽模式：20 波后进入线性递增段（HP/移速/护甲/魔抗每波 +固定增量，具体值见 21）
- 速度倍率有上限（防止快兵移速爆表）
- 无尽模式后期采用**软上限**机制：HP 增长率在 50 波后阶梯式衰减，防止指数膨胀

---

## 9. 暴击与穿透说明

- 暴击：独立判定通道，不与伤害加成相乘冲突
- 穿透：贯穿伤害衰减按"穿透人数"指数衰减（系数见 21）
- **公式中所有"上限/下限/常数"集中在 21 文档配置化**，便于关卡设计/平衡调试单点修改
