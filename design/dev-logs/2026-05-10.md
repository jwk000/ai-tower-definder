# 开发日志 — 2026-05-10

## 16-美术资产设计文档

### 背景
之前 demo 阶段使用极简几何形状 + 文字标签的组合，表现力不足。需要在不引入外部美术素材的前提下，充分利用 PixiJS WebGL 渲染引擎的能力，大幅增强视觉表现。

### MDA 分析
- **M (机制)**: 塔防布阵、单位操控、资源经济、波次推进、天气影响
- **D (动态)**: 击杀区构建、阵型调整、天气适应、经济节奏、BOSS 应对
- **A (美学)**: 挑战感（清晰视觉层级）、发现感（主题随机性）、表达感（可读布局）、感官愉悦（粒子+动画）、叙事感（5主题环境故事）

### 设计输出
创建了 `design/16-art-assets-design.md`，包含：

1. **11 层场景分层架构**
   - 在现有 7 层基础上扩展：增加 ShadowLayer（投影）、ScreenFXLayer（屏幕特效）、WeatherLayer 独立、BackgroundLayer 升级为程序化天空+远景

2. **复合几何体系统**
   - 从单一形状升级为 3-8 个几何原语的组合
   - 部件类型库：body_rect/circle/hexagon/triangle + symbol_sword/shield/bow/crown/wings/halo + deco_spikes/particles/glow
   - 每个单位 = 基础部件 + 身份部件 + 动态部件

3. **动画状态机 (Animation FSM)**
   - 6 种状态：SPAWN → IDLE ⇄ ATTACK / HIT → DEATH → 移除（+ SPECIAL 用于 BOSS）
   - 统一数据结构：`{current, timer, phase, config, nextState}`
   - 每帧根据 state + phase 计算所有子部件的 transform

4. **全部 25 种单位的复合几何体详细设计**
   - 6 塔（箭/炮/冰/电/激光/蝙蝠）：基座+塔身+符号特征+等级标识
   - 6 我方单位（盾/剑/弓/祭司/工程/刺客）：身体+武器符号+脚下阴影
   - 7 敌人（小兵/快兵/重装/法师/自爆/指挥官Boss/攻城兽Boss）
   - 6 中立建筑（金矿/能量塔/陷阱/泉水/宝箱/基地/出生点）

5. **弹道系统增强**
   - 炮塔抛物线算法、电塔锯齿闪电算法
   - 各弹道的尾迹+命中特效升级

6. **5 主题完整视觉整合**
   - 每个主题的天空渐变→远景几何→地面配色→装饰物→氛围光

7. **性能设计**
   - 分层重绘策略（静态层脏标记，动态层逐帧）
   - 60fps 预算分配（EntityLayer 6ms 主要开销）
   - ParticleContainer 批量、Graphics 复用、LOD 概念

### 与现有设计对齐

- 升级而非推翻：单形状→复合体、散列动画→状态机、7层→11层
- 对齐 `15-refactoring-plan.md` 的 PixiJS 迁移路线

## Phase 0-2 重构实现

### 关键发现
- **PixiJS 全部是死代码**：`PixiApp.ts`、`PixiRenderer.ts` 已定义但从未被实例化。所有 PixiJS Graphics/Text 对象是孤儿节点（未连接到 Application stage）
- **Canvas 2D 是唯一活跃渲染器**：`Renderer.ts` 的 `push()` 命令缓冲是实际渲染路径
- **`PixiApp` 7 层架构、`PixiRenderer` 向后兼容 API 均已实现但未接线**

### Phase 0: PixiJS 接入游戏循环

**文件变更**:
- `src/render/PixiApp.ts` — 重写：7层→11层、接受现有 canvas、公开 `init(canvas)` 方法、`resize()` 改为 public
- `src/core/Game.ts` — 重写：`new Renderer(canvas)` → `new PixiApp()` + `new PixiRenderer(stage)`，`start()` 改为 async `await pixiApp.init(canvas)`
- `src/render/PixiRenderer.ts` — 新增 `context` 属性（隐藏 canvas 用于文本测量向后兼容）、新增 `resize()` no-op、新增公开 `container` 属性、修复 alpha 逐形状传递 bug
- `src/systems/RenderSystem.ts` — `import { Renderer }` → `import { PixiRenderer }`
- `src/systems/UISystem.ts` — 同上
- `src/systems/LevelSelectUI.ts` — 同上
- `src/main.ts` — 孤儿容器接入：`effectContainer` → `pixiApp.effectLayer`、`weatherOverlayGfx` → `pixiApp.weatherLayer`、`start()` 改为 `.catch()` 处理异步错误

### Phase 1: 场景分层 + 背景渲染

**新增文件**:
- `src/render/BackgroundRenderer.ts` — 5 主题程序化背景（天空渐变 16 色带 + 远山三角叠层 + 云/极光/火山/金字塔/城堡剪影），使用 PixiJS Graphics API

**文件变更**:
- `src/main.ts` — 初始化 `BackgroundRenderer`，传入 `pixiApp.backgroundLayer`

### Phase 2: 动画状态机 + 复合几何体

**新增文件**:
- `src/render/AnimationSystem.ts` — 类型定义：`AnimState` 枚举（Spawn/Idle/Attack/Hit/Death/Special）、`AnimStateConfig`、`AnimInstance`、`AnimTransform`、`createAnimInstance()` 工厂
- `src/data/unitVisuals.ts` — 25 种单位的复合几何体配方：`TOWER_VISUALS`（6 种）、`UNIT_VISUALS`（6 种）、`ENEMY_VISUALS`（7 种），每种包含 3-9 个 ShapePart
- `src/render/CompositeRenderer.ts` — 多部件 PixiJS Graphics 渲染器：`drawEntity(eid, x, y, recipe, anim)`、`hideEntity()`、`removeEntity()`、`clear()`，支持 AnimTransform 缩放/偏移/旋转/染色

**文件变更**:
- `src/systems/RenderSystem.ts` — 新增 `CompositeRenderer` 参数 + `drawCompositeTower()` 方法，6 种塔实体使用复合渲染（带待机上下浮动动画），非塔实体保留 push() 回退
- `src/main.ts` — 初始化 `CompositeRenderer`，传入 `pixiApp.entityLayer`
