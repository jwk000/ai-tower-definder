# UI / UX 设计

> 横屏布局 | 符号优于文字 | 卡牌驱动交互
>
> **v3.0 重写**：根据 [25-card-roguelike-refactor](../10-gameplay/10-roguelike-loop.md) 方案，底部工具栏改为「手牌区」，新增关间节点面板、商店界面、秘境界面、卡池界面、Run 结算界面。删除关卡选择网格（改为 Run 入口）、删除三星菱形显示。

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **符号优于文字** | 场景中不显示单位名称。形状/动画/特效让玩家一眼识别 |
| **卡牌即交互锚** | 战斗中所有部署操作都从手牌发起，手牌区是 UI 的视觉重心 |
| **按需显示信息** | 选中实体时才弹出详情面板；HUD 只展示必要数字 |
| **拖拽优先** | 出卡部署用拖拽，给玩家"打出去"的物理感 |
| **关间正式化** | 关间节点（商店/秘境）使用全屏模态面板，与战斗 UI 隔离 |

### 2.1 区域划分与精确尺寸

| 区域 | 内容 | 尺寸 | 位置 |
|------|------|------|------|
| 顶部 HUD | 能量 / 金币 / 波次 / 敌军 / 水晶 HP / Run 进度 / 倍速 / 暂停 | 1920×36 | y=0 |
| 全屏背景 | 天空渐变 + 远景 + 云 + 天气粒子 | 1920×1080 | 全屏 |
| 地图区域 | 21×9 格战场 | 1344×576 | 居中(x=288, y=动态) |
| **手牌区** | 4 张手牌（横向排列），上限 8 张 | 800×180 | bottom-center 锚点，offset(0, -130)（详见 20-responsive §4.5.2） |
| **牌组/弃牌计数** | 📚 牌组:X/12 🗑 弃牌:X | 50×70 | bottom-right 锚点，offset(-200, -160)（详见 20-responsive §4.5.1） |
| 选中信息面板 | 选中地图实体时浮现 | ~300×100 | 手牌区上方动态浮出 |
| 关间节点面板 | 全屏模态（商店 / 秘境） | 1920×1080 | 模态覆盖 |
| 卡池界面 | 主菜单子页面 | 1920×1080 | 全屏 |
| Run 结算 | 全屏模态 | 1920×1080 | 模态覆盖 |

### 2.2 UI 死区规范（保留 v1.1 设计，扩展手牌区）

死区由 UI 区域的实际位置和尺寸**动态计算**，不允许硬编码：

```typescript
const UI_DEAD_ZONES: UIDeadZone[] = [
  { region: 'top_hud',     hitTest: (x, y) => y < LayoutManager.scaleY(HUD_HEIGHT) },         // HUD 36px
  { region: 'hand_area',   hitTest: (x, y) => y > LayoutManager.viewportH - LayoutManager.scaleY(HAND_HEIGHT) },  // 手牌区 180px
  { region: 'tooltip_overlay', hitTest: (x, y) => SelectedInfoPanel.contains(x, y) },
  // 模态覆盖层（暂停 / 关间节点 / Run 结算）自带"全屏死区"
];
```

事件分发流程：

```
pointerDown(x, y)
  → 模态覆盖层激活时 → 100% 由模态处理
  → 在手牌区内 → 由 HandSystem 处理（卡牌选中、拖出）
  → 在 HUD 内 → 仅展示，拦截不响应
  → 否则 → 由 SelectSystem / DropTargetSystem 处理
```

### 2.3 死区优先级

1. 模态覆盖层（暂停 / 关间面板 / Run 结算）
2. 手牌区
3. 顶部 HUD
4. 选中信息面板

## 3. 手牌系统 UI（v3.0 新增核心）

### 3.1 手牌区布局

```
       ┌───────────────────────────────────────────┐
       │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐          │
       │  │箭塔 │ │剑士✦│ │火球 │ │金矿 │          │  📚牌组:8/12
       │  │ 🏹  │ │ ⚔  │ │ 💥  │ │ ⛏  │          │  🗑弃牌:0
       │  │ ◇3  │ │ ◇2  │ │ ◇3  │ │ ◇4  │          │  （bottom-right
       │  └─────┘ └─────┘ └─────┘ └─────┘          │   独立锚点）
       └───────────────────────────────────────────┘
       手牌区 800×180，bottom-center 锚点 offset(0,-130)
```

| 卡牌槽位 | 尺寸 | 间距 |
|---------|------|------|
| 单卡 | 120×168 | 横向 16px 间距 |
| 整体 | 4 张默认水平居中排列 | 上限 8 张（永久升级） |

权威锚点定义见 [20-responsive-layout §4.5.2](./41-responsive-layout.md#452-手牌区锚点)。

### 3.2 单卡视觉规范

```
┌────────────────┐
│ ╭────────────╮ │   ← 边框：稀有度颜色
│ │            │ │       Common=白 / Rare=蓝 / Epic=紫 / Legendary=金
│ │  单位图形  │ │   ← 主图区：放卡牌单位的几何符号（96×80）
│ │            │ │
│ ╰────────────╯ │
│  剑士 ✦L2      │   ← 名字 + 等级标识（菱形数量）
│  ◇3    ❤120   │   ← 能量消耗 + 关键属性快览
└────────────────┘
   120 × 168
```

| 元素 | 位置 | 样式 | 说明 |
|------|------|------|------|
| 边框 | 整卡 | 2px 实线 | 颜色由稀有度决定 |
| 卡背图案 | 整卡背景 | 主题色半透明 | 单位卡 / 建筑卡 / 法术卡分主题色 |
| 主图区 | 卡牌上部 | 96×80 | 显示卡牌单位的几何符号 |
| 名字 | 卡牌中下部 | 12px 白色居中 | 配置 `name` 字段 |
| 等级菱形 | 名字右侧 | 金色菱形 | L1 不显示，L2-L5 显示对应数量菱形 |
| 能量消耗 | 卡牌底部左 | ◇ + 数字 | 蓝色菱形 + 能量值 |
| 关键属性 | 卡牌底部右 | 紧凑符号 + 数字 | 单位卡显示 HP/ATK，法术卡显示效果摘要 |

### 3.3 卡牌交互流程

| 操作 | 触发 | 视觉反馈 |
|------|------|---------|
| 鼠标悬停 | hover 单张卡 | 卡片向上偏移 12px + 轻微缩放（1.05×），右上方弹出详细信息卡片 |
| 选中卡 | 鼠标按下 | 卡片提升至最前，外发光高亮 |
| 拖出卡 | 拖动 | 卡片消失，光标跟随幽灵预览（单位卡显示建造预览 / 法术卡显示范围预览） |
| 合法目标 | 拖入合法区域 | 地图格绿色高亮 + 卡片轮廓变绿 |
| 非法目标 | 拖入非法区域 | 红色高亮 + 卡片轮廓变红 |
| 释放卡 | 松开鼠标 | - 合法 → 部署 / 释放，扣能量，卡进弃牌堆<br>- 非法 → 取消，卡回手牌（不扣能量） |
| 能量不足 | 选中能量 > 当前能量的卡 | 卡片暗淡 + 显示红色"能量不足"标识 |

### 3.4 牌组 / 弃牌堆指示器

手牌区右侧显示：

```
📚 牌组: 8/12     ← 剩余可抽卡数 / 总卡组大小
🗑 弃牌: 4        ← 弃牌堆数量
```

- 点击 `📚` 牌组图标 → 弹出本局卡组全览（半透明面板）
- 点击 `🗑` 弃牌堆图标 → 弹出弃牌堆全览
- 当牌组用尽时，弃牌堆**自动洗回**牌组（视觉上动画反馈）

### 3.5 法术卡跨波保留标识

带 `persistAcrossWaves=true` 标记的法术卡：
- 卡片右上角显示 ✦ 标识（金色星）
- 波次结束时不进入弃牌堆，留在手牌
- 抽卡时跳过这些卡占用的槽位（不强制重抽）

## 4. 顶部 HUD 规范（v3.0 更新）

| 显示项 | 符号 | 格式 | 说明 |
|--------|------|------|------|
| 能量 | 蓝色菱形 ◇ | `◇ 5/10` | 当前能量 / 上限 |
| 金币 | 金色圆点 ● | `● 120` | 整局 Run 累计金币 |

## 5. 选中信息面板（保留并适配）

仅选中地图实体时浮现，从手牌区上方动态浮出（避免遮挡手牌）：

```
┌──────────────────────────────────┐
│ 🏹  箭塔 ✦L2                      │
│ ❤ 350/350    ⚔ 20    🎯 5格      │
│ [升级 ●150]  [回收]               │
└──────────────────────────────────┘
```

- 实体图形符号 + 名称
- HP 条 + 数值
- 核心属性（ATK / 范围 / 攻速）
- 操作按钮：
  - **升级**：消耗金币（关内只能查看，关内不可升级，灰显并显示"商店内可升级"）
  - **回收**：返还能量（参考 06-economy §7.1 公式）

> **v3.0 关键变更**：关内不可直接花金币升级实体。升级必须在关间商店进行（升级整张卡而非单个实例）。`[升级]` 按钮关内灰显，标注"前往商店升级"。
