## 2. 节点三状态

### 2.1 状态定义

| 状态 | 触发条件 | 视觉 | 可点击 |
|---|---|---|---|
| 已通关 ✓ | `RunManager.currentLevelIdx > 节点序号` | 灰白底 + 金色对勾 + 静止 | 否（仅点击触发"回顾本关战绩"浮窗，可选实现） |
| 当前可挑战 ● | `RunManager.currentLevelIdx === 节点序号` | 关卡主题色 + 发光脉冲（呼吸动画 1.5s 周期）+ 节点上浮 4px | 是 → 触发"挑战关卡 N"主按钮高亮 |
| 未解锁 ○ | `RunManager.currentLevelIdx < 节点序号` | 深色剪影（30% 不透明度）+ 锁形覆盖图标 | 否（点击触发"通过前一关解锁"浮窗提示，2s 淡出） |

### 2.2 路径段视觉

| 路径段（节点 N → N+1） | 触发条件 | 视觉 |
|---|---|---|
| 已通关段 | `currentLevelIdx > N+1` | 实线（金色）+ 静止 |
| 即将走的段 | `currentLevelIdx === N` 且节点 N 是当前 | 虚线（白色）+ 流动动画（marching ants，2s 周期） |
| 未来段 | `currentLevelIdx < N` | 半透明虚线（白色 20% 不透明度）+ 静止 |

### 2.3 节点形状与主题色

> 节点形状由关卡主题决定，与 [15-level-themes](../10-gameplay/15-level-themes.md) 一一对应；主题色仅本文档定义"色相方向"，**精确色值与纹理** → [42-art-assets](./42-art-assets.md)。

| 3 | 山地隘口 | 六边形 | 棕黄 | 木栅 |
| 4 | 雷暴荒原 | 菱形 | 紫蓝 | 闪电柱 ⚡ |
| 5 | 沙漠遗迹 | 三角 | 沙金 | 石柱 |

## 4. 胜利结算 + 3 选 1 一屏面板

### 4.1 触发与定位

- **触发时机**：关 N 战斗胜利（最后一波清空 + 水晶存活），N ∈ {1..8}
- **覆盖关系**：全屏模态覆盖战斗界面，**不返回路线图直到玩家点击 3 选 1**
- **特例**：关 N=8（终战 W23）胜利时**不显示本面板**，直接跳转 Run 结算（→ §6）

### 4.2 面板结构（一屏一体）

```
全屏 1920×1080 模态覆盖战斗界面：

┌──────────────────────────────────────────────────────────────┐
│                  🏆 关卡 N 通过！                              │
│              （主题色边框 + 主题图标）                          │
├──────────────────────────────────────────────────────────────┤
│  战绩                                                          │

├──────────────────────────────────────────────────────────────┤
│  本关奖励                                                      │

| 选项 | 边框色 | 图标 | 副标题语气 |
|---|---|---|---|
| 商店 | 金黄 | 🏪 | 稳健 / 资源管理 |

| 跳过 | 灰白 | ⏭ | 直接 / 零代价 |

### 4.6 跳过选项行为定义（v3.3 新增）

| 维度 | 规则 |
|---|---|
| 触发 | 玩家点击"直接前进" |
| 资源变化 | **零变化**：不消耗金币 / 不消耗 HP / 不获额外奖励 |
| 关卡推进 | 立即触发 `RunManager.advanceToNextLevel()` + 关闭 3 选 1 面板 + 返回路线图（节点 +1 动画） |
| 设计意图 | 当前金币不足以买卡 + 当前 HP 充裕不需要秘境冒险 时的稳健下限；避免强制玩家进入两个不需要的子面板 |
| 平衡考量 | 不在跳过分支提供任何奖励，确保"跳过"是真正的下限选项而非最优路径 |

> **本节修订对 [10-roguelike-loop §3.1](../10-gameplay/10-roguelike-loop.md#31-节点结构) 的影响**：原 §3.1"二选一面板"需扩展为三选一，本文档 §4 接替。10-roguelike-loop 下次重审时应在 §3.1 顶部加 cross-ref 指向本文档。

## 5. 子面板覆盖与关闭返回

### 5.1 覆盖关系总览

```
路线图 (L0)
   └── 战斗界面 (L1，关 N 进行时)
         └── 胜利结算 + 3 选 1 面板 (L2，§4)
               ├── 点击商店 → 商店面板 (L3，全屏覆盖 L2)
               │     └── 关闭 → 节点 +1 → 返回路线图 L0
               ├── 点击秘境 → 秘境面板 (L3，全屏覆盖 L2)
               │     └── 关闭 → 节点 +1 → 返回路线图 L0
               └── 点击跳过 → 节点 +1 → 返回路线图 L0
```

### 5.2 子面板自身规范

| 子面板 | 视觉 + 锚点权威 | 关闭行为定义来源 |
|---|---|---|
| 商店面板 | [40-ui-ux §7](./40-ui-ux.md#7-商店界面-v30-新增) + [41-responsive-layout §4.5.4](./41-responsive-layout.md#454-商店面板) | 本文档 §5.3 |
| 秘境面板 | [40-ui-ux §8](./40-ui-ux.md#8-秘境界面-v30-新增) + [41-responsive-layout §4.5.5](./41-responsive-layout.md#455-秘境事件面板) | 本文档 §5.3 |

> **商店/秘境内容差异**（商品池 / 事件池 / 数值）在各自来源文档定义，本文档**不重复**。本文档只定**关闭返回路径**。

### 5.3 关闭返回流程（本文档权威）

| 子面板 | 关闭触发 | 关闭后流程 |
|---|---|---|
| 商店 | 玩家点击"离开商店"按钮 OR ESC 键 | 立即关闭商店面板 → 立即关闭 3 选 1 面板 → 触发 `RunManager.advanceToNextLevel()` → 路线图节点 N+1 上浮发光 + 路径段 N→N+1 切换为实线 + 主按钮变为"挑战关卡 N+1" |
| 秘境 | 玩家在秘境内选择任一选项（含"离开"）后事件结算完成 | 同上 |
| 跳过 | 玩家在 3 选 1 面板点击"直接前进" | 跳过子面板步骤，直接执行节点推进流程 |

### 5.4 节点推进动画规范

| 阶段 | 视觉 | 时长 |
|---|---|---|
| 1. 面板淡出 | 黑色蒙层 alpha 1 → 0 | 200ms |
| 2. 旧节点 N 状态切换 | 主题色 + 发光 → 灰白 + ✓ 对勾 | 300ms（缓动 ease-out） |
| 3. 路径段 N→N+1 状态切换 | 虚线流动 → 金色实线 | 与 2 同步 |
| 4. 新节点 N+1 状态切换 | 深色剪影 → 主题色 + 发光脉冲 | 300ms |
| 5. 主按钮更新 | "挑战关卡 N" 隐藏 → "挑战关卡 N+1" 浮入 | 200ms |
| 总计 | 1000ms 内完成全部过渡 | — |

### 5.5 异常路径

| 异常 | 处理 |
|---|---|
| 玩家在子面板中点击 ESC | 等价于触发"离开商店/秘境"按钮，进入正常关闭流程（不弹二级确认） |
| 玩家在 3 选 1 面板中按 ESC | **不响应**（避免误触跳过 Run），需明确点击"跳过"卡才能跳过 |
| 玩家在 3 选 1 面板中按右键 / 关闭按钮 | 同上，无效 |
| 浏览器刷新 / Tab 关闭 | RunManager 状态由 [61-save-system](../60-tech/61-save-system.md) 持久化，下次回到主菜单显示"继续 Run"按钮 |

## 6. 终战 W23 特例

### 6.1 关 8 通过后行为

```
关 8 战斗胜利 → 不显示 3 选 1 → 立即触发终战 W23 战斗
                                          ↓
                              终战 W23 胜利 → Run 结算面板
                              终战 W23 失败 → Run 失败结算

### 6.2 路线图上的终战节点行为

| 路线图状态 | 终战节点视觉 | 可点击 |
|---|---|---|
| `currentLevelIdx < 8` | 深色剪影 + 皇冠剪影 + 锁形覆盖 | 否（"通过关 8 解锁") |

## 7. ESC 退出 Run 确认

### 7.1 触发条件

| 玩家位置 | ESC 行为 |
|---|---|
| 路线图（任何节点状态） | 弹出"退出 Run 确认弹窗"（本节定义） |
| 战斗界面 | 弹出暂停菜单（保持现有行为，本文档不接管） |
| 胜利结算 + 3 选 1 面板 | **不响应**（§5.5） |
| 商店 / 秘境子面板 | 等价"离开"按钮（§5.3） |
| Run 结算面板 | **不响应**（强制玩家点击"返回主菜单"/"立即新 Run") |

### 7.2 确认弹窗结构

```
┌──────────────────────────────────────┐
│           ⚠ 退出 Run 确认             │
├──────────────────────────────────────┤
│  退出当前 Run 视为失败：               │
│  • 当前 Run 进度将清空                │
│  • 已积累的金币 / 技能点全部归零        │
│  • 已通关关卡数仅作战绩展示，无 meta 收益│
├──────────────────────────────────────┤
│      [取消]            [确认退出]    │
└──────────────────────────────────────┘
```

> ⚠️ **v3.4 调整**：原 v3.3 描述"金币转碎片"/"按通关关卡数发放碎片入账"已删除。v3.4 死亡纯无回报（详见 [v3.4-MAJOR-MIGRATION §1.3](../v3.4-MAJOR-MIGRATION.md#13-哲学层声明) 哲学层声明）。

### 7.3 锚点

| 元素 | 锚点 | 说明 |
|---|---|---|
| 弹窗背景 | `anchor: middle-center, offset(0, 0)`, `size: 480 × 220` | 居中 |
| 标题 | `anchor: top-center, offset(0, 30)` 相对于弹窗 | "⚠ 退出 Run 确认" |
| 说明文字 | `anchor: middle-center, offset(0, -10)` 相对于弹窗 | 3 行 |
| 取消按钮 | `anchor: bottom-left, offset(60, -30)` 相对于弹窗 | 灰色，关闭弹窗回路线图 |
| 确认退出按钮 | `anchor: bottom-right, offset(-60, -30)` 相对于弹窗 | 红色，触发 Run 失败结算流程 |
| 蒙层 | `anchor: middle-center, offset(0, 0)`, `size: fullWidth × fullHeight` | 60% 不透明黑 |

### 7.4 行为定义

| 按钮 | 行为 |
|---|---|
| 取消 | 关闭弹窗 → 回到路线图 → 无任何状态变化 |
| 确认退出 | 关闭弹窗 → 直接跳转 Run 失败结算（[40-ui-ux §10](./40-ui-ux.md#10-run-结算界面-v30-新增)，v3.4 重构中）；**v3.4 起所有 Run 内资源清零，无 meta 入账** |

### 7.5 设计意图

ESC 在路线图等价"放弃 Run"，但**不**等价"暂停 Run"——Run 是连贯的长征，没有暂停语义。玩家中途想停，必须明确接受"视为失败"的代价。

## 8. 与 RunManager 状态机锁定

### 8.1 不新增字段原则

本文档不在 [`src/unit-system/RunManager.ts`](../../src/unit-system/RunManager.ts) 上新增任何字段。所有路线图状态全部由现有 API 派生：

| UI 状态 | 派生来源（RunManager 既有 API） |
|---|---|
| 当前节点序号 | `RunManager.currentLevelIdx`（既有） |
| 节点三状态判定 | `currentLevelIdx` 与节点序号比较（§2.1） |
| 路径段状态 | 同上 |
| 终战可挑战 | `currentLevelIdx === 9`（即关 8 已通关、终战待战） |
| 顶栏水晶 HP | `RunManager.crystalHp`（既有，初始通过 `initRun()` 设置，由 `damageCrystal()` / `healCrystal()` 修改） |
| 顶栏金币 | `RunManager.gold`（既有，由 `addGold()` 修改） |
| Run 是否死亡 | `RunManager.isDead`（既有） |
| 关后 3 选 1 选定结果 | UISystem 局部状态，**不**持久化（玩家做选择 = 立即触发对应面板，无中间态） |

### 8.2 关后推进时序

```
关 N 战斗胜利
   │
   ▼
UISystem 显示胜利结算 + 3 选 1 面板（§4）
   │
   ▼ (玩家点击 3 选 1)
   │
   ├─ 商店：UISystem 显示商店面板（§5.2）
   │     └─ 商店关闭 → 触发 RunManager.advanceToNextLevel()
   │
   ├─ 秘境：UISystem 显示秘境面板（§5.2）
   │     └─ 秘境结算 → 触发 RunManager.advanceToNextLevel()
   │
   └─ 跳过：直接触发 RunManager.advanceToNextLevel()
   │
   ▼
UISystem 收到 RunManager "currentLevelIdx 变化" 事件
   │
   ▼
路线图重新渲染：旧节点 ✓ / 路径段实线 / 新节点高亮（§5.4 动画）
   │
   ▼
玩家在路线图点击"挑战关卡 N+1"按钮 → 进入关 N+1 战斗
```

### 8.3 路线图入口时机

| 时机 | 路线图是否显示 |
|---|---|
| 主菜单"开始新 Run" | `initRun()` 完成 → 立即显示路线图（关 1 高亮） |
| 主菜单"继续 Run" | 从存档恢复 → 立即显示路线图（按存档 `currentLevelIdx` 高亮） |
| 关 N 胜利 + 3 选 1 选定 + 子面板关闭 | 节点 +1 动画后回到路线图 |
| 终战胜利 / Run 失败 | **不**回路线图，直接 Run 结算 |
| 玩家在路线图按"挑战关卡 N" | 路线图关闭 → 进入关 N 战斗 |

### 8.4 与 GamePhase 的关系

| GamePhase | 路线图渲染 |
|---|---|
| `Deployment` / `Battle` / `WaveBreak` / `Victory` / `Defeat` | 不渲染（路线图与关内 GamePhase **互斥**） |
| Run 启动后 / 关后过渡时 / 玩家手动调出（点击节点按钮） | 渲染 |

> 路线图作为"关外 Run 长征"层，**不**进入 GamePhase 状态机。GamePhase 仅在战斗界面内有意义。
