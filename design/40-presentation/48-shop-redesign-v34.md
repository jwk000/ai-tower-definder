## 2. 资源架构（v3.4 三资源）

### 2.1 三资源梳理表

| 资源 | 作用域 | 获取 | 消耗 | 持久化 |
|---|---|---|---|---|
| **能量 E** | 关内（战斗时） | 每秒回复 + 击杀回收（既有规则） | 出卡（既有规则） | Run 结束清零 |
| **金币 G** | 关间（商店） | 关内击杀 + 关卡通关奖励 + 秘境部分事件 | 商店买卡 / 买功能卡 / 刷新 / 秘境部分事件 | Run 结束清零 |
| **技能点 ✦**（v3.4 新增） | 关间（卡池技能树） | **商店买技能点卡** / 部分秘境事件 / 关卡通关奖励（量小） | 在卡池界面分配到任意单位卡的技能树节点 | Run 结束清零 |

### 2.2 火花碎片彻底废弃声明

> 详见 [v3.4-MAJOR-MIGRATION §1](../v3.4-MAJOR-MIGRATION.md)。本文档强制约束：

- ❌ 商店 UI **不出现**碎片余额显示
- ❌ 商店商品 **不消耗**碎片
- ❌ 商店商品 **不奖励**碎片
- ❌ Run 结算 **不**入账碎片（[47-level-map-ui §7.4](./47-level-map-ui.md#74-行为定义) Run 失败结算需对应修订）
- ❌ 主菜单 **无**碎片余额显示
- ❌ 卡池界面 **无**"未解锁 200 碎片"按钮（所有卡开局即解锁）

### 2.3 技能点资源语义

- **本 Run 临时**：Run 开始时 `skillPoints = 0`，Run 结束时（胜利 / 失败 / ESC 退出）清零
- **关间获取，关间消费**：
  - 获取场景：商店（主要） + 秘境事件（偶现） + 关卡通关额外奖励（小量，按关数递增）
  - 消费场景：**卡池界面（关间）** —— 玩家点开任意单位卡的技能树面板，分配技能点解锁该单位的技能节点
  - **不可在关内（战斗时）消费**——避免战斗暂停纠结
- **不可转换**：技能点不可逆转换为金币 / 不可转换为任何 meta 资源（v3.4 已删除 meta 资源）
- **节点选择对本 Run 永久生效**：玩家在卡池界面给"箭塔"点了"贯穿"节点 → 本 Run 后续所有放出的箭塔实例都有贯穿；Run 结束所有节点选择重置

> 各单位的具体技能树节点 / 节点价格 / 节点效果 → 待建 `design/20-units/22-skill-tree-overview.md` + 各单位独立文档。本文档**不**定义。

## 3. CardType 拓展：shop_item 为第 5 类

### 3.1 CardType 枚举变更

> 由本文档授权拓展，与 [21-unit-roster](../20-units/21-unit-roster.md) §7 卡牌目录 + [10-roguelike-loop §2](../10-gameplay/10-roguelike-loop.md) 协同；旧 4 类语义不变。

| CardType 值 | 含义 | 进入手牌？ | 进入 Run 卡组？ | 占能量？ | 引入版本 |
|---|---|---|---|---|---|
| `unit` | 单位卡（塔/士兵/中立机关等） | ✅ | ✅ | ✅ 出场时占能量 | v3.0 |
| `spell` | 法术卡 | ✅ | ✅ | ✅ 释放时占能量 | v3.0 |
| `trap` | 陷阱卡 | ✅ | ✅ | ✅ 部署时占能量 | v3.0 |
| `production` | 生产建筑卡 | ✅ | ✅ | ✅ 建造时占能量 | v3.0 |
| `shop_item` | **商店商品卡**（v3.4 新增） | ❌ | ❌ | ❌ 商店内点击/拖拽消耗金币 | **v3.4** |

### 3.2 shop_item 与 unit / spell 卡的本质区别

| 维度 | shop_item | unit / spell / trap / production |
|---|---|---|
| 出现位置 | 仅商店橱窗 | 卡池 / 手牌 / 战场 |
| 持有期 | 一次性 —— 购买瞬间结算后从橱窗消失（变"售罄"占位） | 持续 —— 进入 Run 卡组后整 Run 可反复抽到 |
| 货币 | 金币 | 能量（战斗内出卡） |
| 效果时机 | **立即结算**（含消耗品立即生效 / 单位卡立即加入 Run 卡组） | 进入手牌 → 玩家主动出 |
| 是否可在战斗中出现 | 否 | 是 |
| 配置归属 | `src/config/shop/*.yaml`（v3.4 待建） | `src/config/cards/*.yaml`（既有） |

### 3.3 shop_item 子类型枚举（5 种）

> 4 种功能卡 + 1 种"单位卡商品"。注意"单位卡商品"是 shop_item 子类，外观是单位卡但**身份**是 shop_item。

| `shop_item.kind` | 含义 | 购买即时效果 |
|---|---|---|
| `buy_unit_card` | 单位卡商品（前 4 槽专属） | 立即将对应单位卡加入 Run 卡组（不进手牌，下次抽牌可抽到） |
| `restore_crystal_hp` | 水晶恢复卡 | 立即按 50% 当前已损失 HP 恢复（公式 §5.1） |
| `recycle_card` | 卡牌回收卡 | 进入"待用消耗品"状态 —— 玩家在商店内拖拽到左栏卡池中任意 Run 卡组卡 → 该卡从 Run 卡组永久删除 |
| `buy_skill_point` | 技能点卡（1 点装） | 立即 `skillPoints += 1` |
| `buy_skill_point_pack` | 技能点限量包（5 点装，折扣价） | 立即 `skillPoints += 5` |

> **不变式**：shop_item.kind 全闭集，本文档严格定义。新增 kind 必须修订本节并升级 version。

## 4. 前 4 槽固定结构：单位卡保底

### 4.1 槽位与保底机制

每次进入商店（含关后首次进入 + 刷新后），前 4 槽**强制**生成 4 张不同的单位卡：

| 槽 | 稀有度保底规则 |
|---|---|
| 槽 1 | Common 稀有度（保底高频出现） |
| 槽 2 | Common / Uncommon 加权（偏向 Common） |
| 槽 3 | Uncommon / Rare 加权（偏向 Uncommon） |
| 槽 4 | Rare / Epic / Legendary 加权（偏向 Rare，关数递增提升 Epic+ 权重） |

> 具体权重数值表 → 待 [50-mda §18.4](../50-data-numerical/50-mda.md) v3.4 数值表。本节只定保底原则与字段语义。

### 4.2 选卡池规则

| 约束 | 规则 |
|---|---|
| **去重** | 槽 1-4 之间 4 张卡 ID 互不相同 |
| **Run 卡组去重** | 4 槽中不出现"已在当前 Run 卡组中"的卡（v3.4 卡组上限 12 张，避免买无用重复） |
| **类型分布** | 不强制 4 槽全是塔卡，允许塔/士兵/陷阱/生产混排（按全卡池权重） |
| **Run 进度修正** | 关数越靠后，槽 4 出现 Epic/Legendary 概率越高（关 1 仅 5% Epic，关 8 可达 40% Epic + 10% Legendary） |

### 4.3 购买行为

- **操作**：点击单位卡商品 → 弹出确认浮窗（"花费 N 金币购买【箭塔】加入卡组？") → 确认 / 取消
- **生效**：确认后立即 `RunManager.gold -= price`，将单位卡加入 Run 卡组（`unitCardPurchases` 临时记录，§8.2）
- **不进手牌**：购买的单位卡 **不**直接进入当前手牌；只加入 Run 卡组，下次抽牌或下关开局抽牌时按概率进入手牌
- **槽位变化**：购买后该槽位变为"售罄"占位卡（§7.2）—— 直到玩家刷新才重新填充

### 4.4 价格机制

> 价格随稀有度阶梯递增，具体数值 → 待 [50-mda §18.5](../50-data-numerical/50-mda.md)。本文档约束：

| 稀有度 | 价格区间（占位，待 50-mda 锁定） |

## 5. 后 4 槽随机池：4 种功能卡

### 5.1 水晶恢复卡（`restore_crystal_hp`）

| 字段 | 值 |
|---|---|
| 唯一 ID | `shop_item.restore_crystal_hp` |
| 视觉 | 蓝色卡框 + 水晶图标 + 红十字 |
| 标题 | "水晶恢复" |
| 描述 | "恢复 50% 已损失水晶 HP" |
| 价格 | 80-150 G（占位，→ 50-mda §18） |
| 效果公式 | `recoverAmount = floor((crystalMaxHP - crystalCurHP) × 0.5)` |
| 上限 | 不超过 `crystalMaxHP - crystalCurHP`（即不可超过满血） |
| 满血时 | 商品仍出现，但点击触发"满血无需治疗"红字提示，不消耗金币 |
| 购买行为 | 点击 → 确认浮窗 → 确认即 `crystalCurHP += recoverAmount`，槽位变"售罄" |

> 设计意图：自带反通胀 —— 高损血时回得多、低损血时回得少，避免无脑买。

### 5.2 卡牌回收卡（`recycle_card`）

| 字段 | 值 |
|---|---|
| 唯一 ID | `shop_item.recycle_card` |
| 视觉 | 灰色卡框 + 垃圾桶图标 |
| 标题 | "卡牌回收" |
| 描述 | "拖到左栏卡池任意 Run 卡组卡上 → 永久删除该卡（本 Run 内）" |
| 价格 | 40-60 G（占位） |
| 效果时机 | 购买为**两阶段**：①点击卡片 → 扣除金币，进入"待回收"状态；②拖拽到左栏某张 Run 卡组卡 → 完成回收，该卡从 Run 卡组永久删除 |
| 目标限制 | 仅可回收**当前 Run 卡组**中的卡，不可回收：①未拥有的卡（卡池中的灰色版） ②已被本卡组永久删除的卡 |
| 取消机制 | 进入"待回收"状态后，玩家点击 ESC 或商店空白处 → 取消，但 **金币不退还**（明确告知玩家"购买即扣费"） |
| 撤销 | 完成回收后**不可撤销** |
| Run 卡组 ≤ 1 张时 | 商品仍出现但点击触发"Run 卡组不足以删除"红字提示，不消耗金币 |

> 设计意图：精简卡组 + 提高抽卡质量。v3.4 卡组上限 12 张，删 1 张相当于"为下次抽到强卡腾位"。

### 5.3 技能点卡（`buy_skill_point`）

| 字段 | 值 |
|---|---|
| 唯一 ID | `shop_item.buy_skill_point` |
| 视觉 | 紫色卡框 + 星形图标（✦） |
| 标题 | "技能点 ×1" |
| 描述 | "获得 1 个技能点，可在卡池界面分配到任意单位的技能树" |
| 价格 | 60-100 G（占位） |
| 购买行为 | 点击 → 确认浮窗 → `RunManager.skillPoints += 1`，顶栏数字 +1 动画，槽位变"售罄" |
| 数量限制 | 单次商店内最多购买 N 次（n 待 50-mda 锁定，建议 ≤3 次避免单店技能点垄断） |

### 5.4 技能点限量包（`buy_skill_point_pack`）

| 字段 | 值 |
|---|---|
| 唯一 ID | `shop_item.buy_skill_point_pack` |
| 视觉 | 金色卡框 + 5 颗星形图标 + "Limited" 角标 |
| 标题 | "技能点限量包 ×5" |
| 描述 | "获得 5 个技能点（折扣 30% 单价）" |
| 价格 | 350 G（占位）≈ 5 × 100 G × 0.7 |
| 购买行为 | 同单装技能点卡，一次性 +5 |
| 数量限制 | 单次商店内**最多 1 件**（极稀有，按后 4 槽随机权重出 ~5%） |

### 5.5 4 种功能卡随机出现权重框架

> 后 4 槽中 4 种功能卡的出现概率，由权重表决定。本文档定**权重维度**，具体数值 → 50-mda。

| 功能卡 | 出现权重维度 | 权重影响因素 |
|---|---|---|
| 水晶恢复卡 | 基础权重 | 玩家水晶损失越多权重越高（自适应） |
| 卡牌回收卡 | 基础权重 | Run 卡组越大权重越高（≥ 10 张时显著提升） |
| 技能点卡 | 基础权重 | 关数越后权重越高（鼓励后期堆点) |
| 技能点限量包 | 极低权重 | 仅 4-8 关出现，且单次商店 ≤ 1 件 |

**保证项（约束）**：
- 后 4 槽**至少 1 槽**是非"售罄"状态的实际商品（避免后 4 槽全空导致玩家无可买）
- 任一功能卡**不重复出现**于同一商店刷新内（4 种各占至多 1 槽）

## 6. 交互详设

### 6.1 单位卡商品 —— 点击购买

```
[玩家点击单位卡] 
   ↓
[确认浮窗：花费 N 金币购买【箭塔】加入卡组？  取消/确认]
   ↓ (确认)
[扣除金币 + 加入 Run 卡组 + 槽位变"售罄"]
   ↓
[顶栏金币 -N 动画 + 左栏卡池"箭塔"从灰色 → 主题色]
```

### 6.2 水晶恢复卡 —— 点击立即生效

```
[玩家点击水晶恢复卡]
   ↓
[计算 recoverAmount]
   ↓
[recoverAmount == 0 → 显示"满血无需治疗"红字 2s 淡出，不扣金币]
[recoverAmount > 0 → 确认浮窗"恢复 X HP，花费 N 金币？" → 确认 → 扣金币 + crystalCurHP += X + 槽位售罄]
```

### 6.3 卡牌回收卡 —— 两阶段拖拽

```
[玩家点击卡牌回收卡]
   ↓
[Run 卡组 ≤ 1 张 → 显示"Run 卡组不足以删除"红字，不扣金币]
[Run 卡组 ≥ 2 张 → 扣金币 + 商品卡进入"持卡待用"状态，鼠标变垃圾桶图标]
   ↓
[玩家拖拽到左栏卡池任意 Run 卡组卡上]
   ↓
[确认浮窗"永久删除【箭塔】？此操作不可撤销"]
   ↓
[确认 → 从 Run 卡组删除目标卡 + "持卡待用"状态结束 + 槽位变售罄]
[取消 → "持卡待用"继续，玩家可重选目标 或 ESC 放弃（金币不退）]
```

**拖拽视觉**：
- "持卡待用"时商品卡跟随鼠标移动（半透明 + 红色边框）
- 悬停在合法目标卡（Run 卡组卡） → 目标卡红色高亮 + "永久删除"提示
- 悬停在非法目标（卡池未持有卡 / 空白区） → 鼠标变禁止图标

### 6.4 技能点卡 / 限量包 —— 点击立即生效

```
[玩家点击]
   ↓
[确认浮窗"花费 N 金币获得 ✦1/✦5？" → 确认 → 扣金币 + skillPoints += N + 顶栏 ✦ 数字飘字动画 + 槽位售罄]
```

### 6.5 错误反馈统一规范

| 错误场景 | 反馈方式 |
|---|---|
| 金币不足 | 商品卡价格标签红色高亮 + 点击时震动 0.3s + 红字"金币不足" 2s 淡出 |
| 满血时点恢复 | 红字"水晶已满血" 2s 淡出 |
| 卡组不足 1 张点回收 | 红字"卡组不足以删除" 2s 淡出 |
| 技能点卡超过单店上限 | 红字"本店已购满" 2s 淡出 |
| 拖拽回收到非法目标 | 鼠标禁止图标 + 释放后回弹 + 红字"无法回收此卡" 1s |

### 6.6 事务不可撤销原则

| 操作 | 是否可撤销 |
|---|---|
| 购买单位卡 | ❌ 一旦加入 Run 卡组不可移除（除非再买回收卡） |
| 购买水晶恢复 | ❌ HP 加上去不可退 |
| 购买技能点卡 | ❌ 技能点加上去不可退（但技能点本身可不分配到节点） |
| 购买回收卡（一阶段扣费） | ❌ 金币不退还，即便玩家 ESC 取消第二阶段 |
| 完成回收（二阶段） | ❌ 删除的卡不可恢复 |
| 关闭商店离开 | 不需"撤销"概念——离开前所有购买已结算 |

## 7. 刷新与售罄

### 7.1 刷新机制

| 字段 | 值 |
|---|---|
| 触发 | 玩家点击橱窗底栏"🔁 刷新 ●30 G"按钮 |
| 价格 | 30 G（与 v3.0 一致，→ 待 [50-mda §18](../50-data-numerical/50-mda.md) v3.4 校准） |
| 价格递增 | 同一店内连续刷新价格 +10 G 递增（30 → 40 → 50 ...），离开商店后重置 |
| 刷新范围 | 重新生成 8 槽全部商品（含已售槽位的"售罄"占位卡，刷新后全部重置） |
| 金币不足 | 按钮置灰 + 点击红字"金币不足" |

### 7.2 售罄空槽机制

| 状态 | 视觉 | 可交互 |
|---|---|---|
| 商品在售 | 正常商品卡 + 价格标签 | ✅ 可点击购买 / 可拖拽（仅回收卡） |
| 已售罄 | 灰色卡框 + "SOLD" 中央水印 + 价格标签暗淡 | ❌ 鼠标悬停显示"已售罄，可刷新重抽" |

### 7.3 ESC 退出商店

| 玩家位置 | ESC 行为 |
|---|---|
| 商店主界面 | 等价点击"离开商店"，触发 [47-level-map-ui §5.3](./47-level-map-ui.md#53-关闭返回流程本文档权威) 关闭返回流程 |
| 商店内确认浮窗（购买确认） | 等价点击"取消"，关闭浮窗 |
| 商店内"持卡待用"状态（回收卡拖拽中） | 取消拖拽，回到主界面（金币不退） |

### 7.4 离开商店前置约束

| 约束 | 规则 |
|---|---|
| 是否允许带"持卡待用"状态离开商店 | ❌ 不允许；点击"离开"时若处于持卡待用，弹"未完成回收，离开将作废此次购买且不退金币 → 确认/取消" |
| 是否允许商品全售完时强制退出 | ✅ 允许，相当于刷新无用直接离开 |
| 离开后是否可重进商店 | ❌ 本关后只能进 1 次商店，离开即结束（同 [10-roguelike-loop §3](../10-gameplay/10-roguelike-loop.md#3-关间节点系统核心-rogue-抉择替代波间-3-选-1) 节点不可重入原则） |

## 8. 与 RunManager 状态机锁定

### 8.1 RunManager 字段变更（**v3.4 新增**）

| 字段 | 类型 | 初值 | 修改入口 | 持久化 |
|---|---|---|---|---|
| `skillPoints` | `number` | `0`（initRun 时设置） | `addSkillPoints(n)` / `spendSkillPoints(n, unitCardId, nodeId)` 新增方法 | Run 结束清零 |
| `unitCardPurchases` | `Set<string>`（卡 ID） | `new Set()` | 商店买单位卡时 `add(cardId)`；卡组合并到 deck 时清除（即时同步） | Run 结束清零 |
| `crystalCurHP` | （**重命名既有 `crystalHp`** → `crystalCurHP`） | 同既有 | 同既有 + `restoreCrystalHpPercent(0.5)` 新方法 | 同既有 |

> ⚠️ **本文档对 RunManager 的拓展约束**：只新增 2 个字段 + 3 个方法，**不**改动既有 API。`crystalHp` → `crystalCurHP` 重命名属于 v3.4 形态级清理，由 [v3.4-MAJOR-MIGRATION §3.2](../v3.4-MAJOR-MIGRATION.md) 第二轮处理；本文档先保留 `crystalHp` 名字、新增方法 `restoreCrystalHpPercent` 兼容。

### 8.2 关键方法签名（接口骨架，TS 实际签名以代码为准）

```typescript
class RunManager {
  // === v3.4 新增 ===
  skillPoints: number;                         // 本 Run 技能点余额
  unitCardPurchases: Set<string>;              // 本店已购单位卡（防重复）

  addSkillPoints(amount: number): void;        // 商店买技能点卡调用
  spendSkillPoints(                            // 卡池界面技能树点节点调用
    amount: number,
    unitCardId: string,
    nodeId: string,
  ): boolean;                                  // 返回是否成功（余额不足返回 false）

  restoreCrystalHpPercent(percent: number): number;  // 商店买水晶恢复卡调用，返回实际恢复量
  buyUnitCard(cardId: string): boolean;        // 商店买单位卡调用，返回是否成功
  recycleUnitCard(cardId: string): boolean;    // 商店用回收卡删除单位卡调用，返回是否成功

  // === Run 结束清零 ===
  resetRun(): void;                            // 既有 + 增加清 skillPoints / unitCardPurchases
}
```

### 8.3 商店事件接口（UISystem 与 RunManager 协议）

| 事件 | UISystem 触发条件 | RunManager 响应 |
|---|---|---|
| `shop:enter` | 玩家在 3 选 1 面板选商店 | 生成 8 槽商品（按 §4 / §5 规则）+ 切换商店界面 |
| `shop:buy_unit_card` | 玩家确认购买单位卡 | `buyUnitCard(cardId)` + 槽位售罄 |
| `shop:buy_restore` | 玩家确认购买水晶恢复 | `restoreCrystalHpPercent(0.5)` + 槽位售罄 |
| `shop:recycle_start` | 玩家点击回收卡支付金币 | 扣金币 + 进入"持卡待用"UI 状态 |
| `shop:recycle_confirm` | 玩家在拖拽目标上确认删除 | `recycleUnitCard(targetCardId)` + 槽位售罄 |
| `shop:buy_skill_point` | 玩家确认购买技能点卡 | `addSkillPoints(1 or 5)` + 槽位售罄 |
| `shop:refresh` | 玩家点刷新按钮 | 扣金币 + 重新生成 8 槽 |
| `shop:leave` | 玩家点离开 / ESC | 触发 [47 §5.3](./47-level-map-ui.md#53-关闭返回流程本文档权威) 关闭返回流程 → 节点 +1 |

### 8.4 不变式（RunManager 锁死）

- ✅ 任何商店事务后 `crystalCurHP <= crystalMaxHP`
- ✅ 任何商店事务后 `gold >= 0` 且 `skillPoints >= 0`
- ✅ Run 卡组在 `recycleUnitCard` 后 `size >= 1`（不允许删空）
- ✅ Run 卡组在 `buyUnitCard` 后 `size <= 卡组上限`（卡组上限到 12 时禁购买，相关单位卡商品置灰）
- ❌ **绝对禁止**：商店事务过程中触发关卡战斗逻辑或 GamePhase 变化
