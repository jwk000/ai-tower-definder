# 01 — 游戏概述

> 产品定位、目标用户、核心玩法

---

## 1. 产品定位

**一句话**: "可以亲手操控战场的塔防游戏" — 传统TD框架下，加入可移动单位操控、资源链经济和实时技能释放。

### 市场差异化

| 对比 | 传统TD | Tower Defender |
|------|--------|---------------|
| 核心体验 | 布阵 + 观战 | 布阵 + 实时操作 + 经济链 + 动态对抗 |
| 操作密度 | 几乎为零（摆完等结果） | 持续有事做（调阵型、拦截、集火BOSS） |
| 策略深度 | 固定解（一图一最优布阵） | 动态解（敌方AI反制，迫使调整） |

### 参考竞品

- Kingdom Rush — 精致关卡标杆，但布阵后被动等待
- 明日方舟 — 塔防+RPG融合，门槛偏高
- 皇室战争 — 实时对战，操作密度过大
- Vampire Survivors — 爽快但深度不足

Tower Defender 定位为以上四者之间的"**轻策略操作**"缺口。

---

## 2. 目标用户

| 用户层 | 占比 | 特征 |
|--------|------|------|
| **核心用户** | 70% | 25-35岁男性，策略游戏经验，单次15-30分钟碎片时间，追求"有脑但不上头" |
| **次要用户** | 20% | 18-24岁学生，几何美术风格吸引，喜欢研究最优解/追求高分 |
| **边缘用户** | 10% | 轻度休闲玩家，需要低门槛入门引导 |

## 3. 核心玩法闭环

### 3.1 单局循环

```
波次开始
  │
  ├→ 部署/调整阶段（建造、升级、修复塔；部署/移动单位；释放技能；分配资源）
  │    │ 玩家触发"开始波次"（或自动倒计时）
  │    ▼
  ├→ 战斗阶段（敌人沿路径移动；塔自动攻击发射弹道；玩家操控移动单位、释放技能）
  │    │ 所有敌人死亡 / 基地被摧毁
  │    ▼
  ├→ 波次结算（剩余血量奖励金币；下一波敌人预览；资源产出结算）
  │    │
  │    ├→ 不是最后一波 → 回到部署阶段
  │    └→ 最后一波 → 关卡结算
  │
  └→ 关卡结算（星级评定1-3星；解锁新内容；排行榜记录）
```

### 3.2 游戏状态机

```
Deployment → Battle → WaveBreak → (循环) → Victory / Defeat
```

- **Deployment**: 建造/升级/调整阵型
- **Battle**: 敌人行军，塔自动攻击，玩家操控移动单位
- **WaveBreak**: 当前波次结束，结算奖励，显示下一波预览
- **Victory**: 消灭所有波次敌人，基地未被摧毁
- **Defeat**: 基地血量归零（敌人到达终点或被远程摧毁）

### 3.3 三大创新支柱

1. **可移动单位操控** — 拖拽移动、拦截敌人、技能释放，解决"摆好后无事可做"
2. **资源链经济系统** — 三种资源 + 生产建筑链 + 开局运营决策
3. **敌方反制AI** — 敌人可攻击建筑、适应性反制玩家策略、Boss阶段转换
