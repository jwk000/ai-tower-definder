# 07 — 地图与关卡系统

> 网格地图、关卡配置、波次系统、随机初始化

---

## 1. 网格地图

### 1.1 尺寸规格
- 设计分辨率: **1920×1080**（横屏16:9）
- 场景区域: **21×9格**（1344×576px），**水平垂直居中显示**（水平offsetX=288，垂直offsetY动态计算=HUD之后居中于面板之上）
- Tile大小: **64×64px**
- 背景层: **全屏 1920×1080**，覆盖整个画布（天空渐变+远景+云+天气特效）
- 底部操作面板: **1344×100px**，与地图同宽，水平居中

### 1.2 Tile类型

| 类型 | 说明 | 行为 |
|------|------|------|
| `empty` | 空地 | 可建塔（需在路径邻格） |
| `path` | 路径 | 敌人行走，不可建造 |
| `blocked` | 障碍 | 不可通行不可建造 |
| `spawn` | 出生点 | 敌人生成位置 |
| `base` | 基地 | 敌人终点 |

### 1.3 建造限制
- 仅可在**路径的8方向相邻1格内**的空地上建造
- 可建地块视觉高亮（浅绿+边框）

## 2. 关卡配置

### 2.1 关卡结构

每个关卡是一个声明式配置，包含：

| 配置项 | 说明 |
|--------|------|
| 地图数据 | 21×9格 tile 布局 + 路径坐标 + 出生点/基地 |
| 波次配置 | 固定波次数量 + 每波敌人编组 + 特殊规则 |
| 起始资源 | 金币、能量、人口上限、基地HP |
| 可用单位 | 本关允许使用的塔类型和单位类型 |
| 解锁条件 | 需要通过的前置关卡 |
| 随机池 | 天气池、中立单位池（可随机部分） |
| 主题配色 | 地形颜色方案 |
| 装饰物 | 场景障碍物（纯视觉） |

### 2.2 随机初始化机制

关卡初始化时，以下内容可从配置池中随机抽取：

| 随机项 | 池来源 | 说明 |
|--------|--------|------|
| **天气** | `weatherPool` | 从允许的天气类型中随机初始天气 |
| **禁用塔** | `banPool` | 从塔类型池中随机禁用 0-N 种塔，增加策略变数 |
| **中立单位** | `neutralPool` | 从允许的中立类型中随机放置位置和数量 |
| **装饰物** | `decorationPool` | 随机散布主题装饰（树/岩石等） |
| **路径变体** | `pathVariants` | 从多个预设路径中随机选择（增加重玩性） |

```yaml
# 关卡随机配置示例
level_01:
  fixed:
    map: plains_zigzag
    waves: l1_waves
    startingGold: 200
    
  random:
    weatherPool: [Sunny, Rain]
    weatherInit: random_from_pool      # 初始天气随机
    
    banPool: ~                         # L1不禁用任何塔
    
    neutralPool:
      - type: gold_chest
        count: [1, 3]                  # 1-3个随机
        positions: random_empty_tiles  # 随机空地块
      - type: healing_spring
        count: [0, 1]                  # 0或1个
        positions: random_empty_tiles
```

### 2.3 随机种子与可复现性（P2-#16 修复 v1.1）

> **设计目标**：随机初始化必须可复现，便于（1）测试 bug 复现；（2）玩家分享挑战种子；（3）速通竞速。

#### 2.3.1 种子来源

| 模式 | 种子规则 |
|------|----------|
| **关卡模式（首次进入）** | `seed = hash(levelId + difficultyMultiplier + Date.now())`，写入当前 run 的元数据 |
| **关卡模式（重玩）** | UI 提供 "新种子" / "上次种子" 选项；默认新种子 |
| **无尽模式** | 与关卡同；存档保留 `lastSeed` 和 `bestSeed` |
| **每日挑战**（未来扩展） | `seed = hash(date + levelId)`，全球同种子 |

#### 2.3.2 PRNG 实现要求

- 必须使用**确定性 PRNG**（mulberry32 / xoshiro128++ / pcg32）
- **禁止使用** `Math.random()` 在游戏逻辑中
- PRNG 状态保存在 Game 实例中，不能全局共享（不同模块独立流）
- Replay 系统（未来）只需保存 `seed` + 玩家操作序列，可完全复现

```typescript
// 实现示例
class GameRandom {
  private state: number;
  constructor(seed: number) {
    this.state = seed | 0;  // 确保 32-bit 整数
  }
  next(): number {                       // [0, 1)
    let t = (this.state += 0x6D2B79F5) | 0;
    t = Math.imul(t ^ (t >>> 15), t | 1);
    t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
  }
  nextInt(min: number, max: number): number {  // [min, max)
    return Math.floor(this.next() * (max - min)) + min;
  }
  pick<T>(arr: T[]): T {
    return arr[this.nextInt(0, arr.length)];
  }
}
```

#### 2.3.3 多流隔离

| 流 | 用途 | 种子派生 |
|---|------|---------|
| `mapRandom` | 地图/天气/禁用塔/中立单位 | `seed` |
| `waveRandom` | 波次内敌人选择/特殊规则触发 | `seed ^ 0xWAVE` |
| `dropRandom` | 击杀掉落/宝箱 | `seed ^ 0xDROP` |
| `decorRandom` | 装饰物位置（纯视觉，可与逻辑分流） | `seed ^ 0xDECO` |

> **理由**：玩家行为不应影响"未来同种子的同一波次敌人配置"。

## 3. 5个关卡

| # | 名称 | 主题 | 波数 | 起始金币 | 可用塔 | 解锁 | 特征 |
|---|------|------|------|----------|--------|------|------|
| 1 | 平原 | 草原 | 10 | 200 | 箭塔 | 默认 | 教程友好 |
| 2 | 沙漠 | 沙地 | 12 | 250 | 箭、炮 | 通L1 | 分叉路 |
| 3 | 冰原 | 冰雪 | 15 | 300 | 箭、冰 | 通L2 | 双spawn |
| 4 | 火山 | 熔岩 | 15 | 350 | 箭、炮、电、导弹 | 通L3 | 岩浆阻挡 + **飞行敌** |
| 5 | 城堡 | 石堡 | 15 | 400 | 全部 | 通L4 | 迷宫+Boss终关 + 飞行敌常驻 |

### 详细描述

#### L1 平原
- 场景: 青翠草原，土路延伸，哥布林出没
- 路径: 38格Z形，弯拐处是理想建塔点
- 禁用池: 空（不禁用）
- 天气池: [Sunny, Rain]

#### L2 沙漠
- 场景: 古代商路被黄沙半掩，沙漠强盗设伏
- 路径: 48格主路径 + 18格分叉捷径
- 禁用池: 空（不禁用）
- 天气池: [Sunny, Fog, Night]

#### L3 冰原
- 场景: 永恒冰原，双线进攻
- 路径: 双spawn汇入主路径
- 禁用池: 空（不禁用）
- 天气池: [Snow, Fog, Night]
- 特殊: 路径上永久减速15%

#### L4 火山
- 场景: 岩浆河切割大地
- 路径: 被大量blocked包围的窄路径
- 禁用池: [Ice]（可能随机禁用冰塔）
- 天气池: [Sunny, Rain, Fog]
- 特殊: 所有敌人 +15 护甲
- **v1.1 新增**: 第 4 波起引入**飞行敌**（蝙蝠/小妖怪）——制衡导弹塔的地面爆炸，强制玩家搭配反空打击塔

#### L5 城堡
- 场景: 远古要塞内部，石柱迷宫
- 路径: 折叠迷宫，~55格
- 禁用池: [Bat]（可能禁用蝙蝠塔）
- 天气池: [Rain, Night, Fog]
- 特殊: 第 10 波禁止建造
- **v1.1 新增**: 飞行敌持续出现（每波约 20-30% 概率），玩家必须长期保持反空能力

#### L6+ / 无尽模式（v1.1 新增）
- **分散敌**: L6 第 1 波起（或无尽模式第 30 波起）作为"特种敌"出现
- 分散敌移动中周期性散开 + 聚拢，降低导弹塔的密度评分——让"等敌人密集再开炮"的策略失效
- 同时混合飞行敌 + 地面普通敌 + Boss，形成对各种塔流的均衡考验

## 4. 波次配置

### 4.1 波次数据

每波包含:
- 波次序号
- 敌人编组列表（类型 + 数量 + 生成间隔）
- 波次开始延迟（秒）
- Boss波标记
- 特殊规则（可选）
- 出生点索引（多spawn关卡）
- 指定天气（可选，覆盖随机）

### 4.2 特殊规则

| 规则 | 效果 |
|------|------|
| `double_speed` | 本波敌人移速 +100% |
| `fortified` | 本波敌人HP +50% |
| `rush` | 敌人生成间隔减半 |
| `no_build` | 本波期间禁止建造/升级 |

### 4.3 波次流程

1. 部署结束 → 自动倒计时（开局5s，波间3s）→ 可手动跳过
2. 波次开始 → 按编组依次生成敌人
3. 全部敌人死亡 → 波次清除 → 进入波间期
4. 最后一波清除 → 关卡胜利结算

### 4.4 敌人层级（Layer）

> 详见 [18-图层系统](./18-layer-system.md)。本节仅描述与塔交互的层级语义。

| 层级 | 说明 | 可被攻击规则 |
|------|------|------------|
| `Ground` | 地面敌人（哥布林/重装/法师/快兵/Boss 等） | 所有塔都可命中 |
| `Flying` | 飞行敌人（详见 §4.5） | 仅"反空塔"和"全层级塔"可命中；**导弹塔的地面爆炸不命中** |
| `Burrowing` | 钻地敌人（如有，预留） | 仅特殊探测塔可命中 |

> **导弹塔特殊规则**: 导弹的爆炸是地面爆炸——`Flying` 层级敌人不在爆炸范围内（即使坐标重叠也不命中）。这是 L4+ 飞行敌制衡导弹塔流的核心机制。

### 4.5 L4 飞行敌 和 L6 分散敌（v1.1 新增）

为制衡导弹塔的大范围 AOE，引入两类抗性敌人：

#### Flying（飞行敌）— L4 引入

| 属性 | 值 |
|------|-----|
| 层级 | `Flying` |
| 视觉 | 蝙蝠/小妖怪形象，y 轴有 ±8px 振翅浮动 |
| 移动 | 沿路径前进，但**不受地面 AOE 影响**（导弹爆炸、自爆虫、地面减速带均不命中） |
| 可被攻击塔 | 箭塔/冰塔/电塔/激光塔/蝙蝠塔（均可命中飞行）；**导弹塔不可命中**（地面爆炸） |
| 出现波次 | L4 第 4 波起以"小群队 3-5 个"夹在地面波次中出现 |
| 设计意图 | 让玩家学会"反空打击"，并强制搭配非导弹塔 |

#### Splitting（分散敌）— L6 引入（如有 L6+ 关卡或无尽模式）

| 属性 | 值 |
|------|-----|
| 层级 | `Ground` |
| 视觉 | 多触手/触须，移动中**周期性散开 + 聚拢** |
| 行为 | 每 1.5s 在路径垂直方向 ±20~30px 间随机偏移（视为"路径扰动"），导致同波分散敌不会形成密集队形 |
| 设计意图 | **降低导弹塔密度因子评分**——分散敌不会站成一堆，所以一发导弹平均只能命中 1~2 个分散敌（vs 普通敌一发命中 3-5 个） |
| 数值影响 | 不影响 HP/ATK 等基础属性，仅影响"密度评分" |
| 出现波次 | L6 第 1 波起作为"特种敌"出现，无尽模式 30 波后稳定出现 |

> 这两类敌人在 [21-MDA §6（敌人数值）](./21-mda-numerical-design.md) 中定义具体数值。

## 6. 场景装饰

每个关卡有主题障碍物（纯视觉，不阻塞路径）：

| 主题 | 装饰物类型 |
|------|-----------|
| 草原 | 树、灌木、花丛 |
| 沙漠 | 岩石、仙人掌、枯骨 |
| 冰原 | 冰晶、雪松、冰封岩石 |
| 火山 | 熔岩裂缝、烧焦枯树、火山岩 |
| 城堡 | 石柱、火炬台、碎石堆 |

装饰物散布在空地块上，随机摆放，不影响游戏逻辑。
