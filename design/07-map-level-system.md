# 07 — 地图与关卡系统

> 网格地图、8 关 + 终战、波次系统、关间节点、随机初始化
>
> **v3.0 重写**：根据 [25-card-roguelike-refactor](./25-card-roguelike-refactor.md) 方案，关卡结构改为 Roguelike 长征模式（连闯 8 关 + 终战 Boss）。

---

## 1. 网格地图

### 1.1 尺寸规格
- 设计分辨率: **1920×1080**（横屏 16:9）
- 场景区域: **21×9 格**（1344×576 px），**水平垂直居中显示**（水平 offsetX=288，垂直 offsetY 动态计算）
- Tile 大小: **64×64 px**
- 背景层: **全屏 1920×1080**，覆盖整个画布（天空渐变 + 远景 + 云 + 天气特效）
- **底部手牌区**: **1344×140 px**，与地图同宽，水平居中（替代旧的工具栏）

### 1.2 Tile 类型

| 类型 | 说明 | 行为 |
|------|------|------|
| `empty` | 空地 | 可拖卡部署单位/建筑（需在路径邻格） |
| `path` | 路径 | 敌人行走，不可部署 |
| `blocked` | 障碍 | 不可通行不可部署 |
| `spawn` | 敌人出生点 | 位于地图右侧 |
| `base` | **大本营** | 位于地图左侧，HP 跨关卡继承 |

### 1.3 大本营与生成口（v3.0 关键变更）

- **大本营固定在地图左侧**（玩家保卫的核心，HP 全局继承）
- **敌方生成口固定在地图右侧**（部分关卡可有多个生成口）
- 敌人从右向左行进
- 大本营 HP 见 [13-save-system §4](./13-save-system.md)，跨关卡继承

### 1.4 建造限制
- 仅可在**路径的 8 方向相邻 1 格内**的空地上部署（沿用旧规则）
- 合法格位视觉高亮（浅绿 + 边框）
- 拖卡牌到地图时实时高亮可放置格

## 2. 关卡配置

### 2.1 关卡结构

每个关卡是一个声明式配置，包含：

| 配置项 | 说明 |
|--------|------|
| 地图数据 | 21×9 格 tile 布局 + 路径坐标 + 出生点 + 大本营位置 |
| 波次配置 | 固定波次数量（8-10 波）+ 每波敌人编组 + 关底 Boss |
| 难度乘数 | `enemyHpMult` / `enemyDmgMult` / `enemySpeedMult` / `goldRewardMult`（叠加到 21-MDA 敌人基础数值） |
| 主题配色 | 地形颜色方案（9 种主题） |
| 装饰物 | 场景障碍物（纯视觉） |
| 随机池 | 天气池、装饰物池、波次扰动池 |
| 关间节点开关 | `hasInterLevelNode`（1-7 = true，8 = false 直接接终战，9 终战无后续） |

### 2.2 随机初始化机制

关卡初始化时，以下内容可从配置池中随机抽取：

| 随机项 | 池来源 | 说明 |
|--------|--------|------|
| **天气** | `weatherPool` | 从允许的天气类型中随机初始天气 |
| **装饰物** | `decorationPool` | 随机散布主题装饰（树/岩石等） |
| **波次微扰动** | `waveVariantPool` | 同关卡多次 Run 时，波次内敌人比例小幅变化（10-20%）|
| **路径变体** | `pathVariants` | 从多个预设路径中随机选择（部分关卡） |

> **v3.0 删除**：禁用塔池（`banPool`）。原因是塔已变成卡牌，开局抽卡组天然提供了"禁用感"，无需额外机制。

```yaml
# 关卡随机配置示例
level_01:
  fixed:
    id: 1
    theme: plains
    waveCount: 8
    hasMidEliteWave: false
    endBoss: orc_chieftain
    hasInterLevelNode: true

    # 难度乘数（关 1 = 基准 1.0）
    enemyHpMult: 0.8
    enemyDmgMult: 0.8
    enemySpeedMult: 1.0
    goldRewardMult: 1.0

  random:
    weatherPool: [Sunny, Rain]
    weatherInit: random_from_pool
    decorationPool: [tree, bush, flower]
    waveVariantPool: [standard, more_speeders, more_tanks]
```

### 2.3 随机种子与可复现性

> 沿用旧 §2.3 设计，但作用范围扩大到整局 Run。

#### 2.3.1 种子来源

| 模式 | 种子规则 |
|------|----------|
| **新 Run** | `seed = hash(timestamp + randomBytes)`，写入当前 Run 的元数据 |
| **继续上次 Run**（关间存档点） | 从存档读取 `runSeed`，继续派生 |
| **挑战种子**（未来扩展） | 玩家手动输入种子开 Run |
| **每日挑战**（未来扩展） | `seed = hash(date)`，全球同种子 |

#### 2.3.2 PRNG 实现要求

- 必须使用**确定性 PRNG**（mulberry32 / xoshiro128++ / pcg32）
- **禁止使用** `Math.random()` 在游戏逻辑中
- PRNG 状态保存在 Run 实例中，关间存档点保存 PRNG 状态
- Replay 系统（未来）只需保存 `seed` + 玩家操作序列

```typescript
class RunRandom {
  private state: number;
  constructor(seed: number) {
    this.state = seed | 0;
  }
  next(): number { /* mulberry32 */ }
  nextInt(min: number, max: number): number { /* ... */ }
  pick<T>(arr: T[]): T { /* ... */ }
  weighted<T>(items: Array<{value: T, weight: number}>): T { /* ... */ }
}
```

#### 2.3.3 多流隔离

| 流 | 用途 | 种子派生 |
|---|------|---------|
| `drafterRandom` | 开局抽 12 张卡组 | `seed ^ 0xDRAFT` |
| `mapRandom` | 地图/天气/装饰物 | `seed ^ 0xMAP` |
| `waveRandom` | 波次内敌人选择 | `seed ^ 0xWAVE` |
| `dropRandom` | 击杀掉落 | `seed ^ 0xDROP` |
| `shopRandom` | 商店商品池 | `seed ^ 0xSHOP` |
| `mysticRandom` | 秘境事件抽取 | `seed ^ 0xMYSTIC` |

### 3.1 详细描述

#### L1 平原
- 场景: 青翠草原，土路延伸
- 路径: 38 格 Z 形
- 敌人池: 哥布林、骷髅、狼
- 天气池: [Sunny, Rain]
- 设计目的: 让玩家熟悉拖卡部署 + 能量节奏

#### L2 森林
- 场景: 茂密林地，光影斑驳
- 路径: 38-42 格曲线
- 敌人池: + 快速狼骑兵
- 天气池: [Sunny, Rain, Fog]
- 设计目的: 引入快速敌人，玩家学会"远程兵远端拦截"

#### L3 沼泽
- 场景: 雾气弥漫的湿地
- 路径: 双线汇入
- 敌人池: + 治疗祭司、毒蛇
- 天气池: [Rain, Fog]
- 设计目的: 引入治疗类敌人，玩家学会"威胁度评分"机制

#### L4 雪山
- 场景: 永恒冰原
- 路径: 双 spawn 汇入主路径
- 敌人池: + **飞行敌**（蝙蝠/小妖）
- 天气池: [Snow, Fog, Night]
- 设计目的: 强制玩家在卡组中考虑反空能力

#### L5 沙漠
- 场景: 古商路被黄沙半掩
- 路径: 48 格主路径 + 18 格分叉
- 敌人池: + **分散敌**（多触手怪）
- 天气池: [Sunny, Fog, Night]
- 设计目的: 制衡 AoE 流派，让"密集爆破"失效

#### L6 废墟
- 场景: 远古城市的残骸
- 路径: 折叠走廊
- 敌人池: + **召唤师**（死亡后召唤小怪）、**护盾兵**（破盾才能伤害）
- 天气池: [Night, Fog]
- 设计目的: 引入"特殊机制敌人"，玩家必须搭配多样化卡

#### L7 熔岩
- 场景: 岩浆河切割大地
- 路径: 被 blocked 包围的窄路径
- 敌人池: + **强化自爆兽**、混合飞行敌
- 天气池: [Sunny, Rain, Fog]
- 设计目的: 资源压力最大，玩家必须有成型卡组

#### L8 王城
- 场景: 远古要塞内部，石柱迷宫
- 路径: ~55 格折叠迷宫
- 敌人池: + **隐形刺客**、**反射机甲**
- 天气池: [Rain, Night, Fog]
- 设计目的: 关 8 后直接接终战（无关间节点），玩家必须保留资源应对终战

#### L9 终战 — 深渊
- 场景: 永夜深渊，深紫色光芒
- 路径: 单条长路径，分为三段
- 敌人池: **混合所有 20 种敌人** + 深渊领主 Boss
- 波次结构: 单波超长（约 30-40 波次量的敌人），分 3 阶段
  - 阶段 1: 混合普通敌（精英频率高）
  - 阶段 2: 精英波 + 中段 mini-boss
  - 阶段 3: 深渊领主登场 + 多次召唤增援
- 天气: 永夜（特定固定天气）
- 设计目的: 最终决战的张力最大化，无关间节点接续

## 4. 波次配置

### 4.1 波次数据

每波包含:
- 波次序号
- 敌人编组列表（类型 + 数量 + 生成间隔）
- 波次开始延迟（2-3 秒过渡，无玩家手动跳过）
- Boss / 精英波标记
- 特殊规则（可选）
- 出生点索引（多 spawn 关卡）
- 指定天气（可选，覆盖随机）

### 4.2 波次流程（v3.0 简化）

```
关卡开始
   │
   ▼
波 1：手牌补满 → 战斗 → 全部敌人死亡或抵达大本营
   │ 2-3 秒过渡（无玩家手动跳过，无选项面板）
   ▼
波 2：手牌补满 → 战斗 → ...
   │
   ▼
关底 Boss 波 → Boss 击败 → 关卡通过
   │
   ▼
[关间节点：商店 vs 秘境]（仅关 1-7 之后）
   │
   ▼
下一关
```

> **v3.0 关键变更**：删除旧 §4.2 的"自动倒计时 + 手动跳过开始波次"机制。波次过渡自动进行（2-3 秒），不打断节奏。

### 4.3 特殊规则（保留）

| 规则 | 效果 |
|------|------|
| `double_speed` | 本波敌人移速 +100% |
| `fortified` | 本波敌人 HP +50% |
| `rush` | 敌人生成间隔减半 |
| `elite_wave` | 本波全部为精英敌人 |
| `boss_assist` | 关底 Boss 波时，Boss + 大量小怪同时生成 |

> **v3.0 删除**：`no_build` 规则（关内禁建毫无意义，因为玩家只是拖卡）。

### 4.4 敌人层级（Layer）— 保留

> 详见 [18-图层系统](./18-layer-system.md)。

| 层级 | 说明 | 可被攻击规则 |
|------|------|------------|
| `Ground` | 地面敌人（默认） | 所有塔都可命中 |
| `Flying` | 飞行敌人（L4+ 引入） | 仅反空塔和全层级塔可命中 |
| `Burrowing` | 钻地敌人（预留） | 仅特殊探测塔可命中 |

### 4.5 飞行敌、分散敌、特殊机制敌（v3.0 整合）

为制衡卡牌流派，引入多类抗性/机制敌人：

| 敌人 | 引入关卡 | 机制 hook |
|------|---------|----------|
| **飞行敌**（蝙蝠/小妖） | L4 | 不受地面 AOE 影响 |
| **分散敌**（多触手怪） | L5 | 移动中周期性散开 + 聚拢，降低 AOE 评分 |
| **治疗祭司** | L3 | 治疗周围敌人，**威胁度高优先被攻击** |
| **召唤师** | L6 | 死亡后召唤 3 个小怪 |
| **护盾兵** | L6 | 护盾未破时免疫伤害 |
| **隐形刺客** | L8 | 出生 3 秒内隐形 |
| **反射机甲** | L8 | 反弹 30% 受到的攻击 |
| **强化自爆兽** | L7 | 接近塔时自爆造成 AoE |

具体数值见 [21-MDA §6](./21-mda-numerical-design.md)。详细机制见 [03-unit-data](./03-unit-data.md) 与 [22-new-unit-design](./22-new-unit-design.md)。

## 5. 关间节点（v3.0 新增核心）

每个关卡（1-7）通过后，弹出二选一面板：

| 节点 | 设计意图 |
|------|---------|
| 🏪 **商店** | 稳健消费，明码标价 |
| 🌀 **秘境** | 冒险博弈，事件随机 |

详细规则见 [25-card-roguelike-refactor §3](./25-card-roguelike-refactor.md#3-关间节点系统核心-rogue-抉择替代波间-3-选-1)。

### 5.1 关 8 与终战之间

**没有关间节点**——玩家击败关 8 后**直接进入终战**。这是设计上故意的张力放大，玩家必须在关 7 后的节点为终战做准备。

## 7. 场景装饰

每个关卡有主题障碍物（纯视觉，不阻塞路径）：

| 主题 | 装饰物类型 |
|------|-----------|
| 草原 | 树、灌木、花丛 |
| 林地 | 巨树、蘑菇、藤蔓 |
| 湿地 | 枯木、水洼、水草 |
| 冰原 | 冰晶、雪松、冰封岩石 |
| 沙漠 | 岩石、仙人掌、枯骨 |
| 古城 | 残柱、断壁、藤蔓 |
| 火山 | 熔岩裂缝、烧焦枯树、火山岩 |
| 石堡 | 石柱、火炬台、碎石堆 |
| 深渊 | 漂浮石、紫色火焰、晶簇 |

装饰物散布在空地块上，随机摆放，不影响游戏逻辑。详见 [17-scene-decoration](./17-scene-decoration.md)。
