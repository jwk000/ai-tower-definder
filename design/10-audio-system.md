# 10 — 音效系统

> 音乐与音效完整清单，覆盖全部游戏系统。基于 v2.2 设计文档与代码实现交叉分析。

---

## 一、音效风格总纲

- **整体风格**：电子合成器 + 8-bit 元素，匹配几何美术风格
- **时长规范**：UI 音效 < 100ms，战斗音效 100–500ms，技能/Boss 500–1000ms，BGM 循环
- **格式**：WAV（开发期）/ MP3 或 OGG（发布期）
- **引擎**：Web Audio API，预加载到 AudioBuffer
- **延迟要求**：UI 音效 < 50ms
- **同时播放上限**：12 个音效 + 2 个环境 Loop + 1 个 BGM
- **总大小预算**：BGM < 2MB（压缩后），SFX < 500KB（压缩后）

## 十三、技术需求

### 13.1 音频格式
- 开发期：WAV（无损，便于迭代）
- 发布期：MP3（BGM）/ OGG 或 MP3（SFX）
- 采样率：44.1kHz，单声道（SFX）/ 立体声（BGM）

### 13.2 加载策略
- 所有音效预加载到 `AudioBuffer`（Web Audio API）
- BGM 使用 `<audio>` 元素流式播放（避免占用过大内存）
- 环境 Loop 预加载为 `AudioBuffer`，使用 `loop = true`
- 预加载在进入关卡前完成，加载失败不阻塞游戏启动

### 13.3 播放策略
- 音效实例池：每种音效预创建 3–5 个播放实例，避免高频触发时创建/销毁开销
- 节流保护：高频音效（如塔射击、敌人受击）设置最小间隔（20–40ms），避免刺耳堆积
- 优先级剔除：同时播放数超限时，按优先级剔除最低优先级的音效
- 音量分级：UI 音量 0.5 / 战斗 0.6 / 环境 0.3 / BGM 0.4（可配置）

### 13.4 用户控制
- 主音量 0–100%
- BGM 独立音量控制
- SFX 独立音量控制
- 一键静音（保留在 localStorage）

### 13.5 性能预算
- BGM：< 2MB（压缩后，MP3 128kbps）
- SFX 全部：< 500KB（压缩后）
- 环境 Loop：每条 < 200KB（压缩后）
- 首屏加载音频总大小：< 3MB
