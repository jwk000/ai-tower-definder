## 三、地格评分系统（核心机制）

### 3.1 评分概述

导弹塔攻击时，不选择单个敌人，而是评估地图上所有**可达地格**的"攻击收益"，选择得分最高的地格发射导弹。

评估范围：**所有有敌人存在的地格**（至少有一个敌人站在该地格上，或该地格在敌人所在格的相邻范围）

### 3.2 评分因子

地格 `(row, col)` 的攻击收益评分由三个因子加权计算：

#### 因子一：距离因子（Base Proximity）

> **敌方单位距离基地越近，分数越高。**

```
距离因子 = (基地距离_最大值 - 敌人到基地的路径距离) / 基地距离_最大值 × 距离权重
```

- 基地位置：地图上的 `base` tile
- 路径距离：敌人到基地的**沿路径步数**（不是直线距离），沿 `enemyPath` 行进方向计算当前位置距基地的剩余步数
- 每个地格取该格上**所有敌人中距离基地最近的那个**作为该格的距离值
- `基地距离_最大值` = 地图路径总长度（步数）
- `距离权重` = 0.35（可调）

> **设计意图**：优先打击最紧迫的威胁——即将到达基地的敌人。这符合塔防游戏"防漏"的核心体验。

#### 因子二：密度因子（Enemy Density）

> **以该地格为中心，爆炸范围内敌方单位越多，分数越高。**

```
密度因子 = min(爆炸范围内敌方单位数 / 密度上限, 1.0) × 密度权重
```

- 爆炸范围 = 导弹塔当前等级的 `爆炸半径`（L1=120px）
- `密度上限` = 8（超过8个敌人不再额外加分，防止极端情况失真）
- `密度权重` = 0.45（可调）

> **设计意图**：鼓励玩家在敌人密集时开火，最大化AOE收益。这是导弹塔"等待最佳时机"的核心动力。

#### 因子三：等级因子（Enemy Tier）

> **攻击目标单位的等级越高，评分越高。**

```
等级因子 = sum(每个在爆炸范围内敌人的 tier_multiplier) / (敌人数量 × 最大tier倍率) × 等级权重
```

| 敌人层级 | tier_multiplier |
|---------|-----------------|
| L1 普通（小兵、快兵） | 1.0 |
| L2 精英（重装、法师、自爆虫） | 2.0 |
| L3 BOSS（指挥官、攻城兽） | 4.0 |

- `等级权重` = 0.20（可调）

> **设计意图**：导弹塔对高价值目标有天然偏好。打中BOSS比打中小兵更有价值，引导导弹优先处理最危险的敌人。

### 3.3 综合评分公式

```
total_score = 距离因子 + 密度因子 + 等级因子 + 同格bonus
```

**同格bonus**: 如果敌人恰好站在被评估的地格上（而非仅在爆炸范围内），额外加0.05分。这确保导弹优先打在敌人"身上"而非空旷的地格。

所有因子归一化到 [0, 1] 范围，权重总和 = 1.0。

### 3.4 计算流程

```
每帧（当装弹完成时）：
  1. 收集所有存活的敌方单位及其位置
  2. 遍历每个有敌人的地格（去重，同一格多个敌人只评估一次）
  3. 对每个地格计算综合评分：
     a. 计算距离因子：取该格上距离基地最近的敌人
     b. 计算密度因子：以该格为中心，统计爆炸半径内所有敌人
     c. 计算等级因子：对爆炸半径内所有敌人按tier加权
     d. 加同格bonus
  4. 选取评分最高的地格
  5. 向该地格发射导弹
```

### 3.5 边界规则

| 场景 | 处理方式 |
|------|---------|
| 没有敌人 | 不发射，等待 |
| 多个地格评分相同 | 优先选距离基地更近的 |
| 爆炸超出地图边界 | 正常计算，爆炸范围裁剪到地图内 |
| 唯一敌人 | 直接锁定该敌人所在格（评分 = 距离因子 + 等级因子） |
| BOSS免疫眩晕 | 导弹的爆炸伤害对BOSS完全有效，不受免疫影响 |

| 弹道形状 | 复合几何体：弹体（矩形）+ 弹头（三角形）+ 尾焰（渐变三角+粒子尾迹） |

| 尾焰粒子 | 飞行时后方生成红色→橙色渐变粒子，每0.05s发射2-3个 |

### 4.2 爆炸效果

| 阶段 | 时间 | 效果 |
|------|------|------|
| 命中瞬间 | 0ms | 白色闪光（全屏alpha 0.3，持续0.05s） |
| 爆炸核心 | 0-0.2s | 中央白→橙→红渐变球形快速膨胀至爆炸半径 |
| 冲击波环 | 0.05-0.4s | 红色冲击波环从中心向外扩散，alpha从1.0衰减到0 |
| 烟尘蘑菇 | 0.1-1.5s | 蘑菇云升起（灰色圆顶 + 底部扩散圆环），alpha衰减 |
| 地面焦痕 | 0.2-5.0s | 命中点地面留下黑色焦痕（不透明圆，半径=爆炸半径×0.7），逐渐消退 |
| 碎片粒子 | 0-0.8s | 20-30个随机方向碎片粒子（橙色/红色/灰色），有重力的抛物线运动 |

### 4.3 伤害计算

```
基础伤害 = 导弹塔ATK

物理减伤 = 基础伤害 × (1 - 敌人护甲 / (护甲 + 100))

实际伤害 = 减伤后伤害 × 距离衰减系数
```

**距离衰减**（仅L5热压弹头被动生效）：

| 距爆炸中心距离 | 伤害比例 |
|--------------|---------|
| 0 - 30% 半径 | 130%（热压弹头加成） |
| 30% - 60% 半径 | 100% |
| 60% - 100% 半径 | 70% |

无L5被动时：全爆炸范围100%伤害。

### 4.4 屏幕震动

| 参数 | 值 |
|------|-----|
| 触发 | 导弹爆炸时 |
| 强度 | 6px |
| 频率 | 25Hz |
| 持续时间 | 0.3s |
| 衰减 | ease-out |

### 7.2 攻击系统集成

导弹塔需要特殊的攻击逻辑，不应走现有的 `AttackSystem` 的 `nearest` 目标选择。建议：

1. **在 `AttackSystem` 中增加导弹塔分支**：检测 `towerType === Missile` 时走独立的评分+地格选择逻辑
2. **导弹飞行**：复用 `ProjectileSystem`，但导弹的目标是 `(x, y)` 坐标而非 `entityId`
3. **爆炸伤害**：复用 `ProjectileSystem.applySplash()` 的AOE伤害逻辑
4. **视觉特效**：复用 `ExplosionEffect` 组件（可能需要扩展爆炸半径和持续时间参数）

### 7.3 地格坐标系统

导弹塔需要知道：
- 地图的 `tileSize`（64px），用于地格坐标 → 像素坐标转换
- 基地位置（`base` tile）
- 所有敌人的实时位置（已有 `Position` 组件）
- `enemyPath` 路径，用于计算路径距离

### 7.6 导弹定位地格的实现方式（关键设计决策）

导弹塔的与众不同在于**攻击目标不是实体，而是地格坐标**。现有 `ProjectileSystem` 的弹道需要一个 `targetId`（实体ID）来飞行。解决方案：

**创建临时"打击标记"实体**：
1. 导弹塔通过评分系统选出最佳地格 `(row, col)`
2. 将该地格中心转换为像素坐标：`(cx, cy) = (offsetX + col * tileSize + tileSize/2, offsetY + row * tileSize + tileSize/2)`
3. 创建一个不可见的"marker"实体（含 `Position` 组件，不含 `Visual` 组件）
4. 发射的 `Projectile.targetId` 指向该 marker 实体
5. 弹道到达 → `ProjectileSystem.onHit()` 触发 → `applySplash()` 在 `(hitX, hitY)` 处造成AOE伤害
6. marker 实体在命中后销毁

```typescript
// 伪代码（AttackSystem中的导弹塔分支）
const targetRow = bestTile.row;
const targetCol = bestTile.col;
const targetX = scene.offsetX + targetCol * tileSize + tileSize / 2;
const targetY = scene.offsetY + targetRow * tileSize + tileSize / 2;

// 创建临时标记实体
const markId = world.createEntity();
world.addComponent(markId, Position, { x: targetX, y: targetY });

// 发射导弹，目标指向标记实体
spawnProjectile(world, towerId, markId, towerTypeVal);
// 标记实体在 ProjectileSystem.onHit 命中后由 world.destroyEntity 清理
```

**优点**：完全复用现有 `ProjectileSystem` 的飞行、命中、AOE、爆炸特效逻辑，零侵入性修改。

## 八、平衡性考量

### 8.1 与其他塔的对比

| 塔 | 单发伤害 | 攻速 | 范围 | AOE | 造价 | 特色 |
|----|---------|------|------|-----|------|------|
| 箭塔L1 | 10 | 1.0/s | 180 | 无 | 50G | DPS基准 |
| 炮塔L1 | 30 | 0.25/s | 180 | 80px | 80G | 溅射AOE+眩晕 |
| 导弹塔L1 | 80 | 0.15/s | 全图 | 120px | 200G | 全图+智能打击 |

**DPS对比（L1）**：
- 箭塔：10 DPS（单体）
- 炮塔：7.5 DPS（AOE）
- 导弹塔：12 DPS（AOE全图），但必须在敌人密集时才能发挥AOE价值

### 8.2 风险与限制

| 风险 | 缓解措施 |
|------|---------|
| 单体伤害过高，替代所有其他塔 | 极长装弹 + 高造价 + 高人口，确保不能"导弹塔海" |
| 全图射程过于OP | 飞行时间（200px/s穿过1344px地图 = 最多6.7秒），给敌人移动时间 |
| 评分系统可能不直观 | 可选的发射预览：鼠标悬停时显示第一名地格的评分和预估伤害 |
| BOSS波过于克制 | BOSS单体评分高是设计意图，但导弹长装弹期让玩家不能完全依赖 |

### 8.3 建议的关卡开放

| 关卡 | 是否可用 |
|------|---------|
| L1 平原 | ❌ |
| L2 沙漠 | ❌ |
| L3 冰原 | ❌ |
| L4 火山 | ✅ 解锁（与炮塔、电塔一同可用） |
| L5 城堡 | ✅ |
| 无尽模式 | ✅ |

> 理由：导弹塔属于高级战略武器，应在玩家已经熟悉基础塔防后再解锁。L4火山有大量blocked地形造成的窄路径，天然适合AOE塔发挥。

## 九、交互设计

### 9.1 建造

- 在建造面板中作为第7个塔选项出现（需要先解锁）
- 图标：红色三角形（导弹弹头符号） + 雷达环，底色深灰
- 建造动画：从地下升起的发射平台 + 导弹装载动画（1.0s）

### 9.2 装弹状态反馈

- **装弹进度环**：塔周围一圈红色弧形进度环，逆时针从12点方向消退
- **装弹完成**：进度环合拢瞬间闪白 + 音效 + 塔顶短暂辉光
- **发射瞬间**：塔身微缩（反冲效果，0.1s）→ 导弹飞出

### 9.3 目标预览（可选特性）

- 鼠标悬停在导弹塔上时，显示当前会选择的目标地格高亮（红色十字准星）
- 显示"预估伤害"和"目标敌人数量"的浮动信息
- 此功能可切换（默认开启，可在设置中关闭）
