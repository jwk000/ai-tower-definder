# Run 长征循环 ─ v3.4 单 Run 闭环（v2.0.0）

> **本文档是 v3.4 形态的 Run 长征权威。** 由 v1.0（v3.0-v3.3 时期 893 行卡牌化+Roguelike+永久解锁方案）完全重写而来。原版已归档至 [archive/10-roguelike-loop_v1.0_2026.05.15.md](../archive/10-roguelike-loop_v1.0_2026.05.15.md)。
>
> **v3.4 形态级变更**（与 v1.0 不兼容）：
> 1. **三资源重定义**：能量 / 金币 / **技能点**（替代火花碎片）。技能点本 Run 临时分配，Run 结束清零。
> 2. **meta 永久积累彻底废弃**：无永久解锁、无卡池升级、无关外科技树。所有卡开局即解锁。
> 3. **Run 结算简化**：死亡无回报，胜利无 meta 收益。**单 Run 闭环**。
> 4. **§3.2 商店内容**完全移出本文档 → 由 [48-shop-redesign-v34](../40-presentation/48-shop-redesign-v34.md) 全权承担。
> 5. **§3.3 秘境奖励/风险重做**：碎片奖励 → 混合奖励（金币+技能点+单位卡按权重）；新增 30% 高风险事件。
> 6. **B4 关卡通关额外抽卡**：每关通关后开一次 3 选 1 单位卡抽卡（v3.4 新机制）。
> 7. **§8 永久解锁系统整节删除**，14 节缩减为 13 节。

## 1. v3.4 玩法定位

| 维度 | v3.4 形态 |
|---|---|
| 核心循环 | 抽卡组 → 战斗 → 关后 3 选 1（商店 / 秘境 / 跳过） → 下一关 → 终战 |
| 单 Run 时长 | 25-40 分钟，连闯 8 关 + 终战（共 9 关 = W1-W23） |
| 失败代价 | **从第 1 关重开，无任何 meta 留存**（决策 B2a） |
| 胜利收益 | **本 Run 闭环**，仅记录"通关次数 + 历届最远 + Hero Score"展示型数据（决策 A2/C1） |
| 单 Run 内策略 | 抽到什么卡、能量怎么分、技能点投哪个单位、商店买什么、秘境冒不冒险 |
| 长线驱动 | 历届最远关卡、Hero Score、Run 通关次数（仅展示，不解锁内容） |

### 1.1 三大支柱（v3.4）

1. **卡组随机性** — 开局抽 12 张卡（全卡池开局已解锁，无 meta 卡池）决定本局流派
2. **关后 3 选 1 抉择** — 商店（稳健）/ 秘境（冒险）/ 跳过（保血）三向博弈
3. **血量继承张力** — 水晶 HP 全程继承，决策 B1a，前期失血付出后期代价

### 1.2 受影响的设计哲学（v3.4 锁定）

- ✅ **保留**：一切皆单位、一切皆卡牌、配置驱动、行为树驱动 AI、符号优于文字
- ✅ **保留**：关卡随机性（天气、装饰物、波次组成、秘境事件池）
- 🆕 **v3.4 新增**：**单 Run 闭环哲学** — 死亡无回报，胜利无 meta 收益，所有进度仅存活于本 Run 内
- 🛑 **v3.4 删除**：永久卡池、火花碎片、关外科技树、meta 商店、跨 Run 解锁

## 2. 卡牌系统

### 2.1 卡牌定义（v3.4）

卡牌是「单位/法术配置 + 打出方式 + (本 Run) 技能点状态」的封装。

```typescript
interface CardConfig {
  id: string;
  name: string;
  type: 'unit' | 'spell' | 'trap' | 'production' | 'shop_item';   // v3.4 新增第 5 类 shop_item，详见 48 §3
  energyCost: number;
  rarity: 'common' | 'rare' | 'epic' | 'legendary';

  // type=unit 时引用现有单位配置
  unitConfigId?: string;

  // type=spell 时引用法术效果
  spellEffectId?: string;
  spellSubtype?: 'damage' | 'control' | 'buff_instance' | 'utility';

  // 部署/释放约束
  placement: {
    targetType: 'tile' | 'enemy' | 'ally' | 'area' | 'global';
    range?: 'cursor' | 'path' | 'anywhere' | 'left-half' | 'right-half';
  };

  // 跨波保留标记（默认 false 波末弃牌；true 可留下波）
  persistAcrossWaves?: boolean;
}
```

> **v3.4 删除字段**：`cardLevel` / `techTree` / `unlocked` 等 meta 持久化字段全部删除（详见 [61-save-system](../60-tech/61-save-system.md) v3.0.0）。

### 2.2 卡牌四大分类（v3.4 战斗内）

| 大类 | 子类 | 说明 |
|---|---|---|
| **建筑卡（Tower）** | 防御塔（单体 / AoE / 控制）、生产塔（金矿 / 能量塔）、辅助塔（治疗 / 增益） |
| **单位卡（Soldier）** | 近战（盾兵 / 剑士）、远程（弓手 / 法师）、空中（飞兵）、特殊（祭司 / 工程师 / 召唤师） |
| **法术卡（Spell）** | 伤害 / 控制 / Buff（包括对塔实例 `instanceLevel +1`）/ Debuff / Utility |
| **陷阱卡（Trap）** | 详见 [27-traps-spells-scene §3](../20-units/27-traps-spells-scene.md#3-陷阱) |

> **第 5 类 shop_item 不进战斗手牌**，仅在商店内呈现，详见 [48 §3](../40-presentation/48-shop-redesign-v34.md#3-cardtype-拓展shop_item-为第-5-类)。

### 2.3 抽卡组与手牌机制（v3.4）

```
开局（进入本局 Run）：
  1. 从「全卡池」（v3.4 所有卡开局即解锁）按稀有度权重随机抽 12 张组成本局卡组
     - 权重：Common 60% / Rare 25% / Epic 12% / Legendary 3%（数值由 50-mda 持有）
     - 保底：至少 1 张建筑卡 + 1 张单位卡 + 1 张法术卡
  2. 第 1 关第 1 波开始前，从卡组顶部抽 4 张到手牌

每波开始：
  1. 从卡组顶部抽牌补满手牌（默认上限 4，固定不变）
  2. 上一波未打出的卡：
     - persistAcrossWaves = true（如「时停」「全屏冰冻」）：保留在手牌
     - persistAcrossWaves = false（绝大多数卡）：进入弃牌堆

战斗中：
  1. 玩家从手牌拖卡到地图部署，消耗能量 E
  2. 单位卡：在地图上生成实体（沿用 20-unit-system）
  3. 法术卡：立即执行效果后进入弃牌堆
  4. 建筑卡：放置到地格，可被攻击摧毁

波次结束：
  1. 结算金币（击杀奖励 + 波次完成基础奖励）
  2. 短暂过渡（2-3 秒）后进入下一波

关卡通过（最后一波 + 关底 Boss 被击败）：
  1. 发放金币 + 技能点关卡通关奖励（数值由 50-mda §18 持有）
  2. 🆕 决策 B4c：弹出「3 选 1 单位卡抽卡」面板（详见 §2.6）
  3. 进入「关后 3 选 1 节点」（详见 §3，UI 在 47 §4）

卡组耗尽时：洗弃牌堆为新卡组（标准 Rogue Deckbuilding）
```

### 2.4 手牌规则（v3.4 简化）

- **手牌上限固定 4 张**（v1.0 的「永久升级扩至 8 张」机制随 meta 系统一并废弃）
- 抽到第 5 张时（已满）→ 该卡直接进入弃牌堆 + 屏幕飘字「手牌已满」（沿用 v1.0 决策）

### 2.5 卡牌升级（v3.4 重设）

> **v3.4 锁定**：本 Run 内所有"升级"由**技能点**承担，本 Run 临时分配，Run 结束清零。
>
> - **技能点来源**：关卡通关奖励、商店购买（金币兑换，详见 [48 §5.3/§5.4](../40-presentation/48-shop-redesign-v34.md#5-后-4-槽随机池4-种功能卡)）、秘境奖励（部分事件）
> - **技能点消费**：投入到「本 Run 单位技能树」节点（详见第 3 轮新建的 22-skill-tree-overview + 22a-22e 全单位技能树文档；当前 v3.4 第 2 轮文档稿尚未落地，**接口暂为占位**）
> - **关内法术卡 `instanceLevel`**：仅作用于场上塔实例，叠加 3 层上限，塔死亡 / 关结束清零（沿用 v1.0 决策，详见 [23-skill-buff §7](../20-units/23-skill-buff.md)）
> - **v3.4 删除**：关外卡池科技树、关间商店「升级 X 卡」服务、跨 Run 持有的卡级 L1→L2→L3

### 2.6 关卡通关 3 选 1 单位卡抽卡（决策 B4c · v3.4 新机制）

每关通关时（W1-W22，终战 W23 例外），在「关后 3 选 1 节点」面板**之前**插入一次单位卡抽卡：

```
关卡 N 通过！选 1 张单位卡加入卡组：

┌──────────┬──────────┬──────────┐
│  弓手    │  炮塔    │  剑士    │
│  Common  │  Rare    │  Common  │
│  E:3     │  E:5     │  E:2     │
└──────────┴──────────┴──────────┘
[ 跳过（不加卡） ]
```

- 池子：当前未在卡组内的所有单位卡（按稀有度权重抽 3 张）
- 选中：直接加入卡组（不消耗任何资源），洗入抽牌堆
- 跳过：不加卡，直接进入关后 3 选 1
- **终战 W23 不触发本机制**（终战是 Run 终点，无关后节点）
- 数值由 [50-mda §18](../50-data-numerical/50-mda.md) 持有（稀有度权重、特殊池规则等）

### 2.7 卡牌死亡 / 销毁

- 单位卡战场被击杀 → **不回弃牌堆**（避免免费换血，沿用 v1.0 决策）
- 建筑卡被摧毁 → 同上，不回弃牌堆
- 法术卡打出后 → 进入弃牌堆，下一轮洗牌可再抽

## 3. 关后节点系统（v3.4 重设）

### 3.1 节点结构（与 47 一致）

**节点 UI 权威**：[47-level-map-ui §4](../40-presentation/47-level-map-ui.md#4-胜利结算--3-选-1-一屏面板)。

```
关卡 N 通过（W1-W22）→ 单位卡 3 选 1（§2.6）→ 关后 3 选 1 节点：

┌─────────────────────────────────────────────────┐
│  关卡 N 通过！选择前往：                              │
├─────────────┬─────────────┬─────────────────────┤
│  🏪 商店    │  🌀 秘境    │  ⏭ 跳过              │
│             │             │                      │
│  确定收益   │  随机事件   │  零成本进入下一关       │
│  详见 48    │  详见 §3.3 │  保血保稳节奏          │
└─────────────┴─────────────┴─────────────────────┘
```

- **三向互斥**：选 1 即进入下一关，不可回头（v1.0 二选一 → v3.4 三选一）
- **终战 W23 前**（即 W22 通关后）：3 选 1 仍可用，玩家进商店补技能点 / 秘境冒险 / 跳过保血面对终战
- **终战 W23 通关后**：直接进入 Run 结算（详见 §6）

### 3.2 商店节点（v3.4 完全交付 48）

**本节内容已完全移出 10-roguelike-loop。请直接阅读 [48-shop-redesign-v34](../40-presentation/48-shop-redesign-v34.md) 全文**（决策 D1）。

10-roguelike-loop 仅保留以下边界声明：

| 维度 | 边界声明 |
|---|---|
| **触发条件** | 关后 3 选 1 节点中选「商店」时进入 |
| **入场资源** | 玩家当前金币余额 + 当前技能点（48 锁定） |

| **金币 ↔ 技能点互通** | 商店内可花金币买技能点（48 §5.3/§5.4，决策 F2） |

> **不要在此处或其它文档重复定义商店内容。任何商店改动只改 48。**

### 3.3 秘境节点（v3.4 重设 · 决策 E1c + E2b）

秘境是**怪物火车风格的事件房间**，进入时显示一段情境文字 + 2-3 个选择，每个选择有明确收益和潜在代价。

**v3.4 秘境奖励改造**（决策 E1c）：旧 v1.0 的「碎片奖励」改为**混合奖励池**（金币+技能点+单位卡按权重）。具体奖励组合由 [27-traps-spells-scene §5 秘境事件池](../20-units/27-traps-spells-scene.md#5-秘境事件池) 持有，本文档不重复定义。

**v3.4 新增 30% 高风险事件**（决策 E2b）：在秘境事件池中，约 30% 的事件可能造成**伤害性代价**（损 HP、删卡、减能量上限等），制造单 Run 闭环下的抉择张力。

```
秘境面板示例（v3.4）：

┌──────────────────────────────────────────────────┐
│  🌀 秘境 — 古老的祭坛                                  │
│                                                  │
│  你发现一座血色祭坛，散发着不祥的气息。               │
│  上面刻着三种契约：                                   │
├──────────────────────────────────────────────────┤
│  [A] 献祭 30% 水晶 HP                                │
│      → 获得 5 技能点 + 1 张 Legendary 单位卡         │
│                                                  │
│  [B] 献祭 1 张随机卡                                  │
│      → 水晶恢复 40 HP + 80 金币                      │
│                                                  │
│  [C] 拒绝契约离开                                      │
│      → 不消耗任何资源直接进入下一关                   │
└──────────────────────────────────────────────────┘
```

**v3.4 秘境特性**：
- 每次秘境**至少有一个零成本选项**（如「拒绝契约离开」），玩家可无代价退出
- 事件结果立即生效，无倒计时
- 已遇过的事件本 Run 不重复
- **30% 高风险事件**用色调区分（红边框），强警示玩家代价
- 奖励池替换详细对照表在 [27-traps-spells-scene §5](../20-units/27-traps-spells-scene.md#5-秘境事件池)（第 4 轮一般文档清理时同步重写）

### 3.4 跳过节点（v3.4 新增 · 决策 A2 锁定）

「跳过」分支零成本：不进商店、不触秘境，直接进入下一关。

| 维度 | 跳过节点 |
|---|---|
| 触发 | 关后 3 选 1 选「跳过」 |
| 资源消耗 | 无 |
| 资源获得 | 无（仅水晶 HP / 卡组 / 金币 / 技能点保持现状） |
| 设计意图 | 保血流派、节奏控制、屯资源对终战 |

> 跳过是 v3.3 引入的稳健选项，v3.4 保留并明确"完全零代价"原则。详见 [47 §4](../40-presentation/47-level-map-ui.md#4-胜利结算--3-选-1-一屏面板)。

### 3.5 商店 vs 秘境 vs 跳过的设计意图

| 维度 | 商店 | 秘境 | 跳过 |
|---|---|---|---|
| 不确定性 | 低（明码标价） | 高（事件随机 + 30% 高风险） | 零（确定无变化） |
| 收益期望 | 稳定中等 | 方差大 | 零 |
| 触发玩家心态 | 资源管理 / 规划 | 冒险 / 博弈 | 保守 / 节奏控制 |
| 适合场景 | 金币充足、卡组成型 | 资源紧张、需爆发 | HP 危险、屯资源对终战 |

## 4. 能量系统（关内出卡资源）

### 4.1 能量定义（v3.4 沿用 v1.0）

**Energy (E)** 是关内唯一的出卡消耗资源，与金币、技能点完全分离。

```
能量规则：
  - 上限：10（固定）
  - 初始（每关第 1 波）：5
  - 每波开始恢复：+5（不超过上限）
  - 战斗中恢复：无
  - 部分特殊卡 / 秘境效果可额外恢复能量
```

> **v3.4 删除**：v1.0 的「永久升级扩展能量上限至 12」机制（随 meta 系统一并废弃）。

### 4.2 能量消耗参考

具体数值表由 [50-mda](../50-data-numerical/50-mda.md) 持有。语义快照（不作为真理源）：

| 卡 | 能量参考 |
|---|---|
| 基础箭塔（L1） | 低（约 3 E） |
| 高级炮塔（L1） | 中-高（约 6 E） |
| 火球术（伤害法术） | 低（约 2 E） |
| 箭雨（AoE 法术） | 中（约 4 E） |
| 治疗水晶 | 中（约 5 E） |
| 终极召唤：龙 | 满（10 E） |

### 4.3 三资源分工（v3.4 决策 F2 锁定）

| 资源 | 范围 | 来源 | 消耗 | 互通规则 |
|---|---|---|---|---|
| **能量 E** | 关内单波 | 每波开始恢复 | 出卡 | 完全独立 |
| **金币 G** | 整局 Run | 击杀 + 波次结算 + 通关奖励 | 关后商店 + 秘境部分事件 | **可在商店内花金币买技能点**（详见 [48 §5.3/§5.4](../40-presentation/48-shop-redesign-v34.md#5-后-4-槽随机池4-种功能卡)）/ 其他完全独立 |
| **技能点 SP** | 整局 Run | 通关奖励 + 商店购买 + 秘境奖励 | 投入「本 Run 单位技能树」节点 | 不能转金币 / 能量；Run 结束清零 |

## 5. 关卡结构

### 5.1 一次 Run 的完整结构（决策 B3a 保留 8 关 + 终战）

```
[主菜单 → 开始新 Run]
  → [开局抽 12 张卡组]
  → [关卡 1: 平原]（8-10 波 = W1-W3 区段）
      → 水晶 HP 从初始值 100 开始（决策 B1a 跨关继承）
  → [单位卡 3 选 1 抽卡]
  → [关后 3 选 1：商店 / 秘境 / 跳过]
  → [关卡 2: 森林]
  → [单位卡 3 选 1]
  → [关后 3 选 1]
  → ...
  → [关卡 8: 王城]
  → [单位卡 3 选 1]
  → [关后 3 选 1]
  → [关卡 9: 终战 Boss 关 = W23]（单波超长，混合所有敌人类型，无关后节点）
      → 击败 → Run 胜利（详见 §6）
      → 失败或中途任何一关水晶 HP=0 → Run 失败（详见 §6）
```

**Run 总波次**：约 70-90 波 / Run（每关 8-10 波，外加终战）。预计时长 25-40 分钟。

> **关卡映射**：W1-W3 = 关 1，W4-W6 = 关 2，... ，W22 = 关 8，W23 = 终战。详见 [47 §1 世界地图](../40-presentation/47-level-map-ui.md#1-世界地图总览)。

### 5.2 水晶机制（沿用 v1.0 · 决策 B1a 锁定跨关 HP 继承）

> 水晶（Crystal）是玩家方核心防御实体，位于地图左侧路径终点。

#### 5.2.1 定位与外观

- **类型**：特殊单位（`unit_kind=crystal`），固定占用 1 个 tile，不可被部署 / 移除 / 移动
- **视觉**：红色水晶造型，悬浮 + 辉光呼吸。详见 [42-art-assets §13.13](../40-presentation/42-art-assets.md) 与 [44-visual-effects §10](../40-presentation/44-visual-effects.md)

#### 5.2.2 无敌规则（沿用 v1.0）

水晶**免疫一切伤害源**——任何敌人的普攻、技能、AoE、毒、点燃、自爆、Boss 必杀、反弹伤害等**都无法直接扣除水晶 HP**。

- 唯一 HP 消耗途径是**主动秒杀**（见 5.2.3）
- 正向操作仍生效：「治疗水晶」法术卡、商店「水晶 HP 恢复」道具（详见 [48 §5.2](../40-presentation/48-shop-redesign-v34.md#5-后-4-槽随机池4-种功能卡)）、秘境治疗事件等按既定规则结算
- 秘境「献祭 X% 水晶 HP」等主动负向操作仍生效（属于玩家自愿付出的代价）
- 取消旧规则：**敌人抵达终点不再扣 HP**，由「秒杀消耗 1 HP」彻底替代

#### 5.2.3 秒杀攻击（核心战斗机制 · 沿用 v1.0）

水晶把进入其攻击范围的敌人立即秒杀（无视目标的 HP、护盾、无敌、霸体、闪避），每次秒杀**消耗水晶 1 HP**。

| 字段 | 语义 | 默认值 | 真理源 |
|---|---|---|---|
| `crystal.attackRange` | 攻击范围（地格数） | 1 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.attackCooldown` | 攻击间隔（秒） | 0.5 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.hpCostPerKill` | 每次秒杀消耗的 HP | 1 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.maxKillsPerTick` | 单次冷却内最大击杀数 | 1 | [50-mda](../50-data-numerical/50-mda.md) |

**目标选择**：范围内多敌人时按「最近距离 > 进入范围时间」排序，每次秒杀 1 个。

**Boss 例外**：终战 Boss（`abyss_lord`）携带 `immune_to_crystal_kill = true`，水晶**无法秒杀 Boss**。Boss 进入水晶范围也不消耗 HP，但 Boss 仍受水晶的无敌保护（无敌规则继续生效）。

#### 5.2.4 水晶 HP 变化（决策 B1a 跨关继承）

```
初始 HP：100（固定，v3.4 删除了 v1.0 的「baseHpBonus 永久升级」机制）

水晶 HP 变化：
  - 秒杀 1 个敌人：-1 HP
  - 商店「水晶 HP 恢复」：+N HP（详见 48 §5.2）
  - 秘境治疗事件：+N HP
  - 「治疗水晶」法术卡（关内）：+N HP
  - 秘境献祭 / 高风险事件：按事件配置 -N HP
  - 关卡通过：HP 跨关继承（决策 B1a），不自动恢复
  - 关后 3 选 1 选「跳过」：HP 跨关继承（决策 B1a）

水晶 HP = 0 → 水晶碎裂动画 → Run 失败
```

> **决策 B1a 锁定**：水晶 HP 全程跨关继承，每关失血在后期付出代价；恢复路径仅限商店购买 / 秘境治疗 / 法术卡。
