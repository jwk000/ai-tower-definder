# 经济系统（v3.4 单 Run 闭环版）

> ✅ **v3.4 形态级第 2 轮已完成（2026-05-15）**：本文档已从 v1.0.0「三资源含火花碎片 meta」**整文重写**为 v3.0.0「单 Run 闭环 + 技能点 SP」。
>
> **v3.4 三资源轴**：**能量 E（关内单波）/ 金币 G（本 Run 关间）/ 技能点 SP（本 Run 技能树）**。所有资源在 Run 结束时清零，单 Run 闭环无 meta 积累。
>
> **核心变更**：
> - ~~火花碎片（跨局 meta）~~ → **彻底废弃**，替换为 **技能点 SP（本 Run 临时）**
> - ~~金币 → 碎片 1:1 转换公式~~ → **彻底废弃**，Run 结束未花完金币直接清零
> - ~~主菜单永久解锁/永久升级机制~~ → **彻底废弃**，单 Run 闭环无任何 meta 沉淀通道
> - ✅ 新增 **金币 → SP 单向兑换**（50 G / 1 SP，仅商店内可换，单商店上限 3 SP）

> **数值真理源**：本文档不持有数值，所有数值（能量消耗 / 金币掉落 / 商店价格 / SP 兑换率 / 节点单价）以 [50-mda](../50-data-numerical/50-mda.md) 为准。本文档仅说明**规则与结构**。

## 1. 三种资源（v3.4 认知边界）

```
┌─────────────────────────────────────────────────────────┐
│  能量 E       ←→  关内单波范围      → 出卡部署            │
│   ↕ 每波回 5                                            │
│  金币 G       ←→  本 Run 关间范围    → 商店购买 / 秘境消费 │
│   │  ↓ 商店单向兑换（50 G / 1 SP，上限 3/商店）           │
│   ▼                                                     │
│  技能点 SP    ←→  本 Run 技能树范围  → 关后技能树节点      │
│                                                          │
│  Run 结束：三资源全部清零，无 meta 沉淀                   │
└─────────────────────────────────────────────────────────┘
```

| 资源 | 作用范围 | 持久性 | 来源 | 消耗用途 |
|------|---------|--------|------|---------|
| **能量 E** | 关内单波 | 每波开始重置 | 每波开始恢复 +5（不超过上限 10） | 战斗中拖卡部署消耗 |
| **金币 G** | 本 Run 关间 | 跨波次跨关卡累积；**Run 结束清零** | 击杀敌人 + 波次结算 + 秘境事件 | 关间商店 8 槽 / 秘境事件 |
| **技能点 SP** | 本 Run 技能树 | 跨关卡累积；**Run 结束清零** | 关卡通关（N×2 SP）+ 秘境事件（5/15/30/50 SP）+ 商店兑换（金币换） | 关后技能树节点（3/6/10/15 SP） |

**关键设计原则（v3.4 不变式）**：
1. 三种资源**作用范围严格分层**：能量不能跨波保留为金币、金币不能跨 Run 持有、SP 不能用来购买商店物品
2. **跨货币锁定**：仅允许 **金币 → SP** 单向兑换（商店内），其它任意方向禁止（10-roguelike-loop §11 不变式 v3.4-INV-08）
3. **Run 结束清零**：三资源全部归零，无 meta 沉淀通道（10-roguelike-loop §11 不变式 v3.4-INV-01）

## 2. 能量 E（关内出卡资源）

### 2.1 规则

| 维度 | 规则 |
|------|------|
| **上限** | **固定 10**（v3.4 锁定，不可永久升级，10-roguelike-loop §11 不变式 v3.4-INV-09） |
| **每波开始恢复** | +5（不超过上限） |
| **战斗中恢复** | **无被动恢复**（每波回合制） |
| **跨波保留** | 上一波末剩余能量保留进入下一波，受上限约束 |
| **额外恢复** | 部分特殊卡 / 秘境事件可额外回能；商店「能量瓶」道具 40 G/瓶（48 §3 槽 6） |

### 2.2 能量消耗示例（具体数值见 50-mda §12）

| 卡类型 | 典型能量消耗 |
|--------|------------|
| 基础建筑卡（箭塔 L1） | 3 E |
| 高级建筑卡（炮塔 L1 / 激光塔 L1） | 5-6 E |
| 基础单位卡（剑士 / 弓手） | 2-3 E |
| 高级单位卡（祭司 / 召唤师） | 4-5 E |
| 法术卡（火球术 / 减速） | 2-3 E |
| AoE 法术（箭雨） | 4 E |
| 终极召唤 / 全屏冰冻 | 8-10 E |

### 2.3 能量与塔的解耦

> **重要差异**：旧版能量是「能量塔产出 → 释放主动技能」，v3.0 起改为「每波回合制 → 出任何卡」。
>
> 能量塔（重命名「能量水晶」）作为可选建筑卡保留在卡池中，效果是「下波开始 +3 E」或临时上限提升（具体见 50-mda §12），但**不是默认拥有的资源生产链**。

## 3. 金币 G（本 Run 关间货币）

### 3.1 流入途径

| 来源 | 触发时机 | 数值（50-mda §13） |
|------|----------|---------------------|
| **击杀敌人** | 敌人死亡瞬时 | 按敌人 `bounty` 配置（普通 3-8 / 精英 15-25 / Boss 80-150） |
| **波次完成** | 全部敌人死亡 | 基础 20 G + 5 G × 关卡序号 |
| **关卡通过** | 关底 Boss 击败 | 100 G + 50 G × 关卡序号 |
| **秘境事件** | 部分事件选项 | 视事件 0-200 G |
| **生产建筑卡（如金矿）** | 部署后每秒产出 | 见 50-mda §13.2（金矿 L1 约 3 G/s） |

### 3.2 流出途径（仅在关间节点消耗）

| 节点 | 消耗用途 | 价格范围 |
|------|---------|---------|
| **商店 8 槽**（48 §3） | 前 4 槽单位卡（30/60/120/240 G） + 后 4 槽功能卡（SP 兑换 50 G/SP / 能量瓶 40 G / 水晶修复 80 G / 手牌刷新 0 G） | 30-240 G |
| **秘境** | 部分事件需消耗金币换取效果 | 20-150 G |

### 3.3 关内不能花金币的设计意图

**战斗中金币只攒不花**，这是 v3.0 设计核心、v3.4 维持：
- 给玩家明确的「攒钱节奏感」（关卡内只关注出卡 + 击杀）
- 关间一次性决策让"用金币干什么"成为正式策略时刻
- 简化关内 UI（顶部 HUD 只显示能量 + HP，金币以累加方式呈现）

### 3.4 初始金币

```
Run 起始金币：0 G（v3.4 锁定，10-roguelike-loop §11 不变式 v3.4-INV-02）
```

每局都从零开始累积，**v3.4 单 Run 闭环无 meta 永久升级通道**（v1.0 时期"永久升级 Run 起始金币 +50"机制已废弃）。

### 3.5 金币 → 技能点 SP 单向兑换（v3.4 新增）

**这是 v3.4 唯一允许的跨货币转换通道**，仅商店内可触发。

| 维度 | 规则 |
|------|------|
| **兑换率** | **50 G : 1 SP** 固定（50-mda §13.3） |
| **触发位置** | 商店第 5 槽（48 §3 槽 5 / 50-mda §14.3） |
| **单商店上限** | 3 SP（即单商店最多消耗 150 G 换 SP） |
| **全 Run 极限** | 24 SP（8 关 × 3 SP，但实际受关卡金币预算约束更低，50-mda §17.2.3 预测 5-15 SP 流量贡献） |
| **方向** | **单向**：金币 → SP 允许；SP → 金币 / SP → 物品 禁止 |

**设计意图**：
- 给「金币富裕但 SP 缺乏」的玩家一条流通路径（如关后金币 200 G 但已点完关键节点 0 SP）
- 兑换率 50 G/SP 与关卡通关 SP 奖励（N×2 SP）量级匹配，避免后期金币爆量 dump 破坏 SP 流量节奏
- 单商店上限 3 SP 防止「关后 800 G 全部换 16 SP」破坏 SP 流量曲线

详见 [50-mda §13.3](../50-data-numerical/50-mda.md#133-金币--技能点-sp-单向兑换v34) + [50-mda §17.2.3](../50-data-numerical/50-mda.md#1723-商店兑换-sp-流量)。

### 3.6 Run 结束金币处理（v3.4 关键变更）

```
Run 结束（胜利 / 失败 / 退出）：
  剩余金币 G → 直接清零，无任何 meta 沉淀
```

**v1.0 → v3.4 对比**：

| 维度 | v1.0（v3.0/v3.1） | v3.4 |
|------|------|------|
| Run 结束未花完金币 | 1:1 转火花碎片入账 meta | **直接清零** |
| 跨 Run 持有金币 | 永久升级"Run 起始金币 +50/+100"可解锁 | **彻底废弃**（起始永远 0） |
| 玩家激励 | "攒下不花，留给下一局" | **本 Run 花完更划算**（不花就浪费） |

详见 [50-mda §13.4](../50-data-numerical/50-mda.md#134-v30v34-金币迁移说明)。

## 4. 技能点 SP（本 Run 技能树资源）

> 🛑 **v3.0/v3.1 火花碎片机制已彻底废弃（v3.4 重写）**：
> - ~~Run 结束按到达关卡发放 10-200 碎片~~ → **删除**
> - ~~未花完金币 1:1 转碎片~~ → **删除**
> - ~~主菜单卡池永久解锁（50-500 碎片/张）~~ → **删除**，所有卡开局即解锁
> - ~~主菜单永久升级（水晶 HP / 手牌上限 / 能量上限）~~ → **删除**，全部固定锁定
>
> v3.4 单 Run 闭环下不存在 meta 持久资源。**§4 本节定义本 Run 临时技能点 SP**，与 50-mda §17 镜像。

### 4.1 SP 核心属性

| 属性 | 值 | 说明 |
|------|------|------|
| **作用范围** | 本 Run | 跨关卡累积，但 Run 结束清零 |
| **Run 起始 SP** | 0 | 不可通过任何 meta 通道继承（v3.4-INV-03） |
| **持久性** | Run 内累积；**Run 结束清零** | 无 meta 存档 |
| **使用方向** | **不可逆消耗**：投入技能树节点 | 不可退还、不可兑换金币 |

### 4.2 SP 获取流量

| 来源 | 数值 | 全 Run 累积 |
|------|------|------------|
| **关卡通关奖励** | 关 N × 2 SP（关 1 = 2 SP / 关 8 = 16 SP） | 72 SP（关 1-8 线性累加） |
| **终战通关额外** | +20 SP（不是 18，突出终极感） | 92 SP 累计（含终战） |
| **秘境事件 SP 奖励**（50-mda §17.2.2） | 5 / 15 / 30 / 50 SP 分档（权重 40/35/20/5） | 期望 25-50 SP / Run |
| **商店金币兑换**（§3.5） | 50 G / 1 SP，单商店上限 3 SP | 期望 5-15 SP / Run |
| **全 Run 流量** | 商店党 100-112 SP / 秘境党 120-140 SP / 平衡型 110-130 SP | 详见 50-mda §17.2.4 |

### 4.3 SP 消耗：技能树节点

**节点单价**（按 depth 递增，50-mda §17.3）：

| depth | SP 单价 | 节点效果定位 |
|-------|---------|------------|
| 1 | 3 SP | 基础强化（如箭塔 +20% ATK） |
| 2 | 6 SP | 中级特性（如箭塔 multi_shot） |
| 3 | 10 SP | 高级特性（如电塔 chain_jumps） |
| 4 | 15 SP | 终极变招（如电塔 chain_amplify） |

**单塔满级估算**：
- 普通塔（3 节点）= 3+6+10 = 19 SP
- 电塔（4 节点）= 3+6+10+15 = 34 SP

→ 玩家 Run 内可「满级 1 塔 + 半级 1 塔」，符合「本 Run 实质性升级 1-2 个塔关键节点」目标。

### 4.4 SP 路径切换规则

| 维度 | 规则 |
|------|------|
| **路径切换** | **0 SP 自由切换**（任意时刻在已点节点间切换主路径） |
| **路径重置** | **不可重置**（已花的 SP 不可退还，v3.4 无 meta 资源 → 无返还概念） |
| **流派引导** | 鼓励"先点 1 条路径打到底"的流派策略 |

详见 [50-mda §17.4](../50-data-numerical/50-mda.md#174-路径切换与重置v34) + 第 3 轮 22-skill-tree-* 文档（待新建）。

## 7. 关内回收机制（v3.4 保留）

| 维度 | v3.4 规则 |
|------|----------|
| **建造时扣** | 仅能量 E（金币不参与关内消耗） |
| **回收返还** | 能量 E（不返还金币） |
| **回收公式** | `返还能量 = 0.5 × 该卡能量消耗 × (currentHP / maxHP)` |
| **建造冷却** | 3 秒内不可回收 |
| **战斗保护** | 受伤/攻击中 2 秒内不可回收 |

**v1.0 → v3.4 处理**（清理冗余对比）：

| 旧机制 | v3.4 处理 |
|--------|----------|
| 战斗中升级塔 | **删除**：升级路径改为本 Run 技能树（关后节点） |
| 战斗中建造扣金币 | **仅扣能量 E**（关内金币只攒不花） |
| 战斗中回收建筑 | **保留**：返还能量 E，不返还金币 |
| 战斗中购买技能 | **删除** |

详细公式见 [50-mda §12.3](../50-data-numerical/50-mda.md#123-能量回收公式)。

## 8. 已删除机制（v3.4 形态）
