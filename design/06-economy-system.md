## 2. 资源流入流出（规则）

### 2.1 流入途径（具体数值见 21-MDA §3/§4/§9）

| 维度 | 流入类型 |
|------|----------|
| 主动击杀（瞬时） | 击杀敌人单位 → 单位 `bounty` 配置值 |
| 被动产出（持续） | 金矿/能量塔每秒产出，由建筑等级决定 |
| 阶段奖励（波次结束） | 完成波次时一次性奖励 |
| 随机事件 | 宝箱、关卡道具掉落 |

### 2.2 流出途径

| 用途 | 消耗规则 |
|------|----------|
| 建造塔/单位/建筑 | 一次性扣金币，按 21-MDA 配置 |
| 升级塔 | 一次性扣金币（升级费），等级越高越贵 |
| 修复建筑 | 按"满血/损血"比例计算金币（如损失 60%HP → 修复费用 = 原价 × 0.6 × 修复系数） |
| 升级人口上限 | 一次性扣金币 |
| 释放技能 | 一次性扣能量 |

## 3. 生产建筑链

```
金矿(L1) → 金矿(L2) → 金矿(L3)
能量塔(L1) → 能量塔(L2) → 能量塔(L3)
```

| 规则 | 说明 |
|------|------|
| 升级方式 | 与塔相同，点击建筑 → 升级按钮 |
| 产出周期 | 每秒固定金币/能量（数值见 21-MDA） |
| 可被攻击 | ✅ 敌人可摧毁，HP 由 21-MDA 定义 |
| 占地格位 | 与塔同（空地） |
| 回收 | 同塔回收规则（见 §4） |

## 4. 回收机制（P2-#15 修复 v1.1）

> **旧版问题**：旧 "回收实体返还 50% 总投资" 表述模糊：(1) 升级费算不算总投资？(2) 是否有冷却保护？(3) 是否防止建造后立即拆除滥用？

### 4.1 回收规则

| 维度 | 规则 |
|------|------|
| **回收返还比例** | 累计造价 × `refundRatio`（默认 0.5；具体数值见 21-MDA） |
| **累计造价定义** | **建造费 + 所有升级费之和**（明确包含升级费） |
| **HP 折扣** | 若回收时 HP < 100%，按 HP 比例线性衰减返还：`refund = totalCost × refundRatio × (currentHP / maxHP)` |
| **冷却保护** | 建造后 **3 秒**内不可回收（防止误触误操作 + 防滥用刷资源） |
| **战斗中保护** | 该建筑/塔在最近 2 秒内**有受到伤害**或**正在攻击**时，不可回收（防"摸鱼"刷取） |
| **建造立刻拆退款率** | 建造 < 5 秒（已过冷却）且未参与任何战斗 → 返还 90%（高比例退款，承认误建） |
| **特殊单位** | 路障/中立机关有独立 `refundRatio` 配置（见 22 §4.4） |

### 4.2 回收 UI

- 选中建筑后，UI 显示：
  - 当前预计返还金币（按 §4.1 规则实时计算）
  - 冷却剩余时间（如有）
  - 战斗保护提示（如有）

### 4.3 实现约定

```typescript
function calculateRefund(entity: BuildEntity): number {
  if (entity.timeSinceBuild < 3) return 0;             // 建造冷却
  if (entity.timeSinceLastDamage < 2) return 0;        // 战斗保护
  if (entity.timeSinceLastAttack < 2) return 0;        // 战斗保护
  
  if (entity.timeSinceBuild < 5 && !entity.everInCombat) {
    return entity.totalCost * 0.9;                     // 误建撤回
  }
  
  const hpRatio = entity.currentHp / entity.maxHp;
  return entity.totalCost * entity.refundRatio * hpRatio;
}
```

## 5. 经济节奏（设计意图）

| 阶段 | 特征 | 玩家决策 | 设计目标 |
|------|------|----------|----------|
| **前期**（关卡前 20%） | 金币紧张，无法同时铺塔 + 经济 | "经济 vs 防御" — 先铺金矿还是先造塔？ | 关键抉择期 |
| **中期**（20%-60%） | 经济建筑开始回本 | "滚雪球" — 升级经济还是造更多攻击塔？ | 多策略可行 |
| **后期**（60%-100%） | 资源溢出 | 全局 Buff 提升，应对高强度波次 | 玩家充分表达建造意图 |

经济节奏校验：见 21-MDA §10 经济节奏验证表（各关卡每波平均收入 vs 平均必要支出对照）。

## 6. 人口系统（P2-#28 v1.1 细节）

| 维度 | 规则 |
|------|------|
| **初始上限** | 关卡参数决定（见 21-MDA §6） |
| **升级方式** | 任意时刻可花金币升级 + N（N 由 21-MDA 决定） |
| **人口消耗** | 每个我方移动单位占用 1 人口（远程兵 1，BattleMage/旗手等高级单位 2） |
| **死亡释放** | 单位死亡时立即释放人口 |
| **超出上限处理** | 不可建造，按钮变灰显示"人口不足"，不消耗资源 |
| **升级硬上限** | 12（与无尽模式上限一致，防止刷兵海） |

## 7. 经济异常防护（P2-#28 v1.1 新增）

### 7.1 反滥用规则

| 风险场景 | 防护机制 |
|----------|----------|
| **快速建造-拆除-建造** | 同位置 5 秒冷却（拆除后该格位 5 秒内不可建同类） |
| **战斗期刷取** | §4.1 战斗保护（受伤/攻击中 2 秒内不可回收） |
| **金矿被攻击立刻拆除拿钱** | §4.1 HP 折扣（残血回收损失最大化） |
| **回收返还利用率攻击** | 三色冷却综合保护（建造冷却 + 战斗保护 + HP 折扣） |
| **死循环造金币** | 经济类塔（金矿/赏金塔）有"该波次最多 N 个"配置（见 21-MDA） |

### 7.2 经济上限保护

- 单局金币上限：999,999（防整数溢出 & UI 显示）
- 单局击杀奖励显示上限：99（单次击杀奖励 cap）
