# 09 — UI/UX设计

> 横屏布局 | 符号优于文字 | 一眼识别

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **符号优于文字** | 场景中不显示单位名称。用形状、动画、特效让玩家一眼识别 |
| **图标化工具栏** | 底部按钮用图形符号而非文字，紧凑且直观 |
| **按需显示信息** | 选中实体时才弹出详情面板；HUD只展示必要数字 |
| **拖拽操作** | 建造/部署都用拖拽，直观的物理感 |

### UI 死区规范（P2-#20 修复 v1.1）

> **旧版问题**：旧代码 `main.ts` 中"x < 160 / y < 60 不路由建造点击"硬编码在事件分发处，新增 UI 区域时容易遗漏。需要规范化为 UI 区域配置。

#### UI 死区 = 玩家点击不应触发地图建造的区域

死区由 UI 区域的实际位置和尺寸**动态计算**，不允许硬编码：

```typescript
interface UIDeadZone {
  region: string;          // 区域名（如 'top_hud', 'bottom_panel'）
  hitTest: (x: number, y: number) => boolean;  // 命中测试
}

const UI_DEAD_ZONES: UIDeadZone[] = [
  {
    region: 'top_hud',
    hitTest: (x, y) => y < LayoutManager.scaleY(HUD_HEIGHT),  // 默认 HUD_HEIGHT=60
  },
  {
    region: 'bottom_panel',
    hitTest: (x, y) => y > LayoutManager.viewportH - LayoutManager.scaleY(PANEL_HEIGHT),
  },
  {
    region: 'tooltip_overlay',
    hitTest: (x, y) => SelectedInfoPanel.contains(x, y),  // 选中信息面板动态判定
  },
  // 暂停菜单、结算面板等覆盖层自带"全屏死区"
];

function isInUIDeadZone(x: number, y: number): boolean {
  return UI_DEAD_ZONES.some(zone => zone.hitTest(x, y));
}
```

#### 事件分发流程

```
pointerDown(x, y)
  → if (isInUIDeadZone(x, y)) → 由 UISystem 处理点击
  → else → 由 BuildSystem / SelectSystem 处理（地图建造、单位选中等）
```

#### UI 死区的优先级

1. 模态覆盖层（暂停菜单、胜利/失败面板）→ 100% 拦截所有点击
2. 工具栏按钮（建造按钮、技能图标）→ 100% 拦截
3. HUD（顶部金币/能量条等）→ 仅展示，拦截但不响应
4. 信息面板（选中实体后弹出）→ 在其边界内拦截

### 区域划分与精确尺寸

| 区域 | 内容 | 尺寸 | 位置 |
|------|------|------|------|
| 顶部HUD | 金币/能量/人口/波次/敌军/倒计时/倍速/暂停 | 1920×36px | y=0 |
| 全屏背景 | 天空渐变+远景+云朵+天气粒子+风线+光效 | 1920×1080px | (0,0) 全屏 |
| 地图区域 | 21×9格战场（1344×576px） | 1344×576px | 水平居中(x=288)，垂直居中于HUD与面板之间 |
| 底部工具栏 | 塔/单位/陷阱/生产按钮（80×80px图形符号） | 1344×100px | 水平居中(x=288)，位于地图下方(y=800) |
| 选中信息面板 | 工具栏右侧，显示选中实体详情 | ~300×100px | 工具栏最右区域，按需显示 |

## 3. 交互流程

### 3.1 核心操作

| 操作 | 步骤 | 反馈 |
|------|------|------|
| **建造塔** | 点击工具栏塔图标 → 拖出幽灵预览 → 松手放置 | 合法格绿色边框，非法格红色，建造粒子特效 |
| **部署单位** | 拖拽单位图标到地图合法位置 | 同上 |
| **选中查看** | 点击地图实体 | 底部弹出信息面板（名称/等级/属性/操作按钮） |
| **升级/回收** | 选中实体 → 信息面板操作按钮 | 升级粒子特效 / 回收消散动画 |
| **移动单位** | 拖拽地图上已有单位 | 实体跟随光标，松手落位 |
| **释放技能** | 选中单位 → 底部技能按钮 → 点目标 | 技能特效 |

### 3.2 拖拽建造视觉

- 从按钮拖出时，半透明幽灵预览跟随光标
- 合法格: 绿色边框 + 浅绿填充
- 非法格: 红色边框 + 浅红填充
- 松手: 合法格→建造动画(scale 0→1弹入) / 非法格→取消

### 3.3 选中信息面板

仅选中实体时在底部显示，含：
- 实体图形符号 + 名称
- HP条 + 数值
- 核心属性（ATK/范围/攻速）
- 操作按钮（升级/回收/技能）

未选中实体时，地图区域零文字。

## 6. 场景内精简文字原则

> **v2.3 更新**：保留实体头顶 HUD（血条/名字/等级）作为游戏必要信息。场景主体仍通过形状+颜色+动画+弹道识别，但头顶 HUD 提供精确数值反馈。

| 旧方案 | 新方案 |
|--------|--------|
| 圆形 + "小兵"标签 | 复合几何体（圆体+双眼+描边）+ 步行弹跳动画 + **头顶 HUD（名字/血条）** |
| 圆形 + "箭塔"标签 | 复合几何体（基座+塔身+箭头尖）+ 弹道连射 + **头顶 HUD（名字/等级/血条）** |
| 六边形 + "基地"标签 | 复合几何体（主体六边形+护盾光环+旗帜）+ 脉动 + **头顶 HUD（名字/血条）** |
| 圆形 + "BOSS"标签 | 复合几何体（大体圆+皇冠+光环+披风）+ **头顶 HUD（名字/BOSS血条）** |

玩家通过**形状+颜色+动画+弹道**四要素组合快速识别单位类型，配合**头顶 HUD** 获取精确状态信息。

### 6.1 实体头顶 HUD 规范

每个实体头顶显示三层信息，垂直堆叠排列：

```
         ◆ ◆ ◆          ← 等级指示器（仅塔/可升级单位）
        箭塔              ← 名字标签（12px, 白色, 居中对齐）
   ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓     ← 血条（始终可见，颜色按HP比例变化）
    ╔══════════╗
    ║  实体图形  ║         ← 实体主体（形状+颜色+动画）
    ╚══════════╝
```

| 元素 | 位置 | 尺寸/样式 | 显示条件 |
|------|------|----------|----------|
| **血条** | 实体底部下方 2px | 宽=实体尺寸×1.2（最小28px），高=6px；底色#222，填充色按HP比例：绿(>60%)/黄(30-60%)/红(<30%) | 始终显示（满血也显示） |
| **名字标签** | 血条上方 6px | 字体 12px，#ffffff，居中对齐 | 始终显示，内容来自单位配置的 `name` 字段 |
| **等级指示器** | 名字上方 4px | 金色小菱形（6px），每个代表 1 级，水平排列 | 仅塔和可升级单位显示；等级=1时不显示 |

> **P0-#2 修复说明（v1.1）**: 头顶 HUD 的等级菱形是**辅助识别**手段，不是唯一识别手段。塔的等级主要通过**塔底纹章 + 塔顶颜色 + 辉光强度**识别（详见 [16-art-assets §12.1.1b 多维等级标识系统](./16-art-assets-design.md#1211b-多维等级标识系统p0-2-修复)），头顶菱形仅作为精确数值的补充展示。
>
> 这样做的好处：
> 1. 远距离（无法看清头顶 HUD 时）也能识别等级
> 2. 缩略图/小尺寸视图（如建造面板预览）下识别一致
> 3. 颜色弱视玩家可通过纹章尺寸（12→14→16→18px）和辉光强度识别

### 6.2 各实体类型的 HUD 显示

| 实体类型 | 血条 | 名字 | 等级 |
|----------|------|------|------|
| **塔** | ✅ 始终显示 | ✅ 塔配置名（箭塔/炮塔/冰塔…） | ✅ 金色菱形（Lv1不显示） |
| **敌方单位** | ✅ 始终显示 | ✅ 敌人配置名（小兵/快兵/重装兵…） | ❌ 不显示等级 |
| **BOSS** | ✅ 始终显示（加宽，×1.5） | ✅ BOSS 配置名 | ❌ 不显示等级 |
| **我方士兵** | ✅ 始终显示 | ✅ 士兵配置名（盾卫/剑士/弓手…） | ✅ 如有等级则显示 |
| **陷阱** | ❌ 不显示（1HP不可摧毁） | ✅ "地刺" | ❌ 不显示等级 |
| **生产建筑** | ✅ 始终显示 | ✅ 建筑配置名（金矿/能量塔） | ✅ 金色菱形 |
| **基地/出生点** | ✅ 始终显示 | ✅ 配置名 | ❌ 不显示等级 |
| **弹道/特效** | ❌ 不显示 | ❌ 不显示 | ❌ 不显示 |
