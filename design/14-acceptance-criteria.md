## 1. 功能验收标准

| 等级 | 标准 |
|------|------|
| **通过** | 功能按设计运行，无阻塞性 Bug，核心路径无异常 |
| **需修改** | 功能基本运行但存在体验问题或边界情况未处理 |
| **不通过** | 功能不可用 / 严重偏离设计 / 存在数据错误 |

## 2. 各阶段验收门禁

### MVP（Phase 1）✅

- 浏览器中可完成完整一局 Run（从关 1 打到任意 fail / 通关任一关）
- 1 种塔卡 + 1 种单位卡 + 1 种法术卡 + 3 波 + 1 张地图
- 核心闭环：抽卡组 → 出卡部署 → 战斗 → 关卡通过/失败

### 系统完善（Phase 2）

- 完整卡牌系统：抽卡组、抽手牌、出卡部署、弃牌堆、洗牌
- 完整能量系统：每波 +5、上限 10、跨波保留
- 完整金币系统：击杀掉落、波次结算、关间消费
- 关间节点系统：商店（4 商品 + 移除 + 刷新）、秘境（12 事件池）
- 水晶 HP 跨关继承，水晶无敌但秒杀进入攻击范围的敌人（每杀 -1 HP），HP=0 水晶碎裂失败
- AI 行为树驱动所有单位（含 v3.0 新增敌人类型）
- 技能系统（卡片附加 buff、双轨临时升级）

### 内容填充（Phase 3）

- 8 关 + 终战完整可玩，主题视觉区分明显
- 卡池含 ≥ 30 张卡（覆盖 Common/Rare/Epic/Legendary 全部稀有度）
- 关间存档点：浏览器关闭重开后能继续 Run
- 永久卡池界面：解锁/升级卡可正常消费火花碎片
- 永久升级面板：水晶 HP / 能量上限 / 手牌上限可解锁
- 流派识别正常（结算面板显示流派标签）
- 天气系统视觉和数值生效

### 打磨（Phase 4）

- 60fps 稳定（含全屏装饰 + 拖卡动画）
- 音效播放无延迟/卡顿（卡牌系统所有音效齐全）
- 新手引导覆盖核心操作（抽卡 → 拖卡部署 → 关间决策）
- 所有已知 Bug 修复
- **响应式布局**：1280×720、1920×1080、2560×1440、3440×1440 (21:9) 下表现正常
- 手牌区缩放适配低分辨率

### 3.6 水晶机制与 HP 继承

> 水晶（前称大本营）是玩家方的核心防御实体。详细机制见 [25 §6.2](./25-card-roguelike-refactor.md#62-水晶机制核心防御实体)。

#### 6a. 水晶基础与 HP 继承

- [ ] Run 开始时 HP = `permanentUpgrades.baseHpMax`（默认 1000）
- [ ] 关卡通过后 HP 保留（不重置）
- [ ] HP ≤ 0 → 触发水晶碎裂动画 → Run 失败 → 从第 1 关重开
- [ ] 商店血药 / 秘境事件可补血（不超过最大 HP）
- [ ] HUD 显示 💎 当前/最大，HP < 30% 闪烁红色 + 辉光呼吸频率加快
- [ ] 水晶占用地图左侧 1 格固定位置，不可部署 / 移除 / 移动

#### 6b. 无敌规则（免疫伤害）

- [ ] 敌人普攻、AOE、毒、点燃、自爆、反弹伤害**均不能扣除水晶 HP**
- [ ] Boss 必杀技、终战 Boss 全屏攻击**均不能扣除水晶 HP**
- [ ] 主动负向操作（秘境献祭 / 黑市卖血等）仍能正常扣 HP（属于玩家自愿付出的代价）

#### 6c. 秒杀攻击

- [ ] 敌人进入水晶 `attackRange` 范围内立即被秒杀（动画 + 音效）
- [ ] 秒杀无视目标 HP、护盾、无敌、霸体、闪避等所有防御机制
- [ ] 每秒杀 1 个敌人，水晶 HP -1（按 `hpCostPerKill`）
- [ ] 攻击冷却 `attackCooldown` 期间，新进入范围的敌人排队等待（按最近距离选择）
- [ ] Boss（携带 `immune_to_crystal_kill = true` 标记）进入范围时**不被秒杀也不消耗 HP**，必须用其他手段处理
- [ ] 终战 Boss `abyss_lord` 必须带 `immune_to_crystal_kill` 标记

#### 6d. 视觉表现

- [ ] 红色水晶（菱形复合几何体）悬浮在底座上方，上下浮动循环
- [ ] 水晶辉光亮度按呼吸频率周期变化
- [ ] 秒杀触发时水晶向目标方向发出红色光束 + 目标位置出现红色爆裂粒子
- [ ] 水晶 HP=0 时触发碎裂动画：水晶分裂为多个碎片 → 向上飘散 → 透明度渐变到 0 → 完全消失
- [ ] 详细视觉规范见 [16-art §13.13](./16-art-assets-design.md#1313-水晶大本营视觉) 与 [12-vfx §10](./12-visual-effects.md#10-水晶视觉与特效v30-新增)

### 3.7 关间节点：商店

- [ ] 关 1-7 通过后弹出二选一面板
- [ ] 选择商店进入商店界面
- [ ] 商店包含 4 件商品：新卡 / 升级卡 / 移除卡 / 血药

## 5. 体验验收标准

| 指标 | 目标 | 底线 |
|------|------|------|
| 帧率 | 60fps | 30fps |
| UI 响应延迟 | <100ms | <200ms |
| 核心操作步数 | ≤2 步 | ≤3 步 |
| 字体可读性 | 清晰可辨 | 可辨认 |
| 玩家识别度（敌人 60px） | 2 秒 90% 准确率 | 3 秒 80% 准确率 |
| Run 平均时长 | 30-45 分钟 | 20-60 分钟 |

### 5.1 核心操作效率

| 操作 | 目标步数 |
|------|---------|
| 出卡部署 | 2（拖卡+松手） |
| 释放法术卡 | 2（拖卡+点目标） |
| 回收建筑 | 2（选建筑→点回收） |
| 关间二选一 | 1（点击商店或秘境） |
| 商店购卡 | 1（点击商品） |
| 查看资源 | 0（始终可见） |

## 6. 测试策略

| 测试类型 | 覆盖范围 | 工具 |
|----------|---------|------|
| **单元测试** | 伤害公式、能量计算、卡牌抽取、PRNG | Vitest |
| **集成测试** | 系统间交互（手牌→部署→战场；商店→卡组更新；秘境事件） | Vitest |
| **回归测试** | v3.0 新增设计的所有验收项（§3） | Vitest + 手动 |
| **手动测试** | Run 完整流程、关间节点、卡池界面 | 浏览器 |
