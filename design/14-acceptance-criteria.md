# 16 — 验收标准

> 功能验收、体验验收、各阶段门禁

---

## 1. 功能验收标准

| 等级 | 标准 |
|------|------|
| **通过** | 功能按设计运行，无阻塞性Bug，核心路径无异常 |
| **需修改** | 功能基本运行但存在体验问题或边界情况未处理 |
| **不通过** | 功能不可用 / 严重偏离设计 / 存在数据错误 |

---

## 2. 各阶段验收门禁

### MVP（Phase 1）✅

- 浏览器中可完成完整一局游戏
- 1种塔 + 1种敌人 + 3波 + 1张地图
- 核心闭环: 部署 → 战斗 → 结算 → 胜负

### 系统完善（Phase 2）

- 6种攻击塔、5种敌人、6种可移动单位
- 敌人反制AI生效（攻击建筑/适应性出兵）
- 技能系统（主动/被动）可运行
- Buff叠加与显示正确
- 生产建筑产出正确
- BOSS阶段转换正确

### 内容填充（Phase 3）

- 5个完整关卡可玩，主题视觉区分明显
- 无尽模式波次递增正确，排行榜记录
- 存档/读档无数据丢失
- 关卡选择界面流畅
- 天气系统视觉和数值生效

### 打磨（Phase 4）

- 60fps稳定
- 音效播放无延迟/卡顿
- 新手引导覆盖核心操作
- 所有已知Bug修复

---

## 3. 体验验收标准

| 指标 | 目标 | 底线 |
|------|------|------|
| 帧率 | 60fps | 30fps |
| UI响应延迟 | <100ms | <200ms |
| 核心操作步数 | ≤2步 | ≤3步 |
| 字体可读性 | 清晰可辨 | 可辨认 |

### 核心操作效率

| 操作 | 目标步数 |
|------|----------|
| 建造塔 | 2（拖拽+松手） |
| 部署单位 | 2（拖拽+松手） |
| 释放技能 | 2（选单位→点技能目标） |
| 查看资源 | 0（始终可见） |

---

## 4. 测试策略

| 测试类型 | 覆盖范围 | 工具 |
|----------|----------|------|
| 单元测试 | 伤害公式、数值计算、工具函数 | Vitest |
| 集成测试 | 系统间交互（Attack→Projectile→Health） | Vitest |
| 手动测试 | UI交互、游戏手感、视觉反馈 | 浏览器 |
