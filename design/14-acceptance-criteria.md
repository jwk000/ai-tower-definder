## 1. 功能验收标准

| 等级 | 标准 |
|------|------|
| **通过** | 功能按设计运行，无阻塞性 Bug，核心路径无异常 |
| **需修改** | 功能基本运行但存在体验问题或边界情况未处理 |

## 2. 各阶段验收门禁

### MVP（Phase 1）✅

- 浏览器中可完成完整一局游戏
- 1 种塔 + 1 种敌人 + 3 波 + 1 张地图
- 核心闭环：部署 → 战斗 → 结算 → 胜负

### 系统完善（Phase 2）

- 7 种攻击塔、9 种敌人（含飞行敌/分散敌）、6 种可移动单位
- 敌人反制 AI 生效（攻击建筑 / 适应性出兵）
- 技能系统（主动/被动）可运行
- Buff 叠加与显示正确（含 Buff 上限）
- 生产建筑产出正确
- BOSS 阶段转换正确

### 内容填充（Phase 3）

- 5 个完整关卡可玩，主题视觉区分明显
- 无尽模式波次递增正确，**随机种子可复现**
- 存档/读档无数据丢失，**版本迁移可恢复**
- 关卡选择界面流畅
- 天气系统视觉和数值生效

### 打磨（Phase 4）

- 60fps 稳定（含全屏装饰）
- 音效播放无延迟/卡顿
- 新手引导覆盖核心操作
- 所有已知 Bug 修复
- **响应式布局**：1280×720、1920×1080、2560×1440、3440×1440 (21:9) 下表现正常

## 3. v1.1 新增设计验收清单（P3-#26 修复）

### 3.1 数值真理源（P0-#1）

- [ ] 所有数值（单位/塔/敌人/波次/经济）只在 21-MDA 中维护
- [ ] 03/05/06/07 等文档无残留具体数值（仅字段语义/公式骨架）
- [ ] 代码 `src/data/gameData.ts` 数值与 21-MDA 完全一致
- [ ] 数值变更测试：修改 21 中某项 → 自动反映到游戏（无需改其它文档）

### 3.2 单位识别度（P0-#2）

- [ ] 9 种敌人在 60px 缩略尺寸下 2 秒内可分类（玩家测试，90% 准确率）
- [ ] 6 种我方单位在 40px 移动中 2 秒内可识别（玩家测试，90% 准确率）
- [ ] 塔等级 L2-L5 通过纹章颜色 + 塔顶强调色 + 辉光 1 秒内可识别（不依赖头顶菱形）
- [ ] 颜色弱视玩家可通过纹章形状/尺寸/辉光强度识别等级

### 3.3 BT 节点接口冻结（P0-#3）

- [ ] BT 节点签名与 23 §0 完全一致
- [ ] 未实现节点有 STUB 占位（不可静默忽略）
- [ ] BT 配置中所有节点名都能在 23 §0 找到对应签名
- [ ] AI 配置（14 套）通过 BT 节点签名校验

### 3.4 导弹塔削弱（P0-#4）

- [ ] 导弹塔射程 = 600px（不再是 9999/全图）
- [ ] L5 热压：中心 10% 半径区域 ×1.2 伤害（非"全图"加成）
- [ ] 不命中飞行敌（cantTargetFlying = true）
- [ ] 不命中分散敌（除非密度满足）
- [ ] L4 引入飞行敌、L6 引入分散敌
- [ ] UI 显示 600px 射程圈

### 3.5 Buff 上限（P1-#8）

- [ ] 单实体总 Buff 数量 ≤ 8（含层数）
- [ ] 超上限时按 LRU 淘汰最早 Buff
- [ ] 玩家技能 Buff 受优先保留
- [ ] Buff 优先级生效：眩晕 > 嘲讽 > 减速 > DOT > 增益 > 标记
- [ ] 来源死亡按规则清除（持续型/一次性/嘲讽各自处理）

### 3.6 士兵 AI 状态机（P1-#10）

- [ ] 远程兵 alert_range ≥ attack_range × 1.1 + 20
- [ ] 状态优先级生效：COMBAT > ALERT > RETURN > IDLE
- [ ] 超界保护：目标超出 max_range 时自动重选
- [ ] 多士兵不扎堆攻击同一弱敌（attackerCount 加权）
- [ ] 软碰撞防抖：personalSpace 维护，无 < 2px 抖动
- [ ] 战斗位置死锁解除（2 秒内无位移自动随机偏移）

### 3.7 路障/弩炮塔拆分（P1-#11）

- [ ] 路障数据使用 `StructureConfig`，与 `TowerConfig` 分离
- [ ] 路障无攻击逻辑（仅反伤被动）
- [ ] 路障占据格位 + 改变寻路 + 飞行敌豁免
- [ ] 弩炮塔弹道层级：不打 LowAir、穿过友军/装饰物不消耗贯穿数
- [ ] 弩炮塔被路障阻挡 + 命中地形墙

### 3.8 难度曲线（P2-#12）

- [ ] 关卡内 4 阶段难度系数：开局 0.8× / 建设 1.0× / 压力 1.3× / 高潮 1.5×
- [ ] 关卡间难度递增使用 `LEVEL_DIFFICULTY × WAVE_DIFFICULTY` 公式
- [ ] 玩家成长校验：每个关键时刻可达成对应塔等级

### 3.9 三星评定（P2-#13）

- [ ] 三星独立可累计（不互斥）
- [ ] ★1 = 通关、★2 = 基地 HP ≥ 80%、★3 = 时间 ≤ targetTime
- [ ] 单项最佳永不降级（重玩取每项历史最佳）

### 3.10 无尽模式（P2-#14）

- [ ] 后期规则池 8 种规则，按权重随机抽取
- [ ] 随机种子保存到存档，可复现
- [ ] 评分公式包含 Boss 击杀 + 无伤波

### 3.11 经济回收（P2-#15）

- [ ] 累计造价 = 建造费 + 所有升级费
- [ ] HP 折扣：refund = totalCost × refundRatio × (hp/maxHp)
- [ ] 建造冷却 3 秒
- [ ] 战斗保护 2 秒（受伤/攻击中不可回收）
- [ ] 误建撤回 90% 退款（5 秒内未参战）

### 3.12 随机种子（P2-#16）

- [ ] 禁用 Math.random()（代码扫描验证）
- [ ] 使用 mulberry32 或同等确定性 PRNG
- [ ] 多流隔离：map/wave/drop/decor 各自独立流
- [ ] UI 提供"上次种子"复现选项

### 3.13 存档版本兼容（P2-#17）

- [ ] CRC32 校验和验证存档完整性
- [ ] 版本迁移注册表覆盖 v1.0 → v1.1
- [ ] 损坏存档可从备份恢复（session/daily/version/manual）
- [ ] 重置前自动创建 manual backup

### 3.14 图层与跨层级（P2-#18）

- [ ] 蝙蝠/飞行敌渲染层级正确（z=6 LowAir）
- [ ] 近战地面单位不攻击飞行敌
- [ ] 地刺不触发飞越的飞行敌
- [ ] 飞行 Boss 不触发地面陷阱

### 3.15 塔射程可视化（P2-#19）

- [ ] 建造预览：虚线圆环 + 塔主色
- [ ] 选中已建塔：实线圆环
- [ ] 升级预览：双圈（当前级 + 下一级）
- [ ] 弩炮塔显示直线弹道扇形
- [ ] AOE 塔显示嵌套双圈（爆炸半径 + 主射程）

### 3.16 UI 死区（P2-#20）

- [ ] 死区由 UIDeadZone 动态计算，无硬编码
- [ ] 模态覆盖层 100% 拦截
- [ ] 工具栏/HUD/信息面板边界正确

### 3.17 装饰物边界（P3-#22/P3-#27）

- [ ] 装饰物不影响游戏逻辑（不触发陷阱、不阻挡寻路、不被攻击）
- [ ] 低性能设备自动降级（隐藏动态生物 + 减少粒子）
- [ ] 装饰物超出视口被裁剪

### 3.18 音效完整性（P3-#25）

- [ ] v1.1 新增单位/塔的音效 ID 完整（飞行敌/分散敌/号令塔/图腾塔/毒藤塔/弩炮塔/赏金塔/路障）
- [ ] 节流保护：高频音效（塔射击/敌人受击）≥ 20ms 间隔
- [ ] 优先级剔除：同时播放 ≤ 12 个 SFX + 2 个环境 Loop + 1 个 BGM

## 4. 体验验收标准

| 指标 | 目标 | 底线 |
|------|------|------|
| 帧率 | 60fps | 30fps |
| UI 响应延迟 | <100ms | <200ms |
| 核心操作步数 | ≤2 步 | ≤3 步 |
| 字体可读性 | 清晰可辨 | 可辨认 |
| 玩家识别度（敌人 60px） | 2 秒 90% 准确率 | 3 秒 80% 准确率 |

### 核心操作效率

| 操作 | 目标步数 |
|------|----------|
| 建造塔 | 2（拖拽+松手） |
| 部署单位 | 2（拖拽+松手） |
| 释放技能 | 2（选单位→点技能目标） |
| 回收建筑 | 2（选建筑→点回收） |
| 查看资源 | 0（始终可见） |

## 5. 测试策略

| 测试类型 | 覆盖范围 | 工具 |
|----------|----------|------|
| **单元测试** | 伤害公式、数值计算、工具函数、Buff 优先级 | Vitest |
| **集成测试** | 系统间交互（Attack → Projectile → Health；BT 节点执行） | Vitest |
| **回归测试** | v1.1 新增设计的所有验收项（§3） | Vitest + 手动 |
| **手动测试** | UI 交互、游戏手感、视觉反馈、识别度 | 浏览器 |
| **性能测试** | 60 FPS 在不同分辨率/装饰强度下保持 | DevTools Performance |
| **存档测试** | 版本迁移、损坏恢复、跨浏览器 | 手动构造测试存档 |
