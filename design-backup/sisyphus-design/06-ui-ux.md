### 区域划分

| 区域 | 位置 | 尺寸 | 内容 |
|------|------|------|------|
| 顶部 HUD | 上 | 1920×80 | 资源栏 (左) + 波次信息 (右) |
| 建造面板 | 左 | 192×840 | 可建造塔列表 + 金币消耗 |
| 地图区域 | 中 | ~1536×840 | 30×17 格战场 |
| 单位面板 | 右 | 192×840 | 可部署单位列表 |
| 底部信息栏 | 下 | 1920×120 | 选中目标详情 / 升级按钮 |

### Phase 1 简化布局

MVP 阶段采用简化版：建造面板在左（x < 160），顶部 HUD（y < 60），无右侧单位面板。

## 2. 交互流程

### 核心操作路径

| 操作 | 步骤 | 限制条件 |
|------|------|----------|
| **建造塔** | 点击左侧塔图标 → 可建位置高亮 → 点击地图放置 | 金币足够 + 目标格为 `empty` |
| **升级塔** | 点击地图上的塔 → 底部显示升级按钮 | Phase 2 实现 |
| **部署单位** | 点击右侧单位头像 → 拖到地图合法位置 | 人口未满 |
| **移动单位** | 拖拽地图上已有单位到新位置 | 战斗中可操作 |
| **释放技能** | 点击单位头像 → 技能按钮 → 选择目标位置 | CD 冷却完毕 |
| **开始波次** | 点击顶部"下一波"按钮 | 上一波结算完成 |

### 输入调度规则

- `InputManager` 每帧 `flush()` 事件队列
- 点击事件先检查 UI 按钮区域，再检查地图建造位置
- 触屏与鼠标事件统一抽象为 `InputAction`
- 当 `x < 160`（建造面板）或 `y < 60`（HUD）时不路由到地图

## 3. 几何美术风格规范

### 设计原则

**"纯几何图形 + 纯色填充"** 极简风格，无渐变无纹理。

### 形状-实体映射

| 游戏实体 | 几何形状 | 颜色 | 文字标签 | 尺寸 |
|----------|----------|------|----------|------|
| 塔 (箭塔) | 圆圈 | 蓝色 | "箭塔" | r=24px |
| 敌人 (小兵) | 圆圈 | 红色 | "小兵" | r=16px |
| 基地 | 六边形 | 蓝灰 | "基地" | r=28px |
| 我方单位 | 正方形 + 内嵌十字 | 绿色 | (类型名) | 24px |
| BOSS | 大圆 | 深红 | (BOSS名) | r=36px |
| 远程敌人 | 六边形 | 橙色 | (类型名) | r=20px |
| 弹道 | 箭头 (箭杆+三角箭头) | 蓝色 | - | 动态 |

### 地形色彩

| 地形 | 颜色 | Hex |
|------|------|-----|
| 空地 (森林) | 亮绿 | `#3a7d44` |
| 路径 | 灰褐 | `#a1887f` |
| 障碍 | 深灰 | `#5d4037` |
| 基地格 | 蓝灰 | - |
| 出生点 | 红棕 | - |

### 特效

- **粒子**: 小圆点飞散（攻击命中 / 死亡）
- **颜色变化**: 受击闪白 100ms，Buff 闪绿
- **血条**: 3px 高，偏移 -14px，绿(>60%)→黄(30-60%)→红(<30%)

### 配色约束

- 主体色 ≤ 5 种，避免视觉噪音
- 团队色：我方蓝色系，敌方红色系
- 高对比度底色，保证可读性

## 4. UI 渲染

### 层级顺序

1. 地图格子 (底色)
2. 实体图形 (圆圈/六边形/箭头)
3. 文字标签 (实体名称)
4. 血条 (每个实体上方)
5. UI 面板 (建造面板、HUD、按钮)
6. 文本叠加层 (`onPostRender`)

`UISystem` 注册在 `RenderSystem` 之后，确保 UI 始终在场景上层。
