# 09 — 开发计划

> 来源: GDD §9 | 版本: v0.2

## 总览

| 阶段 | 名称 | 目标 | 预计迭代 |
|------|------|------|----------|
| **Phase 1** | MVP 核心闭环 | 1张地图跑通完整单局 | 3-5 轮 ✅ 已完成 |

## Phase 1 — MVP 核心闭环 ✅

**目标**: 1张地图跑通完整单局循环

| 序号 | 任务 | 验收标准 | 状态 |
|------|------|----------|------|
| 1.1 | 项目脚手架 (Vite+TS+Canvas) | `npm run dev` 看到 Canvas 画面 | ✅ |
| 1.2 | ECS 框架核心 | Entity/Component/System 运作 | ✅ |
| 1.3 | 网格地图 + 路径渲染 | 30×17 格，路径可见 | ✅ |
| 1.4 | 箭塔 — 建造/攻击/升级框架 | 点击建造，自动攻击最近敌人 | ✅ |
| 1.5 | 小兵敌人 — 路径移动/受击/死亡 | 敌人走完路径，受击掉血死亡 | ✅ |
| 1.6 | 波次系统 | 3 波敌人自动依次生成 | ✅ |
| 1.7 | 基础经济 | 杀敌+10金，造塔-50金，实时显示 | ✅ |
| 1.8 | 胜负判定 | 基地归零→失败；波次清除→胜利 | ✅ |
| 1.9 | 极简 UI | HUD + 建造面板 + 波次按钮 | ✅ |

### MVP 新增（视觉重构 2026-05-07）

| 任务 | 说明 |
|------|------|
| 弹道系统 | `Projectile` 组件 + `ProjectileSystem`，箭塔发射蓝色箭头弹道 |
| 文字标签渲染 | 塔:"箭塔"、敌人:"小兵"、基地:"基地" |
| 血条优化 | 3px 高，-14px 偏移，绿→黄→红 梯度色 |
| 箭头形状渲染 | `Renderer.drawArrow()` 箭杆 + 三角箭头 |
| 系统顺序调整 | movement→attack→projectile→wave→health→economy→build→render→ui |

**MVP 验收标准**: 浏览器中完成完整一局（部署塔 → 杀敌 → 过关/失败），1塔 1敌人 3波 1地图。

---

## Phase 2 — 系统完善

**目标**: 完整玩法可玩

| 序号 | 任务 | 说明 |
|------|------|------|
| 2.1 | 新增 3 种攻击塔 | 炮塔、冰塔、电塔（数值 + 渲染） |
| 2.2 | 新增 4 种敌人 | 快兵、重装、法师、自爆虫 |
| 2.3 | 敌人反制 AI | 攻击建筑、路径适应 |
| 2.4 | 可移动单位系统 | 盾卫 + 剑士（拖拽移动、拦截机制） |
| 2.5 | 技能系统 | 主动 / 被动 / 光环技能 |
| 2.6 | Buff 系统 | 增益 / 减益 / 叠加 |
| 2.7 | 生产建筑 | 金矿 / 能量塔 |
| 2.8 | Boss 系统 | 阶段转换 / 多技能 |
| 2.9 | 完善 UI | 单位面板 / 升级面板 / 技能按钮 |

**Phase 2 验收标准**: 多塔多敌人多单位，技能和 Buff 可用，可移动单位操控流畅。

## Phase 3 — 内容填充

**目标**: 5 个完整关卡 + 无尽模式

| 序号 | 任务 | 说明 |
|------|------|------|
| 3.1 | 设计 5 个关卡 | 含主题变化 (森林/沙漠/冰原) |
| 3.2 | 数值平衡 | 多轮调整难度曲线 |
| 3.3 | 关卡选择界面 | 章节选择 + 星数显示 |
| 3.4 | 无尽模式 | 无限波次递增 + Boss 每5波 |
| 3.5 | 存档系统 | LocalStorage 存进度/设置 |
| 3.6 | 中立单位系统 | 陷阱 / 资源点 / 传送门 |
