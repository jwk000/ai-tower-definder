> 来源: GDD §5 | 版本: v0.2

## 1. 伤害公式

### 基础伤害

```
物理伤害 = 攻击力 × (1 - 护甲/(护甲 + 护甲常数))
魔法伤害 = 攻击力 × (1 - 魔抗/(魔抗 + 魔抗常数))

护甲常数 = 100
魔抗常数 = 100
```

### 最终伤害

```
最终伤害 = 基础伤害 × (1 + 伤害加成%) × (1 - 伤害减免%) × 暴击倍率(如果是暴击)

暴击率 = 基础暴击率 + Buff暴击率 (上限80%)
暴击倍率 = 1.5 + Buff暴伤加成
```

### 设计说明

- 护甲/魔抗采用**除法减伤**公式，保证线性堆叠不会导致零伤或免疫
- `护甲常数=100` 意味着：100护甲 → 50%减伤，200护甲 → 66.7%减伤
- 暴击率硬上限 80%，避免暴击满溢导致数值失控

## 2. 攻击速度公式

```
实际攻击间隔 = 基础攻击间隔 / (1 + 攻速加成%)
攻速加成上限 = 200% (即最高3倍攻速)
```

**示例**: 箭塔 L1 基础攻速 1.0/s，叠加 100% 攻速加成后 → 2.0/s

---

## 3. 移动速度公式

```
实际移速 = 基础移速 × (1 + 移速加成%) × (1 - 减速%)
移速下限 = 基础移速 × 20% (不低于20%)
```

**说明**: 减速不可叠加至 0，最低保留 20% 基础移速，防止单位完全凝固。

## 5. 弹道系统 (Phase 1 实现)

> 已实现于当前 Phase 1，替代即时伤害。

- 攻击塔发射**箭矢弹道**（`Projectile` 组件）飞向目标
- 弹道在 `ProjectileSystem` 中更新位置，到达目标后造成伤害
- 箭矢渲染为蓝色箭头形状：箭杆 + 三角箭头，指向目标方向
- `RenderSystem` 自动识别 `Projectile` 组件，将 `Render.shape` 覆盖为 `'arrow'`

**系统顺序**: `MovementSystem → AttackSystem → ProjectileSystem → WaveSystem → HealthSystem → ...`

弹道系统在 AttackSystem 之后、HealthSystem 之前运行，确保弹道在死亡检测前命中。
