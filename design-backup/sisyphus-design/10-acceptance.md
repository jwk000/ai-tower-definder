## 1. 功能验收标准

| 等级 | 标准 |
|------|------|
| **通过** | 功能按设计运行，无阻塞性 Bug，核心路径无异常 |
| **需修改** | 功能基本运行但存在体验问题或边界情况未处理 |
| **不通过** | 功能不可用 / 严重偏离设计 / 存在数据错误 |

### Phase 1 MVP 功能清单（已完成 ✅）

| 功能 | 验收细节 | 状态 |
|------|----------|------|
| 地图渲染 | 30×17 网格，路径/空地/基地可见 | ✅ |
| 箭塔建造 | 点击建造面板 → 地图放置 → 消耗 50 金币 | ✅ |
| 箭塔攻击 | 自动攻击范围内最近敌人，发射弹道 | ✅ |
| 弹道飞行 | 箭头形状，指向目标，命中造成伤害 | ✅ |
| 敌人移动 | 沿路径从 spawn 到 base，速度 80px/s | ✅ |
| 敌人受伤/死亡 | 血量归零消失，击杀 +10 金币 | ✅ |
| 波次系统 | 3 波依次生成，间隔 3s，手动触发下一波 | ✅ |
| 经济系统 | 起始 200 金，余额实时显示 | ✅ |
| 胜负判定 | 基地血量归零→失败；3波全清→胜利 | ✅ |
| UI 面板 | 顶部 HUD (金币+波次)，左侧建造面板 | ✅ |
| 实体标签 | 塔:"箭塔"、敌人:"小兵"、基地:"基地" | ✅ |
| 血条 | 3px高，偏移-14px，绿黄红梯度 | ✅ |

## 2. 代码验收标准

### 必须满足

- [ ] TypeScript `strict` 模式编译通过
- [ ] 无 `any` 类型，无 `@ts-ignore`
- [ ] ECS 系统职责单一（一个 System 只做一件事）
- [ ] 数值计算公式有单元测试覆盖
- [ ] 所有函数明确返回类型
- [ ] ESM 导入使用 `.js` 扩展名

### 代码规范检查

```bash
npm run typecheck    # 必须零错误
npm run build        # 必须成功
```

## 3. 体验验收标准

| 指标 | 目标值 | 底线 |
|------|--------|------|
| 帧率 | 60fps | 30fps (低端设备) |
| UI 响应延迟 | < 100ms | < 200ms |
| 核心操作步数 | ≤ 3 步 | ≤ 5 步 |
| 首局完成率 | > 90% | > 70% |
| 字体可读性 | 清晰 | 可辨认 |

### 核心操作路径效率

| 操作 | 目标步数 | 当前步数 |
|------|----------|----------|
| 建造塔 | 2 (选塔→点地图) | 2 ✅ |
| 开始波次 | 1 (点下一波) | 1 ✅ |
| 查看资源 | 0 (始终可见) | 0 ✅ |

## 4. 各阶段验收门禁

### Phase 1 (已完成)
- 浏览器中可完成完整一局游戏
- 1塔 / 1敌人 / 3波 / 1地图
- 核心闭环：部署→战斗→结算→胜负

### Phase 2 门禁
- 6种塔、5种敌人、2种可移动单位
- 敌人反制AI生效（攻击建筑 / 适应性出兵）
- 技能系统（主动/被动/光环）可运行
- Buff 叠加与显示正确
- 生产建筑（金矿/能量塔）产出正确

### Phase 3 门禁
- 5个完整关卡可玩，主题视觉区分明显
- 无尽模式波次递增正确，排行榜记录
- 存档/读档无数据丢失
- 关卡选择界面流畅

### Phase 4 门禁
- 60fps 稳定（性能分析通过）
- 音效播放无延迟/卡顿
- 新手引导覆盖核心操作
- 所有已知 Bug 修复

## 5. 测试策略

| 测试类型 | 覆盖范围 | 工具 |
|----------|----------|------|
| 单元测试 | 伤害公式、数值计算、工具函数 | Vitest |
| 集成测试 | 系统间交互（Attack→Projectile→Health） | Vitest |
| 手动测试 | UI交互、游戏手感、视觉反馈 | 浏览器 |

```bash
npm test    # 运行所有测试
```
