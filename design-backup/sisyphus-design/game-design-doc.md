## 1. 市场定位与用户画像

### 1.1 市场分析

| 维度 | 分析 |
|------|------|
| **品类现状** | 塔防 (TD) 是成熟品类，但近年缺乏现象级创新。主流产品分两派：Kingdom Rush 式（精致关卡，被动观战）和 Vampire Survivors 式（幸存者like，反向弹幕）。两者之间缺乏"有操作深度但不累"的中间地带 |
| **机会窗口** | 移动端横屏策略游戏供给不足。玩家对"放置类"疲劳，渴望有操作感的轻策略体验 |
| **竞争差异化** | 传统TD = 布阵 + 观战 → **我们的创新** = 布阵 + 实时操作 + 经济链 + 动态对抗 |

### 1.2 目标用户画像

**核心用户 (70%)：**
- 25-35 岁男性，一二线城市
- 策略游戏经验（玩过 Kingdom Rush / 皇室战争 / 明日方舟）
- 单次游戏时长 15-30 分钟
- 在通勤 / 午休 /睡前碎片时间玩
- 追求"有脑但不上头"的体验

**次要用户 (20%)：**
- 18-24 岁学生群体
- 被几何极简美术风格吸引
- 喜欢研究最优解、追求无尽模式高分

**边缘用户 (10%)：**
- 轻度休闲玩家，被简单几何画风吸引
- 需要低门槛入门引导

### 1.3 产品定位一句话

> **"可以亲手操控战场的塔防游戏"** — 传统TD的框架下，加入可移动单位操控、资源链经济和实时技能释放，让每一波敌人都不无聊。

## 2. 核心创新机制

### 2.1 三大创新支柱

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   ┌──────────┐  ┌──────────┐  ┌──────────┐     │
│   │ 可移动   │  │ 资源链   │  │ 敌方     │     │
│   │ 单位操控 │  │ 经济系统 │  │ 反制AI   │     │
│   └────┬─────┘  └────┬─────┘  └────┬─────┘     │
│        │             │             │            │
│   ┌────┴─────────────┴─────────────┴────┐       │
│   │         活战场塔防                   │       │
│   │    (Living Battlefield TD)          │       │
│   └─────────────────────────────────────┘       │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 2.2 创新一：可移动单位操控

- 玩家拥有 2-4 个可移动单位（英雄 + 小兵）
- **拖拽移动**：战斗中可随时重新部署移动单位位置
- **拦截机制**：移动单位挡在路径上可以阻挡敌人（碰撞体积）
- **技能释放**：每个移动单位有 1 个主动技能（点击释放，冷却制）
- **差异化定位**：
  - 肉盾型：高血量 + 嘲讽技能，挡在塔前吸收伤害
  - 输出型：中等血量 + AOE 技能，放在集火区
  - 支援型：给周围塔加Buff，放在塔群中心
  - 刺客型：高移速 + 单体爆发，用于切后排精英/法师敌人

> **设计意图**：解决传统TD"摆好后无事可做"的核心痛点。移动单位让玩家始终有事做——调整阵型、拦截漏兵、集火BOSS。

### 2.3 创新二：资源链经济系统

- **三种资源**：
  | 资源 | 获取方式 | 用途 |
  |------|----------|------|
  | 金币 G | 击杀敌人 / 金矿建筑 | 建造塔、生产普通单位 |
  | 能量 E | 能量塔自动产出 | 释放技能、激活机关、生产精英单位 |
  | 人口 P | 固定上限 + 可升级 | 限制可移动单位数量 |

- **生产建筑链**：
  ```
  金矿(低级) → 金矿(高级) → 铸币厂(提升全局金币产出效率)
  能量塔(低级) → 能量塔(高级) → 能量枢纽(提升全局能量产出效率)
  ```

- **资源分配决策**：
  - 开局：先铺经济（金矿/能量塔）还是先造防御塔？
  - 中期：升级经济建筑还是造更多攻击塔？
  - 后期：资源溢出后转化为全局Buff？

> **设计意图**：把"资源管理"从简单的"杀敌得钱"升级为有深度的经济小游戏。类似 RTS 的开局运营决策，但更轻量。

### 2.4 创新三：敌方反制 AI

- **敌人可以攻击建筑**：近战敌人经过塔旁会攻击，远程敌人会站桩输出
  - 塔有血量，被摧毁后消失（可花费金币修复）
  - 已摧毁的塔位需要花更多金币重建

- **敌方AI适应性**：
  | 玩家策略 | 敌方反制 |
  |----------|----------|
  | 堆物理塔 | 后续波次出高护甲敌人 |
  | 堆魔法塔 | 后续波次出高魔抗敌人 |
  | 单点集火 | 敌人分兵多路进攻 |
  | 纯塔防不出兵 | 出远程敌人站桩点塔 |
  | 经济流（金矿多） | 出快速突袭单位偷矿 |

- **敌方技能系统**：
  - 精英敌人：1 个技能（如冲锋、护盾、召唤小兵）
  - BOSS 敌人：2-3 个技能 + 阶段转换（血量低于 50% 进入二阶段）
  - 特殊敌人：治疗者（给周围敌人回血）、自爆者（死亡时对周围塔造成伤害）

> **设计意图**：解决"一套阵容打穿所有关卡"的无聊感。迫使玩家根据敌方配置动态调整策略，增加重玩价值。

## 3. 核心玩法闭环

### 3.1 单局循环

```
                波次开始
                   │
    ┌──────────────┴──────────────┐
    │       部署/调整阶段          │
    │  · 建造/升级/修复塔         │
    │  · 部署/移动可移动单位      │
    │  · 使用技能/激活机关        │
    │  · 分配资源                 │
    └──────────────┬──────────────┘
                   │ 玩家点击「开始波次」
    ┌──────────────┴──────────────┐
    │       战斗阶段               │
    │  · 敌人沿路径移动            │
    │  · 塔自动攻击                │
    │  · 玩家操控移动单位          │← 核心操作层
    │  · 实时释放技能              │
    │  · 收集击杀金币              │
    └──────────────┬──────────────┘
                   │ 所有敌人死亡 / 基地被摧毁
    ┌──────────────┴──────────────┐
    │       波次结算               │
    │  · 剩余血量奖励金币          │
    │  · 显示下一波敌人预览        │
    │  · 资源产出结算              │
    └──────────────┬──────────────┘
                   │
         ┌────────┴────────┐
         │ 不是最后一波    │──→ 回到部署阶段
         └─────────────────┘
                   │ 最后一波
    ┌──────────────┴──────────────┐
    │       关卡结算               │
    │  · 星级评定 (1-3星)         │
    │  · 解锁新内容               │
    │  · 排行榜记录               │
    └─────────────────────────────┘
```

### 3.2 游戏模式

#### 关卡模式 (Level Mode)
- 预设地图 + 固定波次 (10-25波)
- 每一关有独特主题和敌人配置
- 三星评定（通关/限时/无塔被摧毁）
- 通关解锁下一关和新内容
- 总计 20-30 关，分 4-5 个主题章节

#### 无尽模式 (Endless Mode)
- 所有已解锁内容可用
- 敌人无限波次，难度递增
- 每 5 波出现 Boss
- 排行榜（存活波数 / 击杀数 / 得分）
- 每日挑战（随机地图 + 随机限制条件）

### 3.3 胜负条件

- **胜利**：消灭关卡所有波次的敌人，基地（水晶/城堡）未被摧毁
- **失败**：基地血量归零（被敌人走到终点 or 被远程敌人摧毁）
- **额外失败条件**（特定关卡）：限时内未消灭所有敌人

## 4. 模块详细设计

### 4.1 建筑系统

#### 4.1.1 攻击塔

| 塔类型 | 主题 | 攻击方式 | 特色机制 |
|--------|------|----------|----------|
| **箭塔** | 物理 | 单体高速 | 升级后增加射程，可切换目标优先级（最近/最远/最弱/最强） |
| **炮塔** | 物理 | 范围AOE | 伤害高但攻速慢，升级后附带击退/减速 |
| **冰塔** | 魔法 | 范围减速 | 叠加减速层数，满层冻结，升级后AOE冰冻 |
| **电塔** | 魔法 | 链式闪电 | 攻击弹跳3-5个敌人，升级后增加弹跳次数和麻痹效果 |
| **毒塔** | 魔法 | 持续伤害 | 叠加中毒层数，每层独立计时，升级后中毒死亡爆炸传染 |
| **光塔** | 魔法 | 单体高伤 | 充能制：充能越久伤害越高，升级后附带破甲 |

#### 4.1.2 生产建筑

| 建筑 | 产出 | 升级效果 |
|------|------|----------|
| 金矿 | 金币/秒 | 升级提升产出速率 |
| 能量塔 | 能量/秒 | 升级提升产出速率 |
| 铸币厂 | 全局金币加成% | 提升全局金币获取 |
| 能量枢纽 | 全局能量加成% | 提升全局能量获取 |

#### 4.1.3 建筑通用属性

```
建筑 {
  id          唯一标识
  type        类型 (attack_tower / production)
  level       等级 (1-5)
  position    地图坐标 (grid_x, grid_y)
  hp          当前血量 / 最大血量
  atk         攻击力
  atk_speed   攻击速度 (次/秒)
  range       攻击范围 (像素)
  cost        建造费用
  upgrade_cost 升级费用表 [L2, L3, L4, L5]
  skill       特殊技能 (部分塔有)
  status      状态 (active / disabled / destroyed)
}
```

#### 4.2.1 单位类型

| 单位 | 定位 | HP | 移速 | 攻击 | 技能 | 人口 |
|------|------|-----|------|------|------|------|

### 4.5 技能与Buff系统

#### 4.5.1 技能类型

| 类型 | 触发方式 | 示例 |
|------|----------|------|
| 主动技能 | 玩家点击释放 + CD | 英雄技能 |
| 被动技能 | 自动触发 | 塔的暴击、吸血 |
| 光环技能 | 持续生效 | 范围Buff/Debuff |
| 条件技能 | 满足条件触发 | 血量低于30%狂暴 |

#### 4.5.2 Buff系统

```
Buff {
  id
  name
  type         (增益/减益)
  attribute    (影响的属性: hp/atk/speed/defense/...)
  value        (数值，支持百分比和绝对值)
  duration     持续时间 (秒)， -1=永久
  stack        最大叠加层数
  source       来源单位ID
  icon         图标标识
}
```

- Buff 叠加规则：同名 Buff 刷新持续时间，可叠加层数上限内叠加
- Buff 显示：单位头顶小图标 + 底部血条颜色变化

### 4.6 战斗数值系统

#### 4.6.1 伤害公式

```
物理伤害 = 攻击力 × (1 - 护甲/(护甲 + 护甲常数))
魔法伤害 = 攻击力 × (1 - 魔抗/(魔抗 + 魔抗常数))

护甲常数 = 100  (可调)
魔抗常数 = 100

最终伤害 = 基础伤害 × (1 + 伤害加成%) × (1 - 伤害减免%) × 暴击倍率(如果是暴击)

暴击率 = 基础暴击率 + Buff暴击率 (上限80%)
暴击倍率 = 1.5 + Buff暴伤加成
```

#### 4.6.2 攻速公式

```
实际攻击间隔 = 基础攻击间隔 / (1 + 攻速加成%)
攻速加成上限 = 200% (即最高3倍攻速)
```

#### 4.6.3 移速公式

```
实际移速 = 基础移速 × (1 + 移速加成%) × (1 - 减速%)
移速下限 = 基础移速 × 20% (不低于20%)
```

### 4.7 地图与关卡系统

#### 4.7.1 地图设计

- 网格系统：地图由 tile 网格组成，每个 tile 有类型
- Tile 类型：
  - `empty` — 空地（可建塔）
  - `path` — 路径（敌人行走）
  - `blocked` — 障碍（不可通行不可建造）
  - `resource` — 资源点（特殊地点）
  - `base` — 基地（敌人终点）
  - `spawn` — 出生点（敌人生成位置）

- 地图尺寸：横屏 16:9 比例，逻辑分辨率 1920×1080
- 每个 tile 尺寸：64×64 像素 → 30×16.875 格 ≈ **30×17 格**

#### 4.7.2 关卡配置

```
关卡 {
  id
  name
  theme          主题 (森林/沙漠/冰原/火山/机械)
  map_data       地图数据 (tile布局 + 中立单位)
  waves          波次配置数组
  starting_gold  初始金币
  starting_energy 初始能量
  starting_pop   初始人口上限
  available_towers 可用塔类型
  available_units  可用移动单位
  star_conditions 三星条件
  unlock_rewards  通关解锁内容
}
```

#### 4.7.3 波次配置

```
波次 {
  wave_number   波次序号
  enemies       敌人编组 [{type, count, spawn_interval}]
  spawn_delay   波次开始延迟(秒)
  special_rules 特殊规则 (如: 全敌人加速50%)
  preview_text  预览提示文字
}
```

| 资源 | 初始值 | 来源 | 用途 |
|------|--------|------|------|
| 金币 | 200 | 杀敌、金矿、宝箱、波次奖励 | 造塔、升级塔、造单位、修建筑 |
| 能量 | 50 | 能量塔、杀精英/BOSS | 释放技能、激活机关 |

**经济节奏设计原则**：

- 中期：经济建筑开始回本，进入"滚雪球"阶段
- 后期：资源溢出，转为全局Buff提升，应对高强度波次

- **配色限制**：主体色 ≤ 5 种，避免视觉噪音

## 10. 验收标准

### 10.1 功能验收标准

| 等级 | 标准 |
|------|------|
| **通过** | 功能按设计运行，无阻塞性 Bug，核心路径无异常 |

- TypeScript strict 模式通过
- 无 `any` / `@ts-ignore`
- ECS 系统职责单一
- 数值计算有单元测试覆盖

### 10.3 体验验收标准

- 60fps 稳定（低端设备 30fps 底线）
- UI 响应延迟 < 100ms
- 交互路径 ≤ 3 步完成核心操作（建造/升级/移动）
