# 调试系统使用指南

## 概述

调试系统提供运行时调试工具，包括：
- **行为树查看器**：可视化查看单位的AI行为树结构和执行状态
- **调试控制台**：查看控制台输出日志

## 快速开始

### 打开调试面板

1. **点击🔧按钮**：屏幕右侧边缘的收起按钮
2. **按F12键**：键盘快捷键切换展开/收起

### 关闭调试面板

1. **点击✕按钮**：面板标题栏右侧的关闭按钮
2. **按Escape键**：收起调试面板

## 行为树查看器

### 功能

- **可视化行为树结构**：以树状图显示行为树节点
- **节点状态高亮**：不同颜色表示节点状态
  - 灰色：空闲
  - 绿色：运行中
  - 蓝色：成功
  - 红色：失败
- **节点详细信息**：悬停在节点上查看参数和状态
- **黑板数据**：查看AI的黑板数据
- **交互操作**：
  - 拖拽：平移视图
  - 滚轮：缩放视图
  - 适应视图：自动调整缩放以显示所有节点
  - 重置视图：恢复默认缩放和位置

### 使用方法

1. 打开调试面板（点击🔧或按F12）
2. 在场景中点击任意单位（敌人、塔、士兵）
3. 行为树查看器自动显示该单位的AI行为树

### 节点类型图标

| 图标 | 节点类型 | 说明 |
|------|----------|------|
| ▶ | 顺序节点 (Sequence) | 依次执行子节点 |
| ? | 选择节点 (Selector) | 依次尝试子节点 |
| ◇ | 条件节点 (Check) | 检查条件 |
| ○ | 动作节点 (Action) | 执行动作 |
| ⬡ | 装饰节点 (Decorator) | 修改子节点行为 |

## 调试控制台

### 功能

- **日志输出**：拦截console输出并显示
- **清除日志**：一键清空日志
- **自动滚动**：新日志自动滚动到底部

### 使用方法

1. 打开调试面板
2. 点击"控制台"标签页
3. 查看日志输出

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `F12` | 切换调试面板展开/收起 |
| `Escape` | 收起调试面板 |

## 调试信息说明

### 行为树节点信息

悬停在行为树节点上会显示：

- **节点名称**：节点的显示名称
- **节点类型**：节点的类型（sequence/selector/check/action等）
- **执行状态**：当前节点的执行状态
- **执行时间**：节点的执行时间（毫秒）
- **参数**：节点的配置参数

### 黑板数据

黑板数据是AI的私有数据存储，包括：

- `target_id`：当前目标实体ID
- `home_x`/`home_y`：回家点坐标
- `patrol_index`：巡逻路径索引
- `wait_until`：等待结束时间
- 自定义数据

## 性能考虑

- 调试面板收起时不会消耗渲染性能
- 行为树查看器仅在展开时渲染
- 建议在不需要时收起调试面板

## 故障排除

### 行为树查看器显示"选择一个单位查看其AI行为树"

1. 确保调试面板已展开
2. 在场景中点击任意单位（敌人、塔、士兵）
3. 确保该单位有AI配置

### 点击单位后没有显示行为树

1. 检查单位是否有AI组件
2. 检查控制台是否有错误信息
3. 确保AI配置已正确注册

### 调试面板不显示

1. 检查是否有JavaScript错误
2. 确保调试管理器已初始化
3. 尝试按F12键

## 扩展调试工具

### 添加自定义日志

```typescript
// 在代码中添加日志（会显示在控制台标签页）
console.log('这是一条日志');
console.warn('警告信息');
console.error('错误信息');
```

### 注册自定义AI配置

```typescript
// 在初始化时注册AI配置
const customAIConfig: BehaviorTreeConfig = {
  id: 'custom_ai',
  name: '自定义AI',
  root: {
    type: 'selector',
    children: [
      // ... 行为树节点
    ]
  }
};

debugManager.registerAIConfigs([customAIConfig]);
```

## 最佳实践

1. **使用有意义的节点名称**：便于在行为树查看器中识别
2. **添加节点参数**：便于调试时查看配置
3. **收起不需要的面板**：减少性能开销
4. **使用快捷键**：提高调试效率
